Why SemPKM

PKM tools force a tradeoff. We don't.

Existing tools make you choose between flexibility and structure. SemPKM gives you both.

The problem

Markdown-first tools are flexible and portable, but structure is implicit and semantics are informal. Object-first tools offer structure and views, but lock you into a platform with no interoperability.

  • Informal structure means no reliable queries or views
  • Platform lock-in means your data isn't truly yours
  • No standard format means fragile integrations

The SemPKM answer

A standards-native semantic datastore with an IDE-grade UX. Your knowledge lives as typed, queryable, interoperable data — and you own all of it.

  • RDF + SPARQL for powerful, reliable queries
  • SHACL shapes drive forms, validation, and linting
  • Export as JSON-LD, project to Obsidian, publish via SOLID
  • Mental Models: install a bundle, get an instant workflow
Features

Everything you need to think in structure

SemPKM combines the power of semantic web standards with an interface designed for humans.

Mental Models

Installable domain bundles — ontology, shapes, views, and projections — that give you an instant, structured PKM experience. No blank-page syndrome.

SHACL-Driven Forms

Forms are auto-generated from SHACL shapes. Constraints drive validation, linting, and layout hints — no custom UI code needed.

Typed Relationships

Edges are first-class: clickable, annotatable, and versionable. Explore your knowledge as a web of typed, meaningful connections.

Graph Visualization

Semantic-aware 2D and 3D graph views with type-based styling, predicate-driven edges, and lenses for focused exploration.

SPARQL-Powered Views

Tables, cards, forms, object pages, and graph views — all backed by SPARQL queries. Compose them into parameterized dashboards.

Standards-Based, No Lock-In

Built on RDF, JSON-LD, SPARQL, and SHACL. Export anything. Project your graph as files for Obsidian. Publish via SOLID and ActivityPub.

Event Sourcing & Audit Trail

Every change is an immutable event. The current state is a derived projection. Full history, complete auditability, automation-ready.

Self-Hosted, Own Your Data

Run SemPKM on your own hardware. Single Docker command. Your knowledge graph stays on your machine — no cloud required.

And More Coming Soon

Filesystem projections for Obsidian, webhook automations, ActivityPub publishing, Mental Model marketplace, and much more on the roadmap.

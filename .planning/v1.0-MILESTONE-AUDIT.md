## Cross-Phase Integration (45/47)

### Verified Connections (45)

All major cross-phase wiring verified:
- EventStore → validation queue → webhook dispatch (complete chain)
- ModelService → PrefixRegistry → LabelService (semantic services)
- ShapesService → form generation → browser CRUD (object lifecycle)
- ViewSpecService → table/cards/graph renderers (view execution)
- AuthService → get_current_user/require_role → all routers (auth enforcement)
- Exception handler → 302 redirects / 403 pages / HTMX fragments (auth UX)
- EventStore.commit(performed_by, performed_by_role) → browser writes (provenance)

### Remaining Gaps (2 — both low severity)

**1. PROV-02 partial: `performed_by_role` missing in API command path**
- `backend/app/commands/router.py` line 124
- `event_store.commit(operations, performed_by=user_iri)` — missing `performed_by_role=user.role`
- Browser writes correctly include both. API writes record user IRI but omit role.
- **Fix:** One-line change

**2. AUTH-03 cosmetic: Invite acceptance ignores `?next=`**
- `frontend/static/js/auth.js` — `handleInviteAccept` function
- Redirects to `/` after success instead of reading `?next=` param
- Other auth paths (login, magic link, auto-verify) correctly use `?next=`
- **Fix:** 2-line change

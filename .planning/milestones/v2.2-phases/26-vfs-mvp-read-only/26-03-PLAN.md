  <action>
    Create backend/app/vfs/cache.py with a module-level TTLCache for directory listings:

    ```python
    """TTL cache for WebDAV directory listing results.

    Prevents SPARQL query on every PROPFIND. Cache is keyed by path string.
    TTL=30s is short enough that edits propagate within one filesystem poll cycle.
    maxsize=256 handles typical PKM installs with many type folders.
    """
    from cachetools import TTLCache

    # Module-level singleton — shared across all collection instances.
    # Thread-safe for read operations; wsgidav calls from a thread pool.
    listing_cache: TTLCache = TTLCache(maxsize=256, ttl=30)
    ```

    Update backend/app/vfs/collections.py to use this cache in `TypeCollection.get_member_names()` and `ModelCollection.get_member_names()`:

    ```python
    from app.vfs.cache import listing_cache

    class TypeCollection(DAVCollection):
        def get_member_names(self) -> list[str]:
            cache_key = f"type:{self._model_id}:{self._type_label}"
            if cache_key in listing_cache:
                return listing_cache[cache_key]
            # ... execute SPARQL, build filename list ...
            listing_cache[cache_key] = result
            return result
    ```

    Apply the same pattern to `RootCollection` and `ModelCollection`. Cache keys:
    - RootCollection: `"root:models"`
    - ModelCollection: `f"model:{model_id}:types"`
    - TypeCollection: `f"type:{model_id}:{type_label}"`

    IMPORTANT: cachetools.TTLCache is NOT thread-safe for writes. Wrap cache writes with a threading.Lock. Add `_cache_lock = threading.Lock()` at module level in cache.py and use it in each `get_member_names()` write:
    ```python
    with _cache_lock:
        listing_cache[cache_key] = result
    ```
    Reads outside the lock are acceptable (worst case: stale miss triggers a SPARQL query).

    from a2wsgi import WSGIMiddleware
    from app.vfs.provider import SemPKMDAVProvider
    from app.vfs.auth import SemPKMWsgiAuthenticator

    At the bottom of main.py (after all other mounts), add the wsgidav mount:

    ```python
    # --- WebDAV VFS Mount ---
    # wsgidav runs in a WSGI thread pool via a2wsgi. SyncTriplestoreClient uses
    # httpx.Client (sync) because wsgidav cannot use async clients.
    # Mounted at /dav — nginx proxies /dav/ to this app with Authorization passthrough.
    _sync_ts_client = SyncTriplestoreClient(
        base_url=settings.triplestore_url,
        repository_id=settings.repository_id,
    )
    _dav_provider = SemPKMDAVProvider(sync_client=_sync_ts_client)

    _dav_config = {
        "provider_mapping": {"/": _dav_provider},
        "http_authenticator": {
            "domain_controller": SemPKMWsgiAuthenticator,
            "accept_basic": True,
            "accept_digest": False,
            "default_to_digest": False,
        },
        "sempkm_db_url": settings.database_url,
        "verbose": 0,
        "logging": {"enable_loggers": []},
        # Read-only: disable all write methods
        "readonly": True,
    }

    _wsgi_dav_app = WsgiDAVApp(_dav_config)
    _asgi_dav_app = WSGIMiddleware(_wsgi_dav_app)
    app.mount("/dav", _asgi_dav_app)
    ```

    Note: The `"readonly": True` config tells wsgidav to return 405 for all write methods (PUT, DELETE, MOVE, COPY, MKCOL, PROPPATCH). This is the correct read-only enforcement — do not rely solely on the DAVProvider to reject writes.

    After adding this, verify the app still starts: check that importing main.py does not raise errors.

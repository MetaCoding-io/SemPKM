<!-- POLSH-03: Object-contextual panel indicator -->

The accent bar must appear on the Relations and Lint panel headers when an object is open in any editor group, and disappear when no object is open.

Implementation approach:
1. Add CSS `.contextual-panel-active .right-section-header` rule with `border-left: 3px solid var(--color-accent)`
2. Add `setContextualPanelActive(isActive)` function in workspace.js that adds/removes `.contextual-panel-active` class on `[data-panel-name]` elements
3. Call `setContextualPanelActive(true)` when a tab is opened (hook into `openTab()` or the htmx `htmx:afterRequest` event after tab load)
4. Call `setContextualPanelActive(false)` when all tabs are closed (hook into removeTab or when activeTabId becomes null)

The `.contextual-panel-active` class should be applied to the `<details>` element (`[data-panel-name]`), not to the header directly. The CSS rule then targets the header as a child:

```css
/* Phase 28 POLSH-03: Object-contextual panel accent bar */
[data-panel-name].contextual-panel-active > .right-section-header,
[data-panel-name].contextual-panel-active > summary.right-section-header {
  border-left: 3px solid var(--color-accent);
  padding-left: 9px; /* Preserve visual alignment: 12px - 3px border = 9px */
}
```

The hook for setContextualPanelActive(true/false):
- In workspace-layout.js, `switchTabInGroup()` is called whenever a tab becomes active. After the function updates the DOM, it can call `window.setContextualPanelActive(true)`.
- When `removeTab()` is called and the result is an empty group (tabs.length === 0 in all groups), call `window.setContextualPanelActive(false)`.
- Alternatively, add a custom event dispatch from workspace-layout.js:
  - After tab switch: `document.dispatchEvent(new CustomEvent('sempkm:tab-activated', { detail: { iri } }))`
  - After all tabs closed: `document.dispatchEvent(new CustomEvent('sempkm:tabs-empty'))`
  - workspace.js listens for these events and calls setContextualPanelActive accordingly.

The custom event approach is cleaner since workspace.js and workspace-layout.js are separate files. Use this approach.

**Step 2: Add setContextualPanelActive() to workspace.js**

Inside the workspace.js IIFE, add this function:

```javascript
// -----------------------------------------------------------------------
// Phase 28: Object-contextual panel indicator (POLSH-03)
// -----------------------------------------------------------------------

/**
 * Toggle the contextual-panel-active class on all [data-panel-name] panels.
 * Called when an object tab is activated (isActive=true) or when all tabs
 * are closed (isActive=false).
 */
function setContextualPanelActive(isActive) {
  document.querySelectorAll('[data-panel-name]').forEach(function(panel) {
    if (isActive) {
      panel.classList.add('contextual-panel-active');
    } else {
      panel.classList.remove('contextual-panel-active');
    }
  });
}

// Listen for tab lifecycle events dispatched by workspace-layout.js
document.addEventListener('sempkm:tab-activated', function() {
  setContextualPanelActive(true);
});

document.addEventListener('sempkm:tabs-empty', function() {
  setContextualPanelActive(false);
});
```

Expose on window:
```javascript
window.setContextualPanelActive = setContextualPanelActive;
```

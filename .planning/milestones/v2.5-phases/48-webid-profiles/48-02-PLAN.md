  <action>
1. Add public profile endpoint to `backend/app/webid/router.py`:
   - Create a SECOND router for public routes: `public_router = APIRouter(tags=["webid-public"])` (no prefix, no auth required)
   - `GET /users/{username}` endpoint:
     - Look up User by username. If not found or `webid_published` is False, return 404.
     - Parse `Accept` header from `request.headers.get("accept", "text/html")`.
     - Compute `base_url` via `get_base_url(request)`.
     - Build profile graph via `build_profile_graph(...)`.
     - If `"text/turtle"` in accept: serialize graph to Turtle (`graph.serialize(format="turtle")`), return `Response(content=..., media_type="text/turtle; charset=utf-8")`.
     - If `"application/ld+json"` in accept: serialize to JSON-LD. Build a WebID-specific JSON-LD context dict with `foaf`, `sec` prefixes (do NOT modify the shared SEMPKM_CONTEXT). Use `graph.serialize(format="json-ld", context=webid_context)`. Return `JSONResponse(content=json.loads(...), media_type="application/ld+json")`.
     - Otherwise (default HTML): parse `webid_links` JSON, compute fingerprint, render `webid/profile.html` template with context: `display_name`, `webid_uri`, `profile_url`, `public_key_pem`, `fingerprint`, `rel_me_links`, `turtle_url` (profile_url with note about Accept header), `jsonld_url`.
   - Register public_router in `main.py` alongside the existing webid_router.

2. Create `backend/app/templates/webid/profile.html`:
   - Standalone HTML page (does NOT extend base.html). Per user decision.
   - DOCTYPE html, lang="en", charset utf-8, viewport meta.
   - `<title>{{ display_name }} - WebID Profile</title>`
   - `<head>` contains:
     - `<link rel="me" href="{{ link }}">` for each link in `rel_me_links`
     - `<link rel="alternate" type="text/turtle" href="{{ profile_url }}">`
     - `<link rel="alternate" type="application/ld+json" href="{{ profile_url }}">`
   - Inline `<style>` block with:
     - CSS custom properties for light/dark themes
     - `@media (prefers-color-scheme: dark)` overrides
     - Clean, minimal layout: centered content, max-width ~600px
     - SemPKM branding (text logo, not image -- just "SemPKM" in a styled span)
   - Body content:
     - SemPKM logo/brand header
     - User's display name as `<h1>`
     - WebID URI displayed as monospace text with copy button (JS `navigator.clipboard`)
     - Public key fingerprint (colon-separated hex, truncated display with "Copy PEM" button)
     - List of profile links (rendered as clickable `<a>` tags with `rel="me"`)
     - Footer with alternate format links: "View as [Turtle] | [JSON-LD]" -- these link to the same URL but with a `?format=turtle` / `?format=jsonld` query param as a convenience (in addition to Accept header). Update the endpoint to also check `request.query_params.get("format")` as a fallback.
   - Copy PEM button: small JS snippet that copies `public_key_pem` to clipboard (embed PEM in a hidden `<pre>` or data attribute).

3. Check if nginx needs a `/users/` location block. The existing catch-all `location /` proxies to FastAPI, so `/users/{username}` will be proxied automatically. Only add a specific nginx block if the Accept header needs forwarding (it does get forwarded by default via proxy_pass). Verify the `Accept` header is passed through -- it should be, since nginx proxies all headers by default. No nginx change needed unless testing reveals otherwise.

<task type="auto">
  <name>Task 2: Create IndieAuth service with code, token, and introspection logic</name>
  <files>backend/app/indieauth/service.py</files>
  <action>
    Create `backend/app/indieauth/service.py` with class `IndieAuthService`:

    Constructor takes `db: AsyncSession`.

    Methods:

    1. `verify_pkce_s256(code_verifier: str, code_challenge: str) -> bool` (static):
       - SHA-256 hash the verifier, base64url encode WITHOUT padding (strip `=`), compare to challenge.
       - Per research pitfall 1: use `base64.urlsafe_b64encode(digest).rstrip(b"=").decode("ascii")`.

    2. `validate_client_id(client_id: str) -> bool` (static):
       - Per IndieAuth spec Section 3.2: scheme must be http or https, path must exist and not be empty, no fragment, no userinfo, no single/double dot path segments.

    3. `async fetch_client_info(client_id: str) -> ClientInfo`:
       - Use httpx.AsyncClient with 5s timeout to GET client_id URL.
       - Parse response with `mf2py.parse()`, look for h-app microformat.
       - Extract name, url, logo from h-app properties.
       - Fallback: extract `<title>` tag from HTML, then hostname from URL.
       - On any error: return ClientInfo with name=client_id, url=client_id, logo=None.
       - Guard against SSRF: reject loopback IPs (127.0.0.0/8, ::1) in client_id hostname.

    4. `async create_authorization_code(user_id: UUID, client_id: str, redirect_uri: str, scope: str, code_challenge: str, code_challenge_method: str, state: str | None) -> str`:
       - Generate code via `secrets.token_urlsafe(32)`.
       - Store SHA-256 hash in `indieauth_codes` with 60-second expiry.
       - Return plaintext code (given to client once).

    5. `async exchange_code(code: str, client_id: str, redirect_uri: str, code_verifier: str) -> tuple[IndieAuthToken, str, str | None]`:
       - Look up code by SHA-256 hash, verify not expired, verify client_id and redirect_uri match.
       - Verify PKCE: `verify_pkce_s256(code_verifier, stored_code_challenge)`.
       - Delete code row (single-use per research pitfall 4).
       - Generate access token (1h TTL) and refresh token (30d TTL) via `secrets.token_urlsafe(32)`.
       - Store token hashes in `indieauth_tokens`.
       - Return (token_row, access_token_plaintext, refresh_token_plaintext).
       - Raise ValueError with descriptive message on any validation failure.

    6. `async refresh_access_token(refresh_token: str) -> tuple[IndieAuthToken, str, str | None]`:
       - Look up by refresh_token_hash, verify not expired.
       - Delete old token row.
       - Create new access token (1h) and new refresh token (30d) -- rotation.
       - Return (new_token_row, new_access_token, new_refresh_token).

    7. `async introspect_token(token: str) -> IntrospectionResponse`:
       - Look up by token_hash. If not found or expired, return `IntrospectionResponse(active=False)`.
       - If valid, load user for `me` URL construction. Return active=True with me, client_id, scope, exp, iat.
       - Need to construct `me` URL: import `build_profile_url` from webid.service.

    8. `async revoke_token(token_hash: str) -> bool`:
       - Delete token row by token_hash. Return True if found and deleted.

    9. `async list_authorized_apps(user_id: UUID) -> list[IndieAuthToken]`:
       - Return all non-expired tokens for user, ordered by created_at desc.

    10. `async cleanup_expired()`:
        - Delete expired codes and tokens. Called opportunistically or via background task.

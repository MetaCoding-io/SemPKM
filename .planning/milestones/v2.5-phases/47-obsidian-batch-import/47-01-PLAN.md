**1. Add ImportResult dataclass to models.py:**

```python
@dataclass
class ImportResult:
    created: int = 0
    skipped_existing: int = 0
    skipped_errors: int = 0
    edges_created: int = 0
    unresolved_links: list[tuple[str, str]] = field(default_factory=list)  # (source_path, target_name)
    errors: list[tuple[str, str]] = field(default_factory=list)  # (path, error_message)
    duration_seconds: float = 0.0

    def to_dict(self) -> dict[str, Any]:
        return {
            "created": self.created,
            "skipped_existing": self.skipped_existing,
            "skipped_errors": self.skipped_errors,
            "edges_created": self.edges_created,
            "unresolved_links": [{"source": s, "target": t} for s, t in self.unresolved_links],
            "errors": [{"path": p, "message": m} for p, m in self.errors],
            "duration_seconds": self.duration_seconds,
        }
```

**2. Update broadcast.py** -- make `stream_sse()` accept a `terminal_events` parameter (default `{"scan_complete", "scan_error"}`) so import can reuse the same function with `{"import_complete", "import_error"}` terminal events. Keep backward-compatible by defaulting.

**3. Create executor.py** with `ImportExecutor` class:

Constructor takes: `scan_result: VaultScanResult`, `mapping_config: MappingConfig`, `extract_path: Path`, `event_store: EventStore`, `triplestore_client: TriplestoreClient`, `user: User`, `broadcast: ScanBroadcast`.

Main method: `async def execute(self) -> ImportResult`

Implementation details:
- **Enhanced wiki-link regex** that captures alias: `WIKILINK_FULL_RE = re.compile(r"(?<!!)\[\[([^\]\|#]+)(?:#[^\]\|]*)?\s*(?:\|([^\]]*))?\]\]")`
- Reuse `TAG_RE` and `CODE_BLOCK_RE` from scanner.py (import them).
- **Deduplication query** at start: SPARQL SELECT for all `sempkm:importSource` values in `urn:sempkm:current` graph. Build a Python set. Skip notes whose relative vault path is already in that set.
- **Pass 1 (Objects):** For each note type group with a type mapping (skip None mappings):
  - For each note file in the group's `sample_notes` list... NO -- iterate ALL markdown files in the vault, matching them to type groups by the scanner's type detection logic. The `sample_notes` only contains up to 10 samples. Instead:
    - Iterate all `.md` files under `extract_path` using `pathlib.Path.rglob("*.md")`
    - For each file, determine which type group it belongs to by matching the scanner's detection signals (frontmatter type field, parent folder, first tag). Use the SAME logic as VaultScanner._detect_type() to determine the group key (`type_name|signal_source`).
    - Look up type mapping from `mapping_config.type_mappings[group_key]`. If None or missing, skip.
    - Read note with `python-frontmatter`: extract metadata, body, wiki-links (with alias), and tags.
    - Build `ObjectCreateParams` with:
      - `type`: the `target_type_iri` from the mapping
      - `slug`: None (let UUID minting handle it to avoid slug collisions per Pitfall 4)
      - `properties`: mapped frontmatter keys per `mapping_config.property_mappings[type_iri]`, plus `"sempkm:importSource": relative_vault_path` for deduplication, plus `"dcterms:title": stem_name` (filename without extension) as the title
    - Call `handle_object_create()` to get an Operation.
    - **Multi-valued tags**: After getting the create Operation, add extra triples for `schema:keywords` directly to `create_op.data_triples` and `create_op.materialize_inserts` (one Literal per tag). Use `_resolve_predicate("schema:keywords")` for the predicate.
    - If body text is non-empty, call `handle_body_set()` to get a body Operation.
    - Commit both operations via `event_store.commit([create_op, body_op], performed_by=user_iri, performed_by_role=user.role)`.
    - Build `filename_to_iri` lookup: `{stem.lower(): object_iri}` for Pass 2.
    - Broadcast `import_progress` SSE event with `phase: "objects"`, `current: i`, `total: total_notes`, `current_file: relative_path`.
    - On exception per note: catch, log, increment `skipped_errors`, add to `errors` list, continue.

- **Pass 2 (Edges):** For each note that was successfully imported:
    - For each wiki-link `(target_name, alias)` extracted during Pass 1:
      - Look up `target_name.lower()` in `filename_to_iri`. If not found, add to `unresolved_links`, continue.
      - Build `EdgeCreateParams` with `source=source_iri`, `target=target_iri`, `predicate="dcterms:references"`, `properties={"rdfs:label": alias}` if alias else `{}`.
      - Call `handle_edge_create()`, commit via event_store. Batch 10 edges per commit for efficiency.
      - Broadcast `import_progress` with `phase: "edges"`, count, current link info.
    - On exception: catch, log, continue.

- **Terminal event:** Broadcast `import_complete` with ImportResult.to_dict() data, or `import_error` on catastrophic failure.

- **Persist ImportResult:** Write `import_result.json` to import directory for the summary endpoint.

- **User IRI pattern:** `URIRef(f"urn:sempkm:user:{user.id}")` (same as command router pattern).

**IMPORTANT: Type detection for all files.**
The scanner detects types using multiple signals. To avoid reimplementing detection, extract a helper function from scanner.py or replicate the detection logic in the executor. The simplest approach: during scan, the scanner already built `NoteTypeGroup` objects with `sample_notes` limited to 10 -- but the actual note-to-group mapping isn't persisted. The executor must re-detect.

Create a method `_detect_group_key(self, rel_path: str, meta: dict, body: str) -> str | None` that mirrors the scanner's 4-signal detection:
1. Frontmatter `type` or `category` field -> `type_name|frontmatter:fieldname`
2. Parent folder name -> `folder_name|folder:folder_name`
3. First tag in body -> `tag_value|tag:tag_value`
4. Fallback: `Uncategorized|uncategorized`

Match the detected group_key against `mapping_config.type_mappings` keys to find the mapping.

  <action>
    1. Create `backend/app/browser/__init__.py` (empty).

    2. Create `backend/app/browser/router.py`:
       - Router prefix: `/browser`
       - `GET /browser/` -> render `browser/workspace.html` (full page with base.html)
         * Query ShapesService.get_types() to get available object types for the nav tree
         * Pass types list to template context
         * If HX-Request header: return `content` block only

    3. Create `backend/app/templates/browser/workspace.html`:
       - Extends `base.html`
       - Load additional CSS: workspace.css
       - Load additional JS: Split.js (CDN), ninja-keys (CDN), workspace.js
       - CDN links per research:
         * `<script src="https://unpkg.com/split.js@1.6.5/dist/split.min.js"></script>`
         * `<script type="module" src="https://unpkg.com/ninja-keys?module"></script>`
       - Content block structure:
         * Three-column container with IDs: `#nav-pane`, `#editor-pane`, `#right-pane`
         * Each pane has a header bar and content area:
           - Left pane: "Explorer" header + `#nav-tree` content area
           - Center pane: Tab bar (`#tab-bar`) at top + `#editor-area` content area below
           - Right pane: "Properties" header + right pane tabs (Properties, Relations, Lint) + `#right-content` area
         * Include `<ninja-keys></ninja-keys>` element at end of body

    4. Create `backend/app/templates/components/_tabs.html`:
       - Jinja2 partial for the tab bar component
       - Tab bar renders a list of open tabs with: label text, dirty indicator (dot), close button
       - Active tab highlighted
       - Empty state: "No objects open" message in editor area
       - Tab data driven by JS (sessionStorage), not server state (per research anti-pattern guidance)

    5. Create `frontend/static/css/workspace.css`:
       - Three-column flex layout for workspace container
       - Split.js gutter styling: 6px wide, cursor col-resize, subtle background color
       - Tab bar styles: horizontal tab list, active tab highlight, dirty dot indicator, close button
       - Pane header styles: fixed height, border-bottom, title text
       - Right pane sub-tabs: Properties | Relations | Lint tab buttons
       - Nav tree: indented tree items, type nodes bold, object leaves with hover highlight
       - Overall: dark sidebar theme (matching IDE feel), light content area

    6. Create `frontend/static/js/workspace.js`:
       - Split.js initialization (per research code example):
         * `Split(['#nav-pane', '#editor-pane', '#right-pane'], { sizes: [20, 50, 30], minSize: [180, 300, 200], gutterSize: 6 })`
         * Save/restore sizes from localStorage (`sempkm_pane_sizes`)
       - Tab management (per research Pattern 4):
         * `TAB_KEY = "sempkm_open_tabs"` in sessionStorage
         * `openTab(objectIri, label)`: add tab to state, render tab bar, load object content via htmx
         * `closeTab(objectIri)`: remove tab from state, render tab bar, load next tab or empty state
         * `switchTab(objectIri)`: mark tab active, load content
         * `markDirty(objectIri)`: set dirty flag, update tab dot indicator
         * `markClean(objectIri)`: clear dirty flag
         * `renderTabBar()`: re-render tab bar from sessionStorage state
       - Keyboard shortcuts:
         * `Ctrl+S` / `Cmd+S`: save current object (prevent default browser save)
         * `Ctrl+W` / `Cmd+W`: close current tab (prevent default browser close)
       - Export functions globally for htmx onclick handlers

  <action>
    1. Create `backend/app/templates/browser/nav_tree.html`:
       - htmx partial template (no base.html extension, just content)
       - Renders type nodes as expandable tree items:
         * Each type node: icon (folder), label (type name from ShapesService), expand/collapse toggle
         * `hx-get="/browser/tree/{{ type_iri | urlencode }}"` `hx-trigger="click once"` `hx-target="#children-{{ type_iri | urlencode }}"` `hx-swap="innerHTML"`
         * Children container: `<div id="children-{{ type_iri | urlencode }}" class="tree-children"></div>`
       - Object leaf nodes (returned by the tree endpoint):
         * Each leaf: icon (file), label (object label), click handler to openTab()
         * `onclick="openTab('{{ object_iri }}', '{{ object_label }}')"` or use htmx to trigger tab open

    2. Add navigation tree endpoints to `backend/app/browser/router.py`:
       - `GET /browser/tree/{type_iri}` -> query objects of the given type from the current graph via SPARQL:
         ```sparql
         PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>
         SELECT ?obj ?label WHERE {
           ?obj rdf:type <{type_iri}> .
           OPTIONAL { ?obj <http://purl.org/dc/terms/title> ?label }
         }
         ```
         Use LabelService to resolve labels for objects. Return rendered HTML list of tree leaf nodes.
       - Render objects as clickable tree leaves that call `openTab(iri, label)` in JS

    3. Update workspace.html to include the nav tree on initial load:
       - Render type nodes in the nav pane using the types list from ShapesService.get_types()
       - Include the nav_tree.html template for initial type listing

    4. Register command palette actions in `frontend/static/js/workspace.js`:
       - Per research Pattern (ninja-keys):
         ```javascript
         const ninja = document.querySelector('ninja-keys');
         ninja.data = [
           { id: 'new-object', title: 'New Object', section: 'Objects', hotkey: 'ctrl+n', handler: () => showTypePicker() },
           { id: 'run-validation', title: 'Run Validation', section: 'Tools', hotkey: 'ctrl+shift+v', handler: () => triggerValidation() },
           { id: 'toggle-nav', title: 'Toggle Navigation Panel', section: 'View', hotkey: 'ctrl+b', handler: () => togglePane('nav-pane') },
           { id: 'toggle-right', title: 'Toggle Properties Panel', section: 'View', handler: () => togglePane('right-pane') },
         ];
         ```
       - `showTypePicker()`: placeholder function that will be implemented in Plan 05 (SHACL forms). For now, log to console or show an alert.
       - `triggerValidation()`: placeholder that will be wired in Plan 06. For now, log to console.
       - `togglePane(paneId)`: toggle visibility of a pane by collapsing its Split.js size to 0 or restoring it
       - Dynamically add "Open: {object label}" entries to command palette as objects are loaded in the nav tree

    5. Initialize command palette on page load:
       - Wait for ninja-keys custom element to be defined before setting data
       - Use `customElements.whenDefined('ninja-keys').then(...)` or DOMContentLoaded

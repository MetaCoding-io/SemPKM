<task type="auto">
  <name>Task 1: Admin router and model management UI</name>
  <files>
    backend/app/admin/__init__.py
    backend/app/admin/router.py
    backend/app/templates/admin/index.html
    backend/app/templates/admin/models.html

  <action>
    1. Create `backend/app/admin/__init__.py` (empty).

    2. Create `backend/app/admin/router.py`:
       - Use `Jinja2Blocks` from `jinja2_fragments.fastapi` (share templates instance from app.state or create new pointing to same directory)
       - Router prefix: `/admin`

       Model management endpoints:
       - `GET /admin/` -> render `admin/index.html` (admin landing page with links to Models and Webhooks sub-pages)
       - `GET /admin/models` -> render `admin/models.html` with list of installed models from ModelService
         * If HX-Request header: return only the `content` block (htmx partial)
         * Otherwise: return full page with base.html shell
       - `POST /admin/models/install` -> accept form data with `path` field, call `model_service.install(Path(path))`, render updated model table partial. On success: return the models table HTML (htmx swaps it in). On error: return error message in an alert div.
       - `DELETE /admin/models/{model_id}` -> call `model_service.remove(model_id)`, return updated model table partial. Handle 404 (not installed) and 409 (user data exists) errors with appropriate messages.

       Use FastAPI Depends for `get_model_service` from dependencies.py.

    3. Create `backend/app/templates/admin/index.html`:
       - Extends `base.html`
       - Content block: Admin Portal heading, two cards linking to Models (`/admin/models`) and Webhooks (`/admin/webhooks`)
       - Cards use hx-get with hx-target="#app-content" for SPA navigation

    4. Create `backend/app/templates/admin/models.html`:
       - Extends `base.html`
       - Content block named `content`:
         * Heading: "Mental Models"
         * Install form: text input for model path + "Install" button
           - `hx-post="/admin/models/install"` with `hx-target="#model-table"` `hx-swap="outerHTML"`
           - Form enctype `application/x-www-form-urlencoded` (standard form)
         * Model table (`id="model-table"`):
           - Columns: Name, Version, Description, Installed, Actions
           - Each row shows model data from ModelService.list_models()
           - Actions column: "Remove" button with `hx-delete="/admin/models/{model_id}"` `hx-target="#model-table"` `hx-swap="outerHTML"` `hx-confirm="Remove model {name}?"`
         * Error/success message area (`id="admin-messages"`) for feedback
         * Use `hx-indicator` for loading state during install/remove

    5. Update `backend/app/main.py`:
       - Import and include admin router: `from app.admin.router import router as admin_router`
       - `app.include_router(admin_router)`

    6. Update `backend/app/shell/router.py`:
       - Remove the placeholder `/admin/` endpoint (now handled by admin router)
  </action>
  <verify>
    With containers running:
    - `curl http://localhost:3000/admin/` returns admin index page
    - `curl http://localhost:3000/admin/models` returns model management page with Basic PKM listed
    - `curl -X POST http://localhost:3000/admin/models/install -d "path=/app/models/basic-pkm"` returns updated table (or error if already installed)
    - The model table displays Name, Version, Description, Installed date, and Remove button for each model
  </verify>
  <done>
    Admin portal serves model management page at `/admin/models` with a table of installed models and Install/Remove actions. htmx handles form submissions and table updates without full page reload. Install and Remove operations call ModelService and return updated table partials.
  </done>
</task>

<task type="auto">
  <name>Task 2: Webhook configuration UI</name>
  <files>
    backend/app/templates/admin/webhooks.html
    backend/app/admin/router.py
  </files>
  <action>
    1. Add webhook endpoints to `backend/app/admin/router.py`:
       - `GET /admin/webhooks` -> render `admin/webhooks.html` with list of configured webhooks from WebhookService
         * If HX-Request header: return only the `content` block
       - `POST /admin/webhooks` -> accept form data with `target_url`, `events` (multi-select), optional `filters` field. Call `webhook_service.create_config()`. Return updated webhook list partial.
       - `DELETE /admin/webhooks/{webhook_id}` -> call `webhook_service.delete_config()`. Return updated webhook list partial.
       - `POST /admin/webhooks/{webhook_id}/toggle` -> toggle enabled/disabled status. Call `webhook_service.update_config(webhook_id, enabled=not current_state)`. Return updated webhook list partial.

       Use FastAPI Depends for `get_webhook_service` from dependencies.py.

    2. Create `backend/app/templates/admin/webhooks.html`:
       - Extends `base.html`
       - Content block:
         * Heading: "Webhooks"
         * Create webhook form:
           - Text input for target URL (required)
           - Checkboxes for event types: `object.changed`, `edge.changed`, `validation.completed` (per CONTEXT.md decision)
           - Optional text input for filters (comma-separated type IRIs or namespaces)
           - "Create" button with `hx-post="/admin/webhooks"` `hx-target="#webhook-list"` `hx-swap="outerHTML"`
         * Webhook list (`id="webhook-list"`):
           - Each webhook shows: target URL, subscribed events (as tags/badges), filters, enabled/disabled toggle
           - Actions: Enable/Disable toggle button, Delete button
           - Delete uses `hx-delete="/admin/webhooks/{id}"` `hx-target="#webhook-list"` `hx-swap="outerHTML"` `hx-confirm="Delete webhook?"`
           - Toggle uses `hx-post="/admin/webhooks/{id}/toggle"` `hx-target="#webhook-list"` `hx-swap="outerHTML"`
         * Empty state: "No webhooks configured" message when list is empty

    3. Note: Webhook dispatch wiring into the command flow is handled in Plan 04-02 Task 3 (not here). This plan only delivers the CRUD UI for webhook configuration.
  </action>

    - `curl -X POST http://localhost:3000/admin/webhooks -d "target_url=https://httpbin.org/post&events=object.changed&events=edge.changed"` creates a webhook and returns updated list
    - `curl -X DELETE http://localhost:3000/admin/webhooks/{id}` removes the webhook
    - The webhook list displays URL, events, and action buttons
  </verify>
  <done>

  </done>

2. **backend/app/events/store.py** — Modify `EventStore.commit()` to accept optional user identity:
   - Change signature: `async def commit(self, operations: list[Operation], performed_by: URIRef | None = None) -> EventResult`
   - After adding the existing event metadata triples (type, timestamp, operation, affected, description), add:
     ```python
     if performed_by:
         event_triples.append((event_iri, EVENT_PERFORMED_BY, performed_by))
     ```
   - Import `EVENT_PERFORMED_BY` from `app.events.models`
   - This is an additive, backward-compatible change. Callers that don't pass `performed_by` (like model auto-install during startup) produce events with no user attribution. Per Research Pitfall 6: treat absence as "system/unknown".
   - The `performed_by` IRI follows the pattern `urn:sempkm:user:{user_uuid}` (constructed by the caller, not by EventStore).
  </action>
  <verify>
    - `docker compose exec api python -c "from app.events.models import EVENT_PERFORMED_BY; print('EVENT_PERFORMED_BY:', EVENT_PERFORMED_BY)"` prints the IRI
    - `docker compose exec api python -c "from app.events.store import EventStore; import inspect; sig = inspect.signature(EventStore.commit); print('commit params:', list(sig.parameters.keys()))"` shows `['self', 'operations', 'performed_by']`
  </verify>
  <done>
    EVENT_PERFORMED_BY constant defined. EventStore.commit() accepts optional performed_by URIRef parameter and includes it in event metadata when provided. Backward compatible -- existing callers without performed_by still work.
  </done>

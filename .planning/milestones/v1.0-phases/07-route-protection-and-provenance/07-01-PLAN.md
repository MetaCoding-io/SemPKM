**2. Add custom HTTPException handler to main.py**:
- Add imports: `from urllib.parse import quote`, `from fastapi.responses import HTMLResponse, JSONResponse, RedirectResponse`, `from fastapi import HTTPException`
- Add a helper `_is_html_route(path: str) -> bool` that returns False for paths starting with "/api/" and True for everything else (all non-API routes are HTML)
- Register `@app.exception_handler(HTTPException)` AFTER the app is created (after line `app = FastAPI(...)`)
- Handler logic for 401:
  - If HTMX request (HX-Request header == "true"): return `HTMLResponse(content='<div class="auth-error">Session expired. <a href="/login.html">Log in again</a></div>', status_code=401)`
  - Else (full page): return `RedirectResponse(url=f"/login.html?next={quote(str(request.url.path), safe='/')}", status_code=302)`
- Handler logic for 403:
  - If HTMX request: return `HTMLResponse(content='<div class="auth-error">Access denied. You do not have permission for this action.</div>', status_code=403)`
  - Else: render the 403.html template via `templates.TemplateResponse(request, "errors/403.html", {"request": request}, status_code=403)` where templates is from `app.state.templates`
- For non-HTML routes OR any other status code: return `JSONResponse(status_code=exc.status_code, content={"detail": exc.detail})`
- IMPORTANT per Research Pitfall 2: API routes (/api/*) must ALWAYS fall through to JSON responses. Do NOT intercept them.

**3. Extend EventStore with performed_by_role** (`backend/app/events/models.py`):
- Add `EVENT_PERFORMED_BY_ROLE = SEMPKM.performedByRole`
- Add `SYSTEM_ACTOR_IRI = URIRef("urn:sempkm:system")` (import URIRef from rdflib)

**4. Extend EventStore.commit() signature** (`backend/app/events/store.py`):

  <action>
**Add to `service.py`** -- `execute_cards_query` method on ViewSpecService:
- Signature: `async def execute_cards_query(self, spec: ViewSpec, page: int, page_size: int, filter_text: str, group_by: str | None) -> dict`
- Use `scope_to_current_graph()` on the spec's SPARQL query.
- Two-phase approach (same as table): (1) get distinct ?s subjects with LIMIT/OFFSET for pagination, (2) fetch all properties for those subjects.
- For each object, also query outbound and inbound relationships (edges where the object is subject or object) using SPARQL. Reuse the same pattern as `browser/router.py` `get_relations` endpoint but batch it for all objects on the page.
- Resolve labels via LabelService for all IRIs (object labels, predicate labels, target labels).
- For body snippets: check for `urn:sempkm:body` property on each object and truncate to first ~100 chars. Also check for `dcterms:description` as a fallback snippet source.
- If `group_by` is provided (a property IRI), group the cards by that property's value. Return groups as `list[dict]` with `group_label` and `cards` keys.
- Return dict: `cards` (list of card data dicts with keys: iri, label, snippet, properties list, outbound_relations list, inbound_relations list), `total`, `page`, `page_size`, `total_pages`, `groups` (if group_by specified), `group_by` (the property IRI or None).
- Each card's `properties` is a list of `{name, value}` dicts. Each relation is `{predicate_label, target_iri, target_label, direction}`.

**Add to `router.py`** -- `GET /browser/views/cards/{spec_iri:path}`:
- Query params: `page` (default 1), `page_size` (default 12 -- cards show fewer per page), `filter` (default ""), `group_by` (default None -- optional property IRI for grouping).
- Calls `execute_cards_query` and renders `cards_view.html`.
- Include the view_toolbar.html at the top with the same filter input and view type switcher.
- The `group_by` param is passed as a query param. The toolbar should show a "Group by" dropdown populated from the spec's columns (the SHACL properties available for this type). When a group_by property is selected, re-request the cards view with that group_by param.

**`backend/app/templates/browser/cards_view.html`**: Server-rendered card grid.
- Include `view_toolbar.html` at the top. Add a "Group by" select dropdown in the toolbar area populated from available columns/properties. The select triggers `hx-get` to reload the cards view with the selected `group_by` param.
- If `groups` is set (grouping active), render groups with group headers (`<h3>` with group label) and card grids per group.
- If no grouping, render a flat card grid.
- Each card uses the CSS flip pattern from Research:
  ```html
  <div class="flip-card" data-iri="{{ card.iri }}">
    <div class="flip-card-inner">
      <div class="flip-card-front">
        <div class="card-title">{{ card.label }}</div>
        <div class="card-snippet">{{ card.snippet }}</div>
        <button class="flip-toggle" onclick="this.closest('.flip-card').classList.toggle('flipped')" title="Flip card">&#x21BB;</button>
      </div>
      <div class="flip-card-back">
        <div class="card-back-header">
          <span class="card-back-title">{{ card.label }}</span>
          <button class="flip-toggle" onclick="this.closest('.flip-card').classList.toggle('flipped')" title="Flip card">&#x21BB;</button>
        </div>
        <div class="card-properties">
          {% for prop in card.properties %}
            <div class="card-prop"><span class="card-prop-name">{{ prop.name }}:</span> {{ prop.value }}</div>
          {% endfor %}
        </div>
        <div class="card-relations">
          {% for rel in card.outbound_relations %}
            <div class="card-rel">
              <span class="card-rel-dir">&#x2192;</span>
              <span class="card-rel-predicate">{{ rel.predicate_label }}</span>
              <a onclick="openTab('{{ rel.target_iri }}', '{{ rel.target_label }}')">{{ rel.target_label }}</a>
            </div>
          {% endfor %}
          {% for rel in card.inbound_relations %}
            <div class="card-rel">
              <span class="card-rel-dir">&#x2190;</span>
              <span class="card-rel-predicate">{{ rel.predicate_label }}</span>
              <a onclick="openTab('{{ rel.target_iri }}', '{{ rel.target_label }}')">{{ rel.target_label }}</a>
            </div>
          {% endfor %}
        </div>
      </div>
    </div>
  </div>
  ```
- Clicking the card title on the front opens the object in a tab: `onclick="openTab('{{ card.iri }}', '{{ card.label }}')"`.
- Include `pagination.html` at the bottom (reuse from 05-01). Pass the same pagination context (page, total_pages, etc.) with the cards endpoint URL.

**Add to `frontend/static/css/views.css`** -- Card flip and grid styles:
- `.card-grid`: CSS grid with `grid-template-columns: repeat(auto-fill, minmax(280px, 1fr))`, gap 16px, padding. Responsive: on screens < 640px, single column.
- `.flip-card`: `perspective: 1000px`, fixed height 260px, cursor default.
- `.flip-card-inner`: `position: relative`, width/height 100%, `transition: transform 0.6s`, `transform-style: preserve-3d`.
- `.flip-card.flipped .flip-card-inner`: `transform: rotateY(180deg)`.
- `.flip-card-front, .flip-card-back`: `position: absolute`, width/height 100%, `backface-visibility: hidden`, border-radius 8px, overflow hidden/auto, padding 16px, background white, box-shadow `0 2px 8px rgba(0,0,0,0.1)`.
- `.flip-card-back`: `transform: rotateY(180deg)`, overflow-y auto (scrollable for lots of properties).
- `.flip-toggle`: position absolute, top-right corner, small circular button with rotation icon, z-index above content.
- `.card-title`: font-size 1.1em, font-weight 600, margin-bottom 8px, cursor pointer (clickable to open object).
- `.card-snippet`: font-size 0.9em, color #666, line-clamp 4 lines with `-webkit-line-clamp`.
- `.card-prop-name`: font-weight 600, color #555.
- `.card-rel`: flex row, gap 4px, font-size 0.85em.
- `.card-rel-predicate`: color #888, font-style italic.
- `.card-rel a`: color #2d5a9e, cursor pointer, text-decoration underline on hover.
- `.card-group-header`: margin-top 16px, font-size 1.1em, font-weight 600, border-bottom 1px solid #ddd.
- CRITICAL per Research anti-pattern: Only animate `transform` and `opacity` during flip. Never animate `width`, `height`, `box-shadow`, or `filter` during transition.

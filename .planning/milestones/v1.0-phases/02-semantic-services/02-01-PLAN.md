<objective>
Create the prefix registry and label resolution service -- the two infrastructure services that all downstream UI phases depend on for human-readable IRI display.

Purpose: Phase 4 and 5 UI surfaces need to display human-readable labels instead of raw IRIs. The prefix registry enables QName rendering (e.g., `dcterms:title` instead of `http://purl.org/dc/terms/title`), and the label service resolves IRIs to their best human-readable label using a configurable precedence chain with batch resolution and caching. These services must be complete and testable via API before downstream phases can use them.

Output: `backend/app/services/prefixes.py` (PrefixRegistry), `backend/app/services/labels.py` (LabelService), updated dependencies and lifespan wiring.
</objective>

3. Create `backend/app/services/prefixes.py` with class `PrefixRegistry`:
   - **BUILTIN dict** (~11 prefixes): rdf, rdfs, owl, xsd, sh, skos, dcterms, schema, foaf, prov, sempkm. Namespaces must match rdflib's canonical URIs exactly. Use the values from `COMMON_PREFIXES` in namespaces.py where they exist, plus add owl (`http://www.w3.org/2002/07/owl#`), sh (`http://www.w3.org/ns/shacl#`), foaf (`http://xmlns.com/foaf/0.1/`), prov (`http://www.w3.org/ns/prov#`).
   - **Three instance dicts**: `_model_prefixes: dict[str, str]` (populated by Mental Model install in Phase 3), `_user_prefixes: dict[str, str]` (user overrides, persisted in triplestore named graph `urn:sempkm:prefixes` in Phase 3+), `_lov_prefixes: dict[str, str]` (populated by LOV import).
   - **Precedence for `_lookup_prefix(prefix) -> Optional[str]`**: user > model > LOV > built-in.
   - **`expand(qname: str) -> Optional[str]`**: Split on first ":", look up prefix, return full IRI or None.
   - **`compact(iri: str) -> str`**: Build reverse map (namespace->prefix) from all layers respecting precedence, find longest matching namespace, return `prefix:localName`. If no match, return original IRI.
   - **Reverse map caching**: `_reverse_cache: Optional[dict[str, str]]` invalidated (set to None) whenever any layer changes. Rebuilt lazily in `_get_reverse_map()`.
   - **`register_model_prefixes(prefixes: dict[str, str])`**: Merge into `_model_prefixes`, invalidate reverse cache.
   - **`register_user_prefix(prefix: str, namespace: str)`**: Add to `_user_prefixes`, invalidate reverse cache.
   - **`get_all_prefixes() -> dict[str, str]`**: Return merged dict (built-in, then LOV, then model, then user -- later layers override earlier).
   - **`async import_lov_prefixes(http_client: httpx.AsyncClient) -> int`**: Fetch from `https://lov.linkeddata.es/dataset/lov/api/v2/vocabulary/list` with 30s timeout. Parse JSON response, extract `prefix` and `nsp` fields. Store in `_lov_prefixes`. Return count of imported prefixes. Handle errors gracefully (log warning, return 0 on failure). Skip entries missing prefix or nsp fields.

4. Update `backend/app/rdf/namespaces.py`:

   - Add owl, sh, foaf, prov to `COMMON_PREFIXES` dict so SPARQL prefix injection covers all standard vocabs.
   - Export the new namespace objects.

1. Create `backend/app/services/labels.py` with class `LabelService`:
   - **Constructor**: Takes `triplestore_client: TriplestoreClient`, `prefix_registry: PrefixRegistry`, `ttl: int = 300` (5 min default), `maxsize: int = 4096`.
     Creates `self._cache = TTLCache(maxsize=maxsize, ttl=ttl)` from cachetools.
     Sets `self._language_prefs: list[str] = ["en"]` (default, configurable via `set_language_prefs()`).
   - **`async resolve(iri: str) -> str`**: Check cache, if miss call `resolve_batch([iri])`, return result or fallback.
   - **`async resolve_batch(iris: list[str]) -> dict[str, str]`**:
     a. Check cache for each IRI, collect misses.
     b. If no misses, return cached results.
     c. Build SPARQL VALUES query with COALESCE for precedence chain:
        ```sparql
        SELECT ?iri ?label WHERE {
          VALUES (?iri) { (<iri1>) (<iri2>) ... }
          OPTIONAL { ?iri dcterms:title ?t . FILTER(LANG(?t) = "" || LANG(?t) = "en") }
          OPTIONAL { ?iri rdfs:label ?r . FILTER(LANG(?r) = "" || LANG(?r) = "en") }
          OPTIONAL { ?iri skos:prefLabel ?s . FILTER(LANG(?s) = "" || LANG(?s) = "en") }
          OPTIONAL { ?iri schema:name ?n . FILTER(LANG(?n) = "" || LANG(?n) = "en") }
          BIND(COALESCE(?t, ?r, ?s, ?n) AS ?label)
        }
        ```
        Use FROM <urn:sempkm:current> to scope to current graph.
        Replace "en" with the first language pref dynamically.
        **Critical**: Use `FILTER(LANG(?t) = "" || LANG(?t) = "{lang}")` to accept both untagged and language-matched literals per research Pitfall 6.
     d. Parse SPARQL JSON results, cache each result, apply fallback for IRIs with no label.
   - **`_iri_fallback(iri: str) -> str`**:
     a. Try `self._prefixes.compact(iri)` -- if result differs from input, it's a QName, return it.
     b. Otherwise, extract local name: split on last "/" then last "#", return `.../localName`.
     c. If no local name extractable, return the full IRI.
   - **`invalidate(iris: list[str])`**: Remove each IRI from cache via `self._cache.pop(iri, None)`.
   - **`set_language_prefs(langs: list[str])`**: Set `self._language_prefs` and clear entire cache (language change invalidates all cached labels).
   - **`_build_lang_filter(var: str) -> str`**: Return SPARQL FILTER clause for the first language pref, e.g., `FILTER(LANG({var}) = "" || LANG({var}) = "en")`.
   - Include proper SPARQL prefix declarations in the query (PREFIX dcterms, rdfs, skos, schema).

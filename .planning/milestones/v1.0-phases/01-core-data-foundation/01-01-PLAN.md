  <action>
1. **backend/app/config.py**: Pydantic BaseSettings class `Settings` with fields:
   - `triplestore_url: str` (default `http://triplestore:8080/rdf4j-server`)
   - `repository_id: str` (default `sempkm`)
   - `base_namespace: str` (default `https://example.org/data/`)
   - `app_version: str` (default `0.1.0`)
   - Use `model_config = SettingsConfigDict(env_file=".env")`.
   - Create module-level `settings = Settings()` singleton.

2. **backend/app/triplestore/client.py**: `TriplestoreClient` class wrapping httpx.AsyncClient:
   - Constructor takes `base_url` and `repository_id`
   - `async def is_healthy() -> bool`: GET `{base_url}/repositories/{repo_id}/size`, return True if 200
   - `async def query(sparql: str) -> dict`: POST to `{base_url}/repositories/{repo_id}` with form-encoded `query=...`, accept `application/sparql-results+json`, return parsed JSON
   - `async def update(sparql: str) -> None`: POST to `{base_url}/repositories/{repo_id}/statements` with form-encoded `update=...`
   - `async def begin_transaction() -> str`: POST to `{base_url}/repositories/{repo_id}/transactions`, return transaction URL from Location header
   - `async def commit_transaction(txn_url: str) -> None`: PUT to txn_url with `action=COMMIT`
   - `async def rollback_transaction(txn_url: str) -> None`: DELETE txn_url
   - `async def transaction_update(txn_url: str, sparql: str) -> None`: PUT to txn_url with form-encoded `update=...` and `action=UPDATE`
   - `async def transaction_query(txn_url: str, sparql: str) -> dict`: POST to txn_url with form-encoded `query=...` and `action=QUERY`, accept `application/sparql-results+json`
   - Implement `async def close()` to close the httpx client
   - Use `application/x-www-form-urlencoded` content type for SPARQL operations (RDF4J expects form-encoded, NOT raw SPARQL body per research anti-patterns)

3. **backend/app/triplestore/setup.py**: `async def ensure_repository(client: httpx.AsyncClient, base_url: str, repo_id: str)`:
   - GET `{base_url}/repositories/{repo_id}/size` to check existence
   - If 404, PUT `{base_url}/repositories/{repo_id}` with the Turtle config from `config/rdf4j/sempkm-repo.ttl` (Content-Type: text/turtle)
   - Also ensure the sentinel triple exists in the current state graph: run SPARQL UPDATE `INSERT DATA { GRAPH <urn:sempkm:current> { <urn:sempkm:current> a <urn:sempkm:StateGraph> } }` (per research Pitfall 1 -- prevents empty graph deletion)

4. **config/rdf4j/sempkm-repo.ttl**: Native store config with `spoc,posc,cspo` indexes (cspo is critical for context-based queries per research Pitfall 7). Use the modern `tag:rdf4j.org,2023:config/` namespace per research state-of-the-art.

5. **backend/app/dependencies.py**: FastAPI dependency `get_triplestore_client()` that returns the TriplestoreClient instance stored on `app.state`.

6. **backend/app/main.py**: FastAPI app factory with:
   - Lifespan context manager that creates TriplestoreClient, calls `ensure_repository`, stores client on `app.state`, yields, then closes client on shutdown
   - Include health router
   - CORS middleware for dev console access (allow origins `*` for development)
   - Title "SemPKM API"

7. **backend/app/health/router.py**: `GET /api/health` endpoint:
   - Uses `get_triplestore_client` dependency
   - Returns `{"status": "healthy"|"degraded", "services": {"api": "up", "triplestore": "up"|"down"}, "version": settings.app_version}`

8. **backend/app/health/__init__.py** and **backend/app/triplestore/__init__.py**: Empty init files.
  </action>

Run `docker compose down` after verification.

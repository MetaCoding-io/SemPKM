**install(model_dir: Path) -> InstallResult:**
Full install pipeline inside an RDF4J transaction for atomicity (per Research Pitfall 3):

1. Parse manifest: `parse_manifest(model_dir)`. On failure, return InstallResult with error.
2. Check duplicate: `is_model_installed(client, manifest.modelId)`. If installed, return error "Model '{modelId}' is already installed. Remove it first to reinstall."
3. Load archive: `load_archive(model_dir, manifest)`. On failure, return error.
4. Validate archive: `validate_archive(archive)`. If not valid, return error with all validation issues.
5. Begin transaction: `client.begin_transaction()`.
6. Try: Write each graph to its named graph via `write_graph_to_named_graph` using `client.transaction_update` (NOT client.update -- must be in-transaction). The registry.py write_graph_to_named_graph should accept an optional sparql_fn parameter to support both standalone and in-transaction modes. If this doesn't work cleanly, build the SPARQL INSERT DATA strings in ModelService and call client.transaction_update directly.
7. Register model metadata: build and execute the registry INSERT DATA within the transaction.
8. If seed data exists, materialize seed data into `urn:sempkm:current` via EventStore.commit() using a synthetic "model.install" event. Create Operation objects for each seed triple as inserts. NOTE: EventStore.commit() manages its own transaction, so seed materialization should happen OUTSIDE the model graph transaction. Alternative approach: write seed data to `urn:sempkm:current` directly via SPARQL INSERT DATA within the model transaction (simpler, but bypasses event sourcing). Per Research open question 1, the recommendation is to use EventStore.commit() for seed data to maintain event sourcing consistency. So: commit the model graph transaction first, THEN materialize seed data via EventStore.commit().
9. Commit transaction: `client.commit_transaction()`.
10. Register model prefixes with PrefixRegistry: `prefix_registry.register_model_prefixes(manifest.prefixes)`.
11. Return InstallResult(success=True, model_id=manifest.modelId, warnings=...).
12. On any error in steps 5-9: rollback transaction and return error.

**remove(model_id: str) -> RemoveResult:**
1. Check model exists: `is_model_installed(client, model_id)`. If not, return error.
2. Load ontology graph from triplestore to check user data: Execute SPARQL CONSTRUCT to get model's ontology graph: `CONSTRUCT { ?s ?p ?o } FROM <urn:sempkm:model:{model_id}:ontology> WHERE { ?s ?p ?o }`. Parse into rdflib Graph.
3. Check for user data: `check_user_data_exists(client, manifest_namespace, ontology_graph)`. If types_with_data is non-empty, return RemoveResult with error listing which types have data.
4. Clear all model named graphs: `clear_model_graphs(client, model_id)`.
5. Unregister model: `unregister_model(client, model_id)`.
6. Clear model prefixes from PrefixRegistry (remove model prefixes -- PrefixRegistry may need a `clear_model_prefixes()` method, or just leave them for now since it's additive and doesn't cause issues).
7. Return RemoveResult(success=True, model_id=model_id).

**list_models() -> list[InstalledModel]:**
Delegate to `registry_list_models(client)`.

Define async function `model_shapes_loader(client: TriplestoreClient) -> Graph`:
This is the real shapes loader that replaces `empty_shapes_loader`. It fetches all SHACL shapes from all installed model shapes graphs.
1. List installed models via SPARQL query on `urn:sempkm:models`.
2. If no models installed, return empty Graph().
3. Build SPARQL CONSTRUCT with FROM clauses for each model's shapes graph: `CONSTRUCT { ?s ?p ?o } FROM <urn:sempkm:model:{modelId}:shapes> ... WHERE { ?s ?p ?o }`.
4. Execute via `client.construct()`, parse turtle result into Graph.
5. Return the shapes graph.

Define async function `ensure_starter_model(model_service: ModelService, starter_path: Path) -> None`:
1. List installed models. If any exist, return (no-op).
2. Log "No models installed, auto-installing Basic PKM starter model".
3. Call `model_service.install(starter_path)`.
4. If result.success, log success. Else, log error with result.errors.

Create `models/basic-pkm/views/basic-pkm.jsonld` as JSON-LD with inline @context.

Define a minimal view spec vocabulary using `sempkm:` namespace terms:
- `sempkm:ViewSpec` — the view type
- `sempkm:targetClass` — which class this view is for
- `sempkm:rendererType` — "table", "card", or "graph"
- `sempkm:sparqlQuery` — the SPARQL query string (stored as xsd:string literal)
- `sempkm:columns` — comma-separated column names for table views
- `sempkm:sortDefault` — default sort column
- `sempkm:cardTitle` — property path for card title
- `sempkm:cardSubtitle` — property path for card subtitle

Define views (at minimum 3 per type = 12 total, but at least one of each renderer type):

For each type, define:
1. A table view (rendererType "table") with SPARQL SELECT and column configuration
2. A card view (rendererType "card") with SPARQL SELECT and card title/subtitle config
3. A graph view (rendererType "graph") with SPARQL CONSTRUCT for relationships

Example for Projects table view:
```
SPARQL: SELECT ?s ?title ?status ?priority ?startDate WHERE { ?s a <bpkm:Project> ; <dcterms:title> ?title . OPTIONAL { ... } }
columns: title,status,priority,startDate
sortDefault: title
```

All SPARQL queries in view specs MUST use full IRIs (not prefixed names) since the queries are stored as string literals. Use `<urn:sempkm:model:basic-pkm:Project>` not `bpkm:Project` inside SPARQL strings.

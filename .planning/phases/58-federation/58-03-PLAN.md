<objective>
Shared graph management, sync pull/apply, notification sending, SPARQL graph scoping, and outbound sync alerts.

Purpose: Wire together the foundation from Plans 01-02 into working federation flows. Users can create shared graphs, invite remote users, pull and apply patches from remote instances (with syncSource loop prevention), copy objects into shared graphs, send notifications, see shared graph data in SPARQL queries, and trigger outbound sync-alert notifications when local changes are committed to shared graphs.

Output: `service.py` (FederationService orchestrating all federation logic), extended router with shared graph CRUD and sync endpoints, extended SPARQL scoping for shared graphs wired into query endpoints, outbound LDN alerts on local writes to shared graphs.

       **FederationService class** (initialized with TriplestoreClient, EventStore):

       **Shared graph management (stored as RDF in `urn:sempkm:federation` metadata graph):**
       - `async def create_shared_graph(name, description, required_model, creator_webid) -> str`:
         a. Mint IRI: `urn:sempkm:shared:{uuid4()}`.
         b. Store metadata triples in `urn:sempkm:federation` graph: name (dcterms:title), description, required_model (sempkm:requiredModel), created timestamp, creator as member.
         c. Return the graph IRI.
       - `async def list_shared_graphs(user_webid) -> list[SharedGraphResponse]`:
         a. SPARQL query on `urn:sempkm:federation` for shared graphs where user_webid is a member.
         b. Include pending count (events with graphTarget matching the shared graph that haven't been pushed).
       - `async def get_shared_graph(graph_iri) -> SharedGraphResponse | None`:
         a. Query metadata from federation graph.
       - `async def leave_shared_graph(graph_iri, user_webid)`:
         a. Remove membership triple. Data stays as frozen snapshot per CONTEXT.md decision.

       **Object copying to shared graph:**
       - `async def copy_to_shared_graph(object_iri, shared_graph_iri, performed_by, performed_by_role)`:
         a. Query all triples for object_iri from CURRENT_GRAPH.
         b. Create Operation with materialize_inserts = those triples, operation_type = "federation.copy".
         c. Commit via EventStore with target_graph=shared_graph_iri.
         d. After successful commit, call `notify_remote_members_of_change(shared_graph_iri, performed_by, event_count=1)` to send LDN sync-alert notifications to all remote members of the shared graph.

       **Outbound sync-alert notifications on local writes:**
       - `async def notify_remote_members_of_change(shared_graph_iri, local_webid, event_count, private_key_pem=None, key_id=None)`:
         a. Query remote members of the shared graph from federation metadata (exclude local_webid).
         b. For each remote member, discover their inbox via their WebID profile or cached contact info.
         c. Build ActivityStreams Update notification JSON-LD with `sempkm:patchCount` (event_count) and `sempkm:graphTarget` (shared_graph_iri).
         d. Sign and POST to each remote member's inbox via `send_notification()`.
         e. Log errors but do not fail the caller -- fire-and-forget for alert delivery.
       - This method is called by `copy_to_shared_graph()` and by the federation router endpoints after any local EventStore.commit() that targets a shared graph (see Task 2 action step for wiring).

       **Invitation flow:**
       - `async def send_invitation(graph_iri, recipient_handle, sender_webid, sender_private_key, sender_key_id)`:
         a. Discover recipient via `discover_webid(recipient_handle)` to get inbox URL.
         b. Build ActivityStreams Offer JSON-LD (per RESEARCH.md pattern).
         c. Include `sempkm:requiredModel` in the offer object.
         d. Sign request with `sign_request()`.
         e. POST to recipient's inbox URL with signed headers.
       - `async def accept_invitation(notification_graph_iri, user_webid)`:
         a. Read the invitation from the notification graph.
         b. Extract shared graph IRI, sender WebID.
         c. Create local shared graph with same IRI.
         d. Store sender WebID as a known contact in federation metadata graph.
         e. Update notification state to "acted".

       **Sync flow:**
       - `async def sync_shared_graph(graph_iri, remote_instance_url, local_webid, private_key_pem, key_id) -> SyncResult`:
         a. Determine `since` timestamp (last sync time from federation metadata, or epoch for first sync).
         b. Build URL: `{remote_instance_url}/api/federation/patches/{graph_id}?since={since}&requester={local_instance_url}`.
         c. Sign request with `sign_request()`.
         d. Fetch patches via httpx with signed headers.
         e. Parse response (PatchExportResponse JSON).
         f. Deserialize patch text via `deserialize_patch()`.
         g. Group A/D tuples into Operations: inserts from A lines, deletes from D lines.
         h. Apply via `EventStore.commit(operations, target_graph=graph_iri, sync_source=remote_instance_url)`.
         i. Update last sync timestamp in federation metadata.
         j. Send sync status alert LDN notification to all remote members of the graph via `notify_remote_members_of_change()`.
         k. Return SyncResult with counts.

       **Notification sending:**
       - `async def send_notification(recipient_handle, notification_type, sender_webid, private_key_pem, key_id, object_iri=None, content=None)`:
         a. Discover recipient.
         b. Build appropriate AS JSON-LD based on type (Announce for recommendation, Note for message) per RESEARCH.md patterns.
         c. Sign and POST to inbox.

       **Contact management (derived from shared graph memberships):**
       - `async def list_contacts(user_webid) -> list[ContactInfo]`:
         a. Query distinct remote WebIDs from shared graph memberships in federation metadata.

       **Shared graph access for SPARQL:**
       - `async def get_user_shared_graphs(user_webid) -> list[str]`:
         a. Return list of shared graph IRIs the user is a member of.
         b. Used by SPARQL scoping to add FROM clauses.

<objective>
HTTP Signature authentication, WebFinger discovery, and LDN inbox receiver.

Purpose: Establish the security and notification foundations for federation. All server-to-server requests must be signed and verified using RFC 9421 HTTP Message Signatures with Ed25519 keys (already in WebID profiles). The LDN inbox endpoint receives notifications from remote instances. WebFinger enables email-like handle discovery (user@domain -> WebID URL).

Output: `signatures.py` (sign/verify wrappers using http-message-signatures library), `webfinger.py` (server + client), `inbox.py` (POST /api/inbox receiver storing in RDF), WebID profile Link header for inbox discovery, WebFinger router registered in main.py.
</objective>

       - Function `async def sign_request(method: str, url: str, headers: dict, body: bytes | None, private_key_pem: str, key_id: str) -> dict`:
         a. Build a signing context from method, URL, headers, body.
         b. If body is not None, compute content-digest (SHA-256 base64) and add `Content-Digest` header.
         c. Use `http_message_signatures.HTTPMessageSigner` with `algorithms.ED25519` and a `SemPKMKeyResolver` that loads the private key from PEM.
         d. Covered components: `@method`, `@target-uri`, `content-type` (if present), `content-digest` (if body present).
         e. Sign the request, return updated headers dict with `Signature` and `Signature-Input` headers added.
         f. If the library's API expects a `requests.Request` object, create one from the parameters, sign it, and extract the signature headers.

       **Note on http-message-signatures API:** The library may require a `requests.Request` or similar object. Build a thin adapter:
       ```python
       import requests as req_lib
       r = req_lib.Request(method, url, headers=headers, data=body)
       prepared = r.prepare()
       signer.sign(prepared, key_id=key_id, covered_component_ids=(...))
       return dict(prepared.headers)
       ```

       **Inbound verification (`verify_request`):**
       - WebID public key cache: `_webid_key_cache = TTLCache(maxsize=64, ttl=3600)` (1 hour, 64 entries per CONTEXT.md).
       - Function `async def fetch_webid_public_key(webid_uri: str, force_refresh: bool = False) -> Ed25519PublicKey`:
         a. Check cache (skip if force_refresh).
         b. Fetch WebID profile URL with `Accept: text/turtle` using httpx.
         c. Parse Turtle response with rdflib, extract `sec:publicKeyPem` value.
         d. Load PEM with `cryptography.hazmat.primitives.serialization.load_pem_public_key`.
         e. Cache and return.
       - `class RemoteKeyResolver(HTTPSignatureKeyResolver)`:
         a. `resolve_public_key(key_id)` calls `fetch_webid_public_key(key_id)` (key_id is the WebID URI).
       - Function `async def verify_request(method: str, url: str, headers: dict, body: bytes | None) -> str`:
         a. Use `HTTPMessageVerifier` with `algorithms.ED25519` and `RemoteKeyResolver`.
         b. Build request object, verify signature.
         c. On `VerifyError`, force-refresh the key cache and retry once (handles key rotation).
         d. Return the key_id (WebID URI of the sender) on success.
         e. Raise `HTTPException(401)` on failure.

       **FastAPI dependency:**
       - `class VerifyHTTPSignature`: callable dependency that extracts method, url, headers, body from FastAPI `Request` and calls `verify_request()`. Returns the sender's WebID URI string. Use as `sender_webid: str = Depends(VerifyHTTPSignature())`.

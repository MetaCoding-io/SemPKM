4. **Create `backend/app/inference/entailments.py`**: Implement selective entailment classification:
   - `ENTAILMENT_TYPES = ["owl:inverseOf", "rdfs:subClassOf", "rdfs:subPropertyOf", "owl:TransitiveProperty", "rdfs:domain/rdfs:range"]`
   - `classify_entailment(s, p, o, ontology: Graph) -> str | None` — uses the heuristic from the research doc:
     - Check `owl:inverseOf`: triple `?x ?q ?y` where ontology says `?q owl:inverseOf ?p` (or `?p owl:inverseOf ?q`) and we can verify the inverse relationship
     - Check `rdfs:subClassOf`: triple `?x rdf:type ?C` where ontology has subclass chain
     - Check `rdfs:subPropertyOf`: triple uses a super-property
     - Check `owl:TransitiveProperty`: property is declared transitive
     - Check `rdfs:domain/rdfs:range`: type assertion from domain/range axiom
     - Return None for unclassifiable triples (filter them out)
   - `filter_by_enabled(triples_with_types: list, enabled_types: set) -> list` — returns only triples whose entailment type is in the enabled set

5. **Create `backend/app/inference/service.py`**: Implement `InferenceService`:
   ```python
   class InferenceService:
       def __init__(self, client: TriplestoreClient, db_session):
           self._client = client
           self._db = db_session

       async def run_inference(self, entailment_config: dict[str, bool]) -> InferenceResult:
           """Full recompute strategy:
           1. Load all data from urn:sempkm:current via CONSTRUCT
           2. Load ontology graphs from all installed models (same pattern as model_shapes_loader)
           3. Merge data + ontology into rdflib working graph; keep originals for diffing
           4. Run owlrl.DeductiveClosure(owlrl.OWLRL_Semantics).expand(working)
           5. Diff: new_triples = set(working) - set(data) - set(ontology)
           6. Filter out blank node triples
           7. Filter out triples already in urn:sempkm:current (user takes precedence)
           8. Classify each triple by entailment type
           9. Filter by enabled entailment types from config
           10. Clear urn:sempkm:inferred (DROP GRAPH or DELETE WHERE)
           11. INSERT DATA inferred triples into urn:sempkm:inferred
           12. Update inference_triple_state table: mark new triples, preserve dismissed/promoted state
           13. Log inference run as event via EventStore
           14. Return InferenceResult with counts and triple details
           """

       async def get_inferred_triples(self, filters: dict) -> list[dict]:
           """Query urn:sempkm:inferred and join with inference_triple_state for status.
           Support filters: object_type, date_range, group_by.
           Return list of {subject, predicate, object, entailment_type, status, inferred_at}."""

       async def dismiss_triple(self, triple_hash: str) -> None:
           """Set status='dismissed' in inference_triple_state.
           Remove the triple from urn:sempkm:inferred graph."""

       async def promote_triple(self, triple_hash: str) -> None:
           """Copy triple to urn:sempkm:current via EventStore.commit().
           Set status='promoted' in inference_triple_state.
           Remove from urn:sempkm:inferred (now it's user data)."""

       async def _load_current_data(self) -> Graph:
           """CONSTRUCT { ?s ?p ?o } FROM <urn:sempkm:current> WHERE { ?s ?p ?o }"""

       async def _load_ontology_graphs(self) -> Graph:
           """Same pattern as model_shapes_loader: query model registry for installed models,
           CONSTRUCT from each model's ontology graph (urn:sempkm:model:{id}:ontology)."""

       async def _get_entailment_config(self) -> dict[str, bool]:
           """Read entailment toggles from settings service. Fall back to manifest defaults."""
   ```

   Define `InferenceResult` as a dataclass with fields: `total_inferred: int`, `by_entailment_type: dict[str, int]`, `new_count: int`, `stale_count: int`, `run_timestamp: str`.

6. **Update `models/basic-pkm/manifest.yaml`**: Add `entailment_defaults` section:

<!-- Critical SPARQL pattern from research doc — use UNION, not just FROM -->

Relations panel dual-graph query pattern:
```sparql
SELECT ?predicate ?object ?source WHERE {
  {
    GRAPH <urn:sempkm:current> {
      <{iri}> ?predicate ?object .
      FILTER(isIRI(?object))
      FILTER(?predicate != rdf:type)
    }
    BIND("user" AS ?source)
  } UNION {
    GRAPH <urn:sempkm:inferred> {
      <{iri}> ?predicate ?object .
      FILTER(isIRI(?object))
      FILTER(?predicate != rdf:type)
    }
    BIND("inferred" AS ?source)
  }
}
```

For FROM-based queries (labels, scope_to_current_graph):
```sparql
SELECT ?s ?p ?o
FROM <urn:sempkm:current>
FROM <urn:sempkm:inferred>
WHERE { ?s ?p ?o }
```

IMPORTANT: Do NOT mix FROM and GRAPH patterns in the same query — see Pitfall 5 in research doc.

<objective>
Build the asynchronous SHACL validation engine that runs after every command commit, stores immutable validation reports as named graphs, and exposes a polling endpoint for validation status.

Purpose: Every data write must be validated against SHACL shapes to maintain data quality. Validation runs asynchronously to avoid blocking the write path (per user decision). Each commit gets its own immutable report tied to its event IRI, enabling audit trails and per-commit conformance status. The polling endpoint lets the UI (Phase 4) display validation results.

Output: `backend/app/validation/` package (queue, report, router), `backend/app/services/validation.py`, updated commands router with validation trigger, updated triplestore client with CONSTRUCT support.
</objective>

   async def construct(self, sparql: str) -> bytes:
       """Execute a SPARQL CONSTRUCT and return raw Turtle bytes."""
       resp = await self._client.post(
           self._repo_url,
           data={"query": sparql},
           headers={"Accept": "text/turtle"},
       )
       resp.raise_for_status()
       return resp.content

3. Create `backend/app/validation/report.py` with class `ValidationReport`:
   - **`@dataclass ValidationReportSummary`**: `report_iri: str`, `event_iri: str`, `conforms: bool`, `violation_count: int`, `warning_count: int`, `info_count: int`, `timestamp: str`.
   - **`@dataclass ValidationResult`**: `focus_node: str`, `severity: str` (one of "Violation", "Warning", "Info"), `path: Optional[str]`, `value: Optional[str]`, `message: str`, `source_shape: Optional[str]`, `constraint_component: Optional[str]`.
   - **`class ValidationReport`**:
     - `__init__(self, event_iri: str, conforms: bool, results: list[ValidationResult], timestamp: str)`.
     - **`@classmethod from_pyshacl(cls, event_iri: str, results_graph: Graph, conforms: bool, timestamp: str) -> "ValidationReport"`**:
       Parse the pyshacl results_graph using SPARQL query (from research Pattern: Parsing the Validation Report Graph).
       Map `sh:Violation` -> "Violation", `sh:Warning` -> "Warning", `sh:Info` -> "Info".
       For each result: extract focusNode, resultPath, value, resultMessage, sourceShape, sourceConstraintComponent.
       If `sh:resultMessage` is missing, auto-generate from constraint metadata: `"{path} {component}: constraint violated"` where path is compacted via str(path) and component is the local name of sourceConstraintComponent.
     - **`summary(self) -> ValidationReportSummary`**: Return summary with counts by severity.
     - **`report_iri`** property: `urn:sempkm:validation:{uuid}` extracted from event_iri pattern `urn:sempkm:event:{uuid}`.
     - **`to_summary_triples(self) -> list[tuple]`**: Generate RDF triples for the summary named graph `urn:sempkm:validations`:
       ```
       <report_iri> a sempkm:ValidationReport .
       <report_iri> sempkm:forEvent <event_iri> .
       <report_iri> sempkm:conforms true/false .
       <report_iri> sempkm:violationCount N .
       <report_iri> sempkm:warningCount N .
       <report_iri> sempkm:infoCount N .
       <report_iri> sempkm:timestamp "..." .
       ```
       Use rdflib URIRef, Literal, XSD.boolean, XSD.integer, XSD.dateTime.

4. Create `backend/app/services/validation.py` with class `ValidationService`:
   - **Constructor**: Takes `triplestore_client: TriplestoreClient`, `shapes_loader: Callable[[], Awaitable[Graph]]` (async callable returning shapes graph -- Phase 3 will provide the real loader; for now accept any callable).
   - **`async def validate(self, event_iri: str, timestamp: str) -> ValidationReport`**:
     a. Fetch current state graph via CONSTRUCT: `CONSTRUCT { ?s ?p ?o } FROM <urn:sempkm:current> WHERE { ?s ?p ?o }`. Parse returned Turtle bytes into rdflib Graph.
     b. Load shapes via `self._shapes_loader()`. If shapes graph is empty (no shapes installed yet), return a synthetic "conforms=True" report with zero results (no shapes = nothing to violate).
     c. Run pyshacl: `await asyncio.to_thread(pyshacl.validate, data_graph, shacl_graph=shapes_graph, allow_infos=True, allow_warnings=True)`.
     d. Parse results into ValidationReport via `ValidationReport.from_pyshacl()`.
     e. Store report: serialize full pyshacl results_graph as Turtle, INSERT DATA into named graph `<report_iri>`. Also INSERT summary triples into `<urn:sempkm:validations>` graph.
     f. Return the ValidationReport.
   - **`async def get_latest_summary(self) -> Optional[ValidationReportSummary]`**: SPARQL query against `urn:sempkm:validations` graph for most recent report, ORDER BY DESC(?ts) LIMIT 1.
   - **`async def get_report_by_event(self, event_iri: str) -> Optional[ValidationReportSummary]`**: SPARQL query for report matching specific event_iri.
   - **Default shapes loader**: A module-level `async def empty_shapes_loader() -> Graph` that returns an empty rdflib Graph. This is the default until Phase 3 provides real shapes.

5. Create `backend/app/validation/queue.py` with class `AsyncValidationQueue`:
   - **Constructor**: Takes `validation_service: ValidationService`.
   - **`@dataclass ValidationJob`**: `event_iri: str`, `timestamp: str`.
   - Uses `asyncio.Queue[ValidationJob]` internally.
   - **`async start()`**: Create `asyncio.create_task(self._worker())`, store task reference.
   - **`async stop()`**: Cancel the worker task.
   - **`async enqueue(event_iri: str, timestamp: str)`**: Non-blocking put on the queue. **Queue coalesce optimization** (per research Pitfall 4): before processing, drain any additional jobs from the queue and only process the latest (since full re-validation makes intermediate results obsolete). Implementation: after `queue.get()`, do a non-blocking drain loop with `queue.get_nowait()` to collect all pending jobs, then validate only the last one. Call `task_done()` for each drained job.
   - **`async _worker()`**: Infinite loop: get job, drain queue for coalescing, call `self._validation_service.validate()`, catch and log exceptions (never crash the worker).
   - **`@property latest_report -> Optional[ValidationReportSummary]`**: Return the most recent report summary (cached in memory after each validation run for fast polling without hitting triplestore).

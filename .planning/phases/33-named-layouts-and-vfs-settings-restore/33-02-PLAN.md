**1. Add `showToast(message, duration)` utility function** to workspace.js (inside the IIFE, near the top utility section). This is a lightweight notification that auto-dismisses:

```javascript
function showToast(message, duration) {
  duration = duration || 3000;
  var toast = document.createElement('div');
  toast.className = 'sempkm-toast';
  toast.textContent = message;
  document.body.appendChild(toast);
  requestAnimationFrame(function () { toast.classList.add('sempkm-toast--visible'); });
  setTimeout(function () {
    toast.classList.remove('sempkm-toast--visible');
    setTimeout(function () { toast.remove(); }, 300);
  }, duration);
}
```

Expose as `window.showToast = showToast;` alongside existing window exports.

**2. Add `_refreshLayoutPaletteItems(ninja)` helper** that rebuilds the layout-restore-* and layout-delete-* child items from `SemPKMLayouts.list()`. Follow the existing dynamic data pattern (filter old items, push new children, reassign `ninja.data`):

```javascript
function _refreshLayoutPaletteItems(ninja) {
  var baseData = ninja.data.filter(function (d) {
    return !d.id.startsWith('layout-restore-') && !d.id.startsWith('layout-delete-');
  });
  var layouts = window.SemPKMLayouts ? window.SemPKMLayouts.list() : [];
  layouts.forEach(function (l) {
    baseData.push({
      id: 'layout-restore-' + l.name,
      title: l.name,
      parent: 'layout-restore',
      handler: function () {
        var result = window.SemPKMLayouts.restore(l.name);
        if (result.success) {
          showToast('Layout "' + l.name + '" restored');
        } else if (result.skipped.length > 0) {
          showToast('Layout restored with ' + result.skipped.length + ' skipped item(s)', 5000);
        }
      }
    });
    baseData.push({
      id: 'layout-delete-' + l.name,
      title: l.name,
      parent: 'layout-delete',
      handler: function () {
        window.SemPKMLayouts.remove(l.name);
        showToast('Layout "' + l.name + '" deleted');
        _refreshLayoutPaletteItems(ninja);
      }
    });
  });
  ninja.data = baseData;
}
```

**3. Add layout commands to `initCommandPalette()`** -- append to the existing `ninja.data` array (after the toggle-edit-mode entry, before the `_loadViewCommandPaletteEntries` call). Add 3 top-level parent items:

```javascript
{
  id: 'layout-save-as',
  title: 'Layout: Save As...',
  section: 'Layout',
  children: ['layout-save-confirm']
},
{
  id: 'layout-save-confirm',
  title: 'Type a layout name above, then select this item to save',
  parent: 'layout-save-as',
  handler: function () {
    // Read the current search text from ninja-keys input as the layout name.
    // ninja-keys is a web component; its search input lives inside its shadowRoot.
    var ninjaEl = document.querySelector('ninja-keys');
    var name = '';
    if (ninjaEl) {
      // Approach 1: Try shadowRoot input (the search field ninja-keys uses)
      try {
        var input = ninjaEl.shadowRoot.querySelector('input[type="text"]');
        if (input) name = input.value;
      } catch (e) {}
      // Approach 2: Fallback — ninja-keys exposes _search or .search property
      if (!name && ninjaEl._search) name = ninjaEl._search;
    }
    name = name ? name.trim() : '';
    if (!name) {
      showToast('Please type a layout name in the search field first', 3000);
      return;
    }
    window.SemPKMLayouts.save(name);
    showToast('Layout "' + name + '" saved');
    _refreshLayoutPaletteItems(ninjaEl);
  }
},
{
  id: 'layout-restore',
  title: 'Layout: Restore...',
  section: 'Layout',
  children: []   // populated by _refreshLayoutPaletteItems
},
{
  id: 'layout-delete',
  title: 'Layout: Delete...',
  section: 'Layout',
  children: []   // populated by _refreshLayoutPaletteItems
}
```

**CRITICAL: No `prompt()` or `window.prompt()`.** Per CONTEXT.md locked decision: "Naming via text input inline in Command Palette (no modal)." The `layout-save-as` item is a parent that opens a sub-view. The user types the desired layout name into the ninja-keys search field (which is already visible at the top of the palette) and then selects the "layout-save-confirm" child item (by clicking it or pressing Enter). The handler reads the search field value from the ninja-keys shadow DOM input element.

**How the inline naming flow works:**
1. User presses Ctrl+K, types "Layout", selects "Layout: Save As..."
2. Palette navigates to the `layout-save-as` parent sub-view, showing one child: "Type a layout name above, then select this item to save"
3. The search field at the top of the palette is now empty and focused -- the user types the desired layout name (e.g., "My Dashboard")
4. User presses Enter or clicks the child item -- handler reads the search field text as the name
5. Layout is saved, toast confirms, palette closes

**Fallback handling:** If the shadow DOM input cannot be read (browser/version issue), show a descriptive toast: "Please type a layout name in the search field first" and do NOT fall back to prompt(). Surface the issue clearly rather than silently adopting a modal. The executor should verify this works during implementation and, if the shadowRoot approach fails, try alternative selectors like `ninjaEl.shadowRoot.querySelector('.ninja-input input')` or check if ninja-keys exposes a public search property.

  <name>Task 1: ShapesService for SHACL shape extraction and form metadata</name>
  <files>
    backend/app/services/shapes.py
  </files>
  <action>
    Create `backend/app/services/shapes.py` implementing server-side SHACL shape traversal per Research Pattern 1.

    1. Define dataclasses (per research architecture):
       - `PropertyShape`: path (str), name (str), datatype (str|None), target_class (str|None), order (float, default 0.0), group (str|None), min_count (int, default 0), max_count (int|None), in_values (list[str]), default_value (str|None), description (str|None)
       - `PropertyGroup`: iri (str), label (str), order (float)
       - `NodeShapeForm`: shape_iri (str), target_class (str), label (str), groups (list[PropertyGroup]), properties (list[PropertyShape])

    2. Implement `ShapesService` class:
       - Constructor takes `TriplestoreClient`
       - `async get_node_shapes() -> list[NodeShapeForm]`: Query all sh:NodeShape instances from installed model shapes graphs. Use SPARQL CONSTRUCT to fetch the shapes graph into rdflib, then traverse with rdflib Python API (not complex SPARQL). Query across all model shapes named graphs (use FROM clauses for each installed model's shapes graph, or query without FROM to get all named graphs).
       - `async get_form_for_type(type_iri: str) -> NodeShapeForm | None`: Get form metadata for a specific target class.
       - `async get_types() -> list[dict]`: Return list of available types (targetClass IRI + label) for the type picker.

    3. SHACL shape graph traversal strategy (per Research Pitfall 2):
       - Use SPARQL CONSTRUCT to fetch the complete shapes graph into an rdflib Graph
       - Query pattern for shapes graphs: CONSTRUCT from model shapes named graphs (use ModelGraphs.shapes_graph pattern: `urn:sempkm:model:{model_id}:shapes`)
       - Traverse the rdflib Graph using Python:
         * Find all `sh:NodeShape` subjects with `sh:targetClass`
         * For each NodeShape, find `sh:property` links to PropertyShape nodes
         * Extract sh:path, sh:name, sh:datatype, sh:class, sh:order, sh:group, sh:minCount, sh:maxCount, sh:defaultValue, sh:description
         * For sh:in lists, use `rdflib.Collection` to traverse the RDF list (rdf:first/rdf:rest chain) -- per Research Pitfall 2
         * For sh:group references, resolve group labels and order from PropertyGroup nodes

    4. Sort properties by sh:order (ascending). Group into PropertyGroup buckets. Required fields (minCount > 0) sorted before optional.

    5. Use the existing SH namespace from rdflib (`from rdflib.namespace import SH`) for all SHACL terms. Import RDF, RDFS from rdflib.namespace for type/label queries.

    6. Resolve human-readable labels: use sh:name first, fall back to rdfs:label on the sh:path target, fall back to local name extraction from the IRI.
  </action>

    1. Create `backend/app/services/webhooks.py` implementing webhook configuration storage and dispatch per Research Pattern 5.

    Define dataclass:
    - `WebhookConfig`: id (str, UUID), target_url (str), events (list[str] -- e.g. ["object.changed", "edge.changed"]), filters (list[str] -- optional type/namespace filters), enabled (bool, default True)

    Implement `WebhookService` class:
    - Constructor takes `TriplestoreClient`
    - Use dedicated named graph `urn:sempkm:webhooks` for config storage (per research code example)
    - `async create_config(target_url: str, events: list[str], filters: list[str] | None = None) -> WebhookConfig`: Store webhook config as RDF triples in webhooks graph. Generate UUID for webhook IRI. Use SPARQL INSERT DATA with GRAPH clause.
    - `async list_configs() -> list[WebhookConfig]`: Query all webhook configs from the webhooks graph. Parse results into WebhookConfig dataclasses.
    - `async get_config(webhook_id: str) -> WebhookConfig | None`: Get a specific webhook config by ID.
    - `async update_config(webhook_id: str, target_url: str | None, events: list[str] | None, filters: list[str] | None, enabled: bool | None) -> WebhookConfig`: Update specific fields of a webhook config. Use DELETE/INSERT WHERE pattern for atomic update.
    - `async delete_config(webhook_id: str) -> bool`: Remove a webhook config from the graph.
    - `async dispatch(event_type: str, payload: dict)`: Fire outbound webhooks for matching event subscriptions. Use `httpx.AsyncClient(timeout=5.0)` for non-blocking POST. Log failures as warnings (fire-and-forget, per research anti-pattern guidance). Only dispatch to enabled webhooks whose events list includes the event_type. Apply filters if specified (e.g., only fire for matching object types).

    RDF triple pattern for webhook storage (per research):
    ```
    <urn:sempkm:webhook:{uuid}> a sempkm:Webhook ;
        sempkm:targetUrl "https://example.com/hook" ;
        sempkm:event "object.changed" ;
        sempkm:event "edge.changed" ;
        sempkm:filter "bpkm:Person" ;
        sempkm:enabled "true"^^xsd:boolean .
    ```

    2. Update `backend/app/dependencies.py`:
       - Import ShapesService and WebhookService
       - Add `async def get_shapes_service(request: Request) -> ShapesService`
       - Add `async def get_webhook_service(request: Request) -> WebhookService`

       - Create `shapes_service = ShapesService(client)` and store on `app.state.shapes_service`
       - Create `webhook_service = WebhookService(client)` and store on `app.state.webhook_service`

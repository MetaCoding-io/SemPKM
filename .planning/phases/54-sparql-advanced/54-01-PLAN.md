   - `shared_at: datetime` with `server_default=func.now()`
   - `last_viewed_at: datetime | None` nullable (for "Updated" badge logic)
   - `UniqueConstraint("query_id", "shared_with_user_id", name="uq_shared_query_user")`

2. `PromotedQueryView` -- for Plan 54-02, create the model now so migration covers both tables:
   - `id: UUID` primary key (default uuid4)
   - `query_id: UUID` FK to `saved_sparql_queries.id` with `ondelete="CASCADE"`, unique=True
   - `user_id: UUID` FK to `users.id` with `ondelete="CASCADE"`, indexed

   - `created_at: datetime` with `server_default=func.now()`

**Migration (008_sharing_promotion.py):**
- `revision = "008"`, `down_revision = "007"`
- Creates `shared_query_access` table with all columns, unique constraint, and indexes
- Creates `promoted_query_views` table with all columns and indexes
- Import `UniqueConstraint` from sqlalchemy. Follow the exact pattern of `007_sparql_tables.py`.

**Schemas (schemas.py):** Add new Pydantic models:
- `ShareableUserOut`: `id: UUID`, `email: str`, `display_name: str | None`
- `SharedQueryOut`: extends `SavedQueryOut` with extra fields: `owner_name: str`, `is_updated: bool` (computed: query.updated_at > access.last_viewed_at or last_viewed_at is None)
- `ShareUpdateRequest`: `user_ids: list[uuid.UUID]` -- the full set of user IDs to share with (replaces current set)

**Router endpoints (router.py):** Add these new endpoints after the existing saved query section:

1. `GET /sparql/users` -- List all non-guest users (for share picker). Returns `list[ShareableUserOut]`. Query: `select(User).where(User.role != 'guest')`.

2. `GET /sparql/saved/{query_id}/shares` -- List user IDs this query is currently shared with. Returns `list[UUID]`. Only the query owner can call this (check `SavedSparqlQuery.user_id == user.id`).

3. `PUT /sparql/saved/{query_id}/shares` -- Set sharing. Accepts `ShareUpdateRequest`. Replace all `SharedQueryAccess` rows for this query_id with the new user_ids list. Owner-only. Cannot share with self (filter out current user.id from list). Returns 200.

4. `GET /sparql/saved?include_shared=true` -- Modify existing `saved_query_list` endpoint to accept optional `include_shared: bool = False` query param. When true, return a JSON object with two keys: `{ "my_queries": [...SavedQueryOut], "shared_with_me": [...SharedQueryOut] }`. For shared queries, join `SharedQueryAccess` with `SavedSparqlQuery` and `User` (the owner) to get `owner_name`. Compute `is_updated` by comparing `SavedSparqlQuery.updated_at > SharedQueryAccess.last_viewed_at` (treat null last_viewed_at as always updated). **Backward compatible:** when `include_shared=false` (default), return the existing `list[SavedQueryOut]` response.

5. `POST /sparql/saved/{query_id}/mark-viewed` -- Update `SharedQueryAccess.last_viewed_at = func.now()` for the current user + query_id pair. Used when recipient loads a shared query into the editor. Returns 204.

6. `POST /sparql/saved/{query_id}/fork` -- Fork a shared query as the current user's own. Create a new `SavedSparqlQuery` with `name="Copy of {original.name}"`, same `description` and `query_text`, user_id=current user. The original must be shared with the current user (check via SharedQueryAccess). Returns 201 with `SavedQueryOut`.

All new endpoints call `_enforce_sparql_role(user, "", False)` at the start for guest rejection.

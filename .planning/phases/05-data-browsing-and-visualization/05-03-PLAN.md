**Add to `service.py`** -- `execute_graph_query` method:
- Signature: `async def execute_graph_query(self, spec: ViewSpec) -> dict`
- The graph view spec SPARQL queries use CONSTRUCT (not SELECT). Use `scope_to_current_graph()` on the spec's query, then call `client.construct()` to get Turtle bytes.
- Parse with `rdflib.Graph().parse(data=turtle_bytes, format="turtle")`.
- Convert triples to Cytoscape-compatible JSON (per Research Example 2):
  - Iterate all triples (s, p, o).
  - Subjects are always nodes. Track `{id, types: set(), label: ""}` in a nodes dict keyed by IRI string.
  - `rdf:type` triples: add o to the node's types set.
  - Object property (o is URIRef): create edge `{source, target, predicate}` and ensure target is in nodes dict.
  - Datatype property: if predicate is a label property (dcterms:title, rdfs:label, foaf:name, skos:prefLabel, schema:name), set as the node's display label.
- Resolve labels for all node IRIs via LabelService (fallback for nodes without label properties).
- Build type-to-color mapping: query installed models for optional `sempkm:nodeColor` on ontology classes. Fall back to an auto-assigned color palette based on hashing the type IRI to one of 10 distinct colors. The palette: `['#4e79a7','#f28e2c','#e15759','#76b7b2','#59a14f','#edc949','#af7aa1','#ff9da7','#9c755f','#bab0ab']` (Tableau 10 palette).
- Return: `{"nodes": [...], "edges": [...], "type_colors": {...}}` where nodes have `{id, label, type}` and edges have `{source, target, predicate, predicate_label}`.

**Add `expand_neighbors` method:**
- Signature: `async def expand_neighbors(self, node_iri: str) -> dict`
- Execute a SPARQL CONSTRUCT that gets all triples where `node_iri` is subject or object (both directions), scoped to current graph.
- Parse and convert the same way as `execute_graph_query`.
- Return the same format `{nodes, edges, type_colors}` for merging into the existing graph.

**Add to `router.py`**:

Does NOT include graph data in the HTML -- the data is loaded via a separate JSON endpoint after the DOM is ready (per Research Pitfall 2: container must be visible before init).
- `GET /browser/views/graph/{spec_iri:path}/data` — JSON endpoint returning the graph data from `execute_graph_query`. Returns `application/json` (NOT an HTML partial). This is fetched by `graph.js` after the container is rendered.
- `GET /browser/views/graph/expand/{node_iri:path}` — JSON endpoint for neighbor expansion. Called by Cytoscape double-click handler.

- An inline `<script>` at the bottom that calls `initGraph('cy-container', '{{ spec_iri }}', {{ type_colors | tojson }})` which is defined in `graph.js`.

**`frontend/static/js/graph.js`**: Cytoscape.js initialization and interaction.
- `initGraph(containerId, specIri, typeColors)`:
  1. Fetch graph data from `/browser/views/graph/{specIri}/data` via `fetch()`.
  2. Initialize Cytoscape.js with the container, elements (mapped from the JSON), semantic styles from `buildSemanticStyle(typeColors)`, and default layout `{name: 'fcose'}` (or `{name: 'cose'}` if fcose not loaded).
  3. Store the `cy` instance on `window._sempkmGraph` for later access.
- `buildSemanticStyle(typeColors)`: Build Cytoscape stylesheet array per Research Example 3. Default node: circular, label below, 30x30, gray background. Per-type: background-color from typeColors mapping. Default edge: bezier, triangle arrow, label from data(label), autorotate, 9px font. Selected node: blue border. Hovered node: slightly larger.
- Event handlers:
  - `cy.on('tap', 'node', ...)`: Click to select. Load the node's details in the right pane via `loadRightPane(nodeId, 'relations')` (existing function from workspace.js).
  - `cy.on('dbltap', 'node', ...)`: Double-click to expand neighbors. Fetch `/browser/views/graph/expand/{nodeIri}`, parse JSON, add new elements to `cy`, run layout on ONLY the new elements (use `newEles.layout({name: currentLayout, animate: true, fit: false})`) per Research Pitfall 6.
- `changeLayout(layoutName)`: Re-run layout on all elements with the given layout name (fcose, dagre, concentric). Animate the transition. Store current layout name for expand operations.
- Pan and zoom are built into Cytoscape.js by default -- no custom code needed.

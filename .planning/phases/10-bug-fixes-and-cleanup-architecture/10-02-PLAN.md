**Step 3: Add JS positioning in `backend/app/templates/forms/_field.html`**

The reference field's search input (around line 82-97) uses `hx-trigger="input changed delay:300ms"`. After htmx swaps the suggestions content, the dropdown needs to be positioned relative to the input. Add a script block at the end of the `_field.html` file (after the `{% endmacro %}` on line 151) — but since this is a Jinja macro file, the positioning JS should instead go into the `object_form.html` script block.

Actually, the cleanest approach is to add an `htmx:afterSwap` event listener in the `object_form.html` script (after line 117). Add this function to the existing IIFE:

```javascript
// Position suggestion dropdowns after htmx swaps content into them
document.addEventListener('htmx:afterSwap', function(evt) {
  var target = evt.detail.target;
  if (!target || !target.classList || !target.classList.contains('suggestions-dropdown')) return;

  // Find the input that triggered the search
  var refField = target.closest('.reference-field');
  if (!refField) return;
  var input = refField.querySelector('.reference-search');
  if (!input) return;

  var rect = input.getBoundingClientRect();
  target.style.top = rect.bottom + 'px';
  target.style.left = rect.left + 'px';
  target.style.width = rect.width + 'px';
});
```

This listener fires each time htmx swaps suggestion results into a `.suggestions-dropdown` div, calculating the position from the input's screen coordinates. Because the dropdown is `position: fixed`, it renders relative to the viewport and escapes any `overflow: hidden/auto` ancestor.

Also add a scroll/resize listener to reposition open dropdowns:

```javascript
// Reposition open suggestion dropdowns on scroll/resize
function repositionDropdowns() {
  var openDropdowns = document.querySelectorAll('.suggestions-dropdown:not(:empty)');
  openDropdowns.forEach(function(dd) {
    var refField = dd.closest('.reference-field');
    if (!refField) return;
    var input = refField.querySelector('.reference-search');
    if (!input) return;
    var rect = input.getBoundingClientRect();
    dd.style.top = rect.bottom + 'px';
    dd.style.left = rect.left + 'px';
    dd.style.width = rect.width + 'px';
  });
}

// Throttled reposition on form section scroll
var formSections = document.querySelectorAll('.object-form-section');
formSections.forEach(function(section) {
  section.addEventListener('scroll', repositionDropdowns);
});
window.addEventListener('resize', repositionDropdowns);
```

NOTE: The `.reference-field` parent still needs `position: relative` for the dropdown's DOM placement. The dropdown is a child of `.reference-field` in the DOM (for htmx targeting), but uses `position: fixed` for visual placement.

In `backend/app/templates/browser/workspace.html`, the views explorer section (line 21-34) has:

```html
<div class="explorer-section-header"
     onclick="this.parentElement.classList.toggle('expanded')"
     hx-get="/browser/views/explorer"
     hx-target="#views-tree"
     hx-trigger="click once"
     hx-swap="innerHTML">
```

Change `hx-trigger="click once"` to `hx-trigger="load"` so that the htmx request fires when the element enters the DOM (i.e., when the workspace loads), instead of waiting for a user click.

The `onclick` handler for toggling the `expanded` class should remain — it controls collapse/expand visibility. The `hx-get` request will fire once on load, populating the `#views-tree` div.

Also update the loading placeholder (line 32) to be more descriptive:

```html
<div class="explorer-section-body" id="views-tree">
    <div class="tree-empty">Loading views...</div>
</div>
```

The `hx-trigger="load"` ensures the request fires exactly once when the element is first rendered. There is no need for a "once" modifier since htmx with `load` trigger fires once by default on initial render.

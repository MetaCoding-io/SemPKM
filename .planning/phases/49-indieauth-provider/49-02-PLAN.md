<task type="auto">
  <name>Task 1: Create IndieAuth router with all endpoints</name>
  <files>backend/app/indieauth/router.py, backend/app/main.py</files>
  <action>
    Create `backend/app/indieauth/router.py` with TWO routers:
    - `router = APIRouter(prefix="/api/indieauth", tags=["indieauth"])` — auth-protected endpoints
    - `public_router = APIRouter(tags=["indieauth"])` — no auth required (metadata, well-known, token, introspect)

    **Endpoints:**

    1. `GET /api/indieauth/metadata` (public_router):
       - Return JSON per IndieAuth spec Section 4.1.
       - Use `get_base_url(request)` for constructing URLs.
       - Fields: issuer, authorization_endpoint, token_endpoint, introspection_endpoint, code_challenge_methods_supported (["S256"]), scopes_supported (from SCOPE_REGISTRY keys), response_types_supported (["code"]), grant_types_supported (["authorization_code", "refresh_token"]), introspection_endpoint_auth_methods_supported (["none"]), authorization_response_iss_parameter_supported (True).

    2. `GET /.well-known/oauth-authorization-server` (public_router):
       - RedirectResponse to `/api/indieauth/metadata` with status_code=307.

    3. `GET /api/indieauth/authorize` (router, requires current_user via dependency):
       - Query params: response_type, client_id, redirect_uri, state, scope (optional, default "profile"), code_challenge, code_challenge_method.
       - Validate: response_type == "code", client_id is valid URL, code_challenge_method == "S256".
       - If user is not authenticated (use try/except on get_current_user or use optional_current_user and redirect if None): redirect to `/login.html?next=` with the full authorize URL as the next param.
       - Fetch client info via `IndieAuthService.fetch_client_info(client_id)`.
       - Validate scopes via `validate_scopes(scope)`.
       - Render consent.html template with: client_info, scopes (with descriptions from SCOPE_REGISTRY), user identity (profile URL), redirect_uri, all original query params passed through as hidden form fields.
       - On validation error: return 400 JSON error with description.

    4. `POST /api/indieauth/authorize` (router, requires current_user):
       - Form params: action ("approve" or "deny"), client_id, redirect_uri, scope, code_challenge, code_challenge_method, state.
       - If action == "deny": redirect to redirect_uri with error=access_denied and state.
       - If action == "approve": create authorization code via service, redirect to redirect_uri with code, state, and iss parameter.
       - The `iss` parameter value is the metadata endpoint URL (per research pitfall 3).

    5. `POST /api/indieauth/token` (public_router):
       - CRITICAL: Use `Form(...)` parameters, NOT JSON body (per research pitfall 2).
       - Params: grant_type, code, client_id, redirect_uri, code_verifier, refresh_token.
       - If grant_type == "authorization_code": exchange code via service, return TokenResponse JSON.
       - If grant_type == "refresh_token": refresh via service, return TokenResponse JSON.
       - On error: return 400 JSON with `error` and `error_description` fields (OAuth2 error format).
       - Construct `me` URL using the user's username from the token row.

    6. `POST /api/indieauth/introspect` (public_router):
       - Form param: token.
       - Call service.introspect_token(), return IntrospectionResponse JSON.
       - Per spec: always return 200 even for invalid tokens (active=false).

    7. `DELETE /api/indieauth/tokens/{token_id}` (router, requires current_user):
       - Revoke a specific token by ID. For the settings "Authorized Apps" UI.
       - Verify the token belongs to the current user before deleting.

    8. `GET /api/indieauth/tokens` (router, requires current_user):
       - List authorized apps for current user via service.list_authorized_apps().
       - Return JSON array of token info (id, client_id, client_name, scope, created_at, expires_at).

    **Register in main.py:**
    - Import `from app.indieauth.router import router as indieauth_router, public_router as indieauth_public_router`
    - Add `app.include_router(indieauth_router)` and `app.include_router(indieauth_public_router)`.
    - Place before `browser_router` (same pattern as webid_router).

    **Run migration:**
    - After code changes, run `docker compose exec api alembic upgrade head` to apply migration 006.
    - Then rebuild docker to pick up new mf2py dependency: `docker compose up -d --build api`.

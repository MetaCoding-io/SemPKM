  <how-to-verify>
Open the Docker stack (http://localhost:3901) and test these scenarios in order:

**1. Basic tab opening**
- Open the workspace (Object Browser)
- Click any object in the nav tree
- Confirm: A dockview tab appears with the object content loaded (not the old .editor-group DOM)
- Expected: Tab bar appears at top of editor area with dockview styling (themed to match SemPKM colors)

**2. Multiple tabs**
- Click 2-3 more different objects
- Confirm: Each opens as a separate tab in the same dockview group
- Click between tabs: each click loads the correct object content
- Expected: Tab switching works without page reload; right pane (Relations, Lint) updates to match active tab

**3. Drag-to-reorder**
- With 3+ tabs open, drag one tab to a different position within the same group
- Expected: Tab reorders; dockview handles the drag natively (no custom JS drag code); no "disappeared" tabs

**4. Drag-to-split**
- Drag a tab to the right edge of the tab bar until a blue overlay appears
- Drop it: a second panel group should appear side by side
- Expected: Two groups visible; both show correct content; resize sash between them is draggable

**5. Layout persistence**
- With 2 groups and multiple tabs open, reload the page (F5)
- Expected: Same group geometry and same tabs restore (dockview fromJSON restores from sessionStorage)

**6. sempkm:tab-activated event**
- Open browser DevTools console
- Run: `document.addEventListener('sempkm:tab-activated', function(e) { console.log('TAB EVENT', e.detail); })`
- Switch between tabs
- Expected: Console shows `{ tabId: '...', groupId: '...', isObjectTab: true }` for object tabs

**7. sempkm:tabs-empty event**
- Open browser DevTools console
- Run: `document.addEventListener('sempkm:tabs-empty', function() { console.log('TABS EMPTY'); })`
- Close all open tabs
- Expected: Console shows "TABS EMPTY" when last object tab is closed

**8. Relations panel (htmx after drag)**
- Open an object tab
- Drag it to create a new group (split)
- Click inside the moved panel's content (e.g., click a relation link or lint button)
- Expected: htmx actions still work — relations load, forms submit; no silent failures

**9. CodeMirror after panel hide (if applicable)**
- Open an object with a Markdown body editor
- Switch to a different tab in the same group (hiding the editor tab)
- Switch back
- Expected: Editor renders at full size, not as a blank strip or single line

**10. View tabs**
- Open a view (click from the VIEWS section in the left nav)
- Expected: Opens in a dockview tab (not as an object tab); isObjectTab=false in event detail

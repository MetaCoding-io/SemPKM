  Modify the `.tree-leaf` div in `tree_children.html` to add HTML5 drag-and-drop attributes:

  1. Add `draggable="true"` to the `.tree-leaf` div.
  2. Add an `ondragstart` handler that sets two data transfer items:
     - `event.dataTransfer.setData('text/iri', '{{ obj.iri }}')` — the object IRI
     - `event.dataTransfer.setData('text/label', '{{ obj.label }}')` — the display label
     - `event.dataTransfer.effectAllowed = 'copy'`

  The result should look like:
  ```html
  <div class="tree-leaf" data-testid="nav-item"
       draggable="true"
       ondragstart="event.dataTransfer.setData('text/iri', '{{ obj.iri }}'); event.dataTransfer.setData('text/label', '{{ obj.label }}'); event.dataTransfer.effectAllowed = 'copy';"
       data-tooltip-label="{{ obj.label }}"
       data-tooltip-type="{{ type_label | default('') }}"
       onclick="openTab('{{ obj.iri }}', '{{ obj.label }}')">
  ```

  Do NOT add `draggable` to the `.tree-empty` div or type header nodes (these are in `nav_tree.html`, not modified here). Only leaf items (individual objects) are draggable per user decision.

  Note: Use lowercase `'text/iri'` and `'text/label'` — HTML5 DnD spec normalizes MIME type strings to lowercase (per RESEARCH.md Pitfall 2).

<task type="auto">
  <name>Task 2: Add canvas drop handlers and drop zone highlight styling</name>
  <files>frontend/static/js/canvas.js, frontend/static/css/workspace.css</files>
  <action>
  **canvas.js changes:**

  1. In `bindEvents()`, add three new event listeners on `state.viewport`:

     ```javascript
     state.viewport.addEventListener('dragover', onDragOver);
     state.viewport.addEventListener('dragleave', onDragLeave);
     state.viewport.addEventListener('drop', onDrop);
     ```

  2. Implement `onDragOver(event)`:
     - Check `event.dataTransfer.types` contains `'text/iri'` (use `indexOf` for ES5 compat).
     - If yes: `event.preventDefault()`, set `event.dataTransfer.dropEffect = 'copy'`, add class `canvas-drop-active` to `state.viewport`.
     - If no: do nothing (ignore drags from other sources).

  3. Implement `onDragLeave(event)`:
     - Only remove the `canvas-drop-active` class if the related target is outside the viewport (check `!state.viewport.contains(event.relatedTarget)`). This prevents flicker when dragging over child elements within the viewport.

  4. Implement `onDrop(event)`:
     - `event.preventDefault()`.
     - Remove `canvas-drop-active` class from `state.viewport`.
     - Read `var iri = event.dataTransfer.getData('text/iri')`.
     - Read `var label = event.dataTransfer.getData('text/label')`.
     - If `!iri` return early.
     - Check `findNode(iri)` — if node already exists: call `setStatus('Already on canvas')` and `if (window.showToast) window.showToast('Already on canvas')`, then return.
     - Convert drop position: `var world = screenToWorld(event.clientX, event.clientY)`.
     - Push new node to `state.nodes`:
       ```javascript
       state.nodes.push({
         id: iri,
         title: label || 'Resource',
         uri: iri,
         x: Math.round(world.x),
         y: Math.round(world.y),
         markdown: '',
         collapsed: false,
       });
       ```
     - Call `renderNodes()` (this also hides the hint text if nodes > 0).
     - Call `setStatus('Added: ' + (label || iri))`.

  **workspace.css changes:**

  Add drop zone highlight style (after the `.spatial-canvas-hint` rule added in Plan 01):

  ```css
  .spatial-canvas-viewport.canvas-drop-active {
    background-color: rgba(37, 99, 235, 0.06);
    outline: 2px dashed var(--color-accent, #2563eb);
    outline-offset: -2px;
  }
  ```

  Also add a `cursor: grab` override so the drop cursor shows correctly:
  ```css
  .spatial-canvas-viewport.canvas-drop-active {
    cursor: copy;
  }
  ```
  (Merge with the above rule.)

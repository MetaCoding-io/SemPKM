   ```javascript
   var GRID = 24;
   function snapToGrid(value) { return Math.round(value / GRID) * GRID; }
   ```

  <name>Task 2: Keyboard navigation and node selection</name>
  <files>frontend/static/js/canvas.js, frontend/static/css/workspace.css</files>
  <action>
1. Add `selectedNodeId: null` to the `state` object at the top of the IIFE.

2. Add a `cycleSelection(direction)` function that sorts nodes spatially (by y then x) and moves selection forward (+1) or backward (-1). When direction is +1 and at end, wrap to first. When -1 and at start, wrap to last.

3. Add an `onKeyDown(event)` handler with these guards:
   - If `document.activeElement` is `INPUT`, `TEXTAREA`, or `SELECT`, return immediately (research pitfall 2).
   - If `document.activeElement` is inside a `.dv-tabs-container` or `.cm-editor`, return (avoid capturing keys from other panels).
   - Only process if canvas tab is active: check `state.mounted && state.viewport`.

4. Implement key bindings per user decisions:
   - Arrow keys: move selected node by `GRID` (24px). If `event.shiftKey`, move by `GRID * 5` (120px). Call `renderNodes()` after.
   - Tab: `cycleSelection(1)`. Shift+Tab: `cycleSelection(-1)`. Always `event.preventDefault()`.
   - Delete / Backspace: call `removeNode(state.selectedNodeId)`, clear selection. No confirmation dialog.
   - Enter: call `toggleExpand(state.selectedNodeId)`.
   - Escape: clear `state.selectedNodeId`, call `renderNodes()`.
   - `s` with ctrlKey/metaKey: call `saveCanvas()`, `event.preventDefault()`.

5. In `renderNodes()`, add CSS class `spatial-node-selected` to the `<article>` element when its `data-node-id` matches `state.selectedNodeId`. Modify the node HTML template (line ~474) to conditionally include the class.

6. In `onLayerClick` or `onPointerDown` on the node header, set `state.selectedNodeId` to the clicked node's ID. This gives click-to-select behavior.

7. Register the keyboard handler in `bindEvents()`: `document.addEventListener('keydown', onKeyDown)`.

8. Add CSS for the focus ring in workspace.css:

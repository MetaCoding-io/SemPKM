   ```python
   class WikilinkResolveRequest(BaseModel):
       titles: list[str] = Field(..., max_length=50, description="Wiki-link display text to resolve to IRIs")

   class WikilinkResolveResponse(BaseModel):
       resolved: dict[str, str | None]  # {title: iri_or_null}
   ```

2. Add a new endpoint to `router.py`:
   ```python
   @router.post("/resolve-wikilinks", response_model=WikilinkResolveResponse)
   async def resolve_wikilinks(

       user: User = Depends(get_current_user),
       client: TriplestoreClient = Depends(get_triplestore_client),
   ):
   ```

3. The endpoint resolves wiki-link title text to object IRIs using SPARQL. For each title in the request:
   - Query the triplestore for objects whose label properties (dcterms:title, rdfs:label, skos:prefLabel, schema:name, foaf:name) match the title text (case-insensitive exact match).
   - Use a single SPARQL query with VALUES clause for batch resolution:
     ```sparql
     SELECT ?title ?iri WHERE {
       GRAPH <urn:sempkm:current> {
         ?iri ?labelProp ?title .
         VALUES ?labelProp {
           <http://purl.org/dc/terms/title>
           <http://www.w3.org/2000/01/rdf-schema#label>
           <http://www.w3.org/2004/02/skos/core#prefLabel>
           <https://schema.org/name>
           <http://xmlns.com/foaf/0.1/name>
         }
       }
       FILTER(?title IN ({titles_list}))
     }
     ```
   - Build the `titles_list` by properly escaping each title as a SPARQL string literal.
   - Return a dict mapping each requested title to its resolved IRI (or null if not found).
   - If multiple IRIs match the same title, return the first one found.

4. Import `WikilinkResolveRequest` and `WikilinkResolveResponse` in router.py.

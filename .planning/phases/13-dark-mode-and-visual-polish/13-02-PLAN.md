  <name>Task 1: CodeMirror Compartment theme switching and highlight.js swap</name>
  <files>
    frontend/static/js/editor.js
    frontend/static/js/theme.js
  </files>
  <action>
**Modify `frontend/static/js/editor.js`**:

1. Import `Compartment` from `@codemirror/state@6` (add to existing imports).
2. Create a module-scope `themeCompartment = new Compartment()` — shared across all editor instances.
3. Define two theme objects using `EditorView.theme({...}, {dark: true/false})`:
   - `darkEditorTheme`: background #282c34, text #abb2bf, caret/cursor #56b6c2, gutters bg #21252b, gutter text #5c6370, gutter border #3e4452, active line bg #2c313a, selection bg #3E4451. Pass `{dark: true}` as second arg so CM6 enables built-in dark defaults for unstyled elements.
   - `lightEditorTheme`: background #ffffff, text #1a1a2e, gutters bg #f8f9fb, gutter text #666, gutter border #e0e0e0. Pass `{dark: false}`.
4. In `initEditor()`, add `themeCompartment.of(getCurrentTheme())` to the extensions array (where `getCurrentTheme()` checks `document.documentElement.getAttribute('data-theme') === 'dark'` to pick dark or light theme).
5. Create and expose `window.switchEditorThemes = function(isDark)` that iterates all entries in the `editors` object and calls `view.dispatch({ effects: themeCompartment.reconfigure(isDark ? darkEditorTheme : lightEditorTheme) })` on each.

**Modify `frontend/static/js/theme.js`**:

1. In the `applyTheme(resolved)` function, add calls to third-party switchers:
   - `if (typeof window.switchEditorThemes === 'function') window.switchEditorThemes(resolved === 'dark');`
   - `switchHighlightTheme(resolved === 'dark');`
   - `switchNinjaTheme(resolved === 'dark');`
2. Add `switchHighlightTheme(isDark)` function: find `document.getElementById('hljs-theme')`, swap href between `github.min.css` and `github-dark.min.css` (full CDN URLs from base.html).
3. Add `switchNinjaTheme(isDark)` function: find `document.querySelector('ninja-keys')`, toggle `.dark` class. The CSS variable overrides in theme.css (from Plan 01) handle the rest.

Note: `switchGraphTheme` is handled in Task 2 below, but theme.js should call it here too:
   - `if (typeof window.switchGraphTheme === 'function') window.switchGraphTheme(resolved === 'dark');`
  </action>
  <verify>
1. Open an object with a Markdown body — verify CodeMirror editor appears
2. Toggle to dark mode — editor background, text, gutters, cursor, and selection all switch to dark colors
3. Toggle back to light — editor returns to light colors
4. Verify cursor position and undo history are preserved across theme switches (type text, undo after switch)
5. View a read-only object with code blocks — verify highlight.js code blocks switch between github and github-dark themes
6. Open command palette — verify ninja-keys has dark styling in dark mode
  </verify>
  <done>
CodeMirror editors switch themes via Compartment.reconfigure() without recreation. highlight.js stylesheet swaps between light and dark. ninja-keys toggles dark class. All third-party text/code rendering responds to theme changes.
  </done>

  <name>Task 2: Cytoscape graph dark mode style rebuild</name>
  <files>

  <action>
**Modify `frontend/static/js/graph.js`**:

1. Modify `buildSemanticStyle(typeColors)` to accept a second parameter `isDark` (default false):
   - When `isDark=true`, change default node style: `'color': '#abb2bf'` (label text), `'background-color': '#5c6370'`, `'border-color': '#3e4452'`
   - When `isDark=true`, change default edge style: `'line-color': '#3e4452'`, `'target-arrow-color': '#3e4452'`, `'color': '#7d8799'` (label text), `'text-background-color': '#282c34'`
   - When `isDark=true`, change selected node: `'border-color': '#56b6c2'` (teal accent instead of blue)
   - Light mode values remain current hardcoded values (which are the existing defaults)
2. Store `window._sempkmTypeColors` when graph data loads (in the existing graph init function) so it is available for theme rebuilds.
3. Create and expose `window.switchGraphTheme = function(isDark)`:
   - Check `window._sempkmGraph` (the Cytoscape instance)
   - If it exists, rebuild styles: `var styles = buildSemanticStyle(window._sempkmTypeColors, isDark); cy.style().fromJson(styles).update();`
   - Also update the container background if needed (though this should be handled by CSS token on the graph container element).
4. Listen for `sempkm:theme-changed` event on `document` as a backup integration point: `document.addEventListener('sempkm:theme-changed', function(e) { switchGraphTheme(e.detail.theme === 'dark'); });`

**Modify `frontend/static/css/theme.css`**:
Add graph container background token if not already present: the `.graph-container` or `#cy` element should use `var(--color-surface)` for its background (this was likely already handled in Plan 01's CSS migration, but verify and add if missing).

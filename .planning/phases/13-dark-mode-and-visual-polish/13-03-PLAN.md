  <action>
**Rewrite `backend/app/templates/errors/403.html`** — replace the current minimal page with a styled permission panel:

1. The 403 page is a standalone page (not using base.html) because it renders outside the authenticated dashboard layout. It needs its OWN anti-FOUC script and theme.css link.

2. Structure:
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Access Denied - SemPKM</title>
    <!-- Anti-FOUC script (same as base.html) -->
    <script>
    (function() {
      var stored = localStorage.getItem('sempkm_theme');
      var theme = 'light';
      if (stored === 'dark') theme = 'dark';
      else if (stored === 'system' || !stored) theme = window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
      document.documentElement.setAttribute('data-theme', theme);
    })();
    </script>
    <link rel="stylesheet" href="/css/theme.css">
    <link rel="stylesheet" href="/css/style.css">
    <!-- Lucide for lock icon -->
    <script src="https://unpkg.com/lucide@0.575.0/dist/umd/lucide.min.js"></script>
</head>
```

3. Body content — a centered panel with:
   - Large lock icon (Lucide `lock` or `shield-alert`, 48px, using `--color-text-faint`)
   - Title: "Access Denied" (h2, `--color-text`)
   - Message explaining the situation: "You don't have permission to access this resource. Your current role does not include the required privileges." (`--color-text-muted`)
   - Two action buttons in a row:
     - "Go Home" — primary button (accent bg), links to `/`
     - "Go Back" — secondary/outline button, calls `history.back()`
   - All styling uses theme tokens so it works in both light and dark mode

4. Style the panel using either inline `<style>` or classes that reference theme.css tokens:
   - `.error-panel`: centered card with max-width 480px, surface background, border, rounded corners, shadow, padding 2.5rem
   - `.error-panel-icon`: 64px circle with `--color-surface-recessed` background, lock icon centered inside
   - `.error-panel-title`: 1.25rem font, `--color-text`, margin-bottom
   - `.error-panel-message`: `--color-text-muted`, line-height 1.6
   - `.error-panel-actions`: flex row, gap 0.75rem, centered
   - `.btn-primary`: `--color-accent` bg, white text, 6px radius, hover darken
   - `.btn-secondary`: transparent bg, `--color-border` border, `--color-text-muted` text, hover `--color-surface-hover` bg

5. Add `<script>lucide.createIcons();</script>` before `</body>` to initialize the lock icon.

**Add error panel tokens to theme.css** if needed (e.g., button styles). Likely already covered by existing accent and surface tokens.

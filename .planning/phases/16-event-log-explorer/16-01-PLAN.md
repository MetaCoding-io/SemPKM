Define two dataclasses:
```python
@dataclass
class EventSummary:
    event_iri: str
    timestamp: str        # ISO datetime string from SPARQL
    operation_type: str
    affected_iris: list[str]   # CSV-split from GROUP_CONCAT result
    performed_by: str | None   # user IRI, None if absent
    description: str | None
```

Define `EventQueryService`:
- `__init__(self, client: TriplestoreClient)` — store as `self._client`
- `PAGE_SIZE = 50` class constant
- `async def list_events(self, cursor_timestamp=None, op_type=None, user_iri=None, object_iri=None, date_from=None, date_to=None) -> tuple[list[EventSummary], str | None]`

In `list_events`:
1. Build SPARQL filter clauses list (start empty):
   - cursor_timestamp: `FILTER(?timestamp < "{cursor_timestamp}"^^xsd:dateTime)`
   - op_type: `FILTER(STR(?opType) = "{op_type}")`
   - user_iri: `FILTER(?performedBy = <{user_iri}>)`
   - object_iri: `FILTER(?affectedIRI = <{object_iri}>)`
   - date_from: `FILTER(?timestamp >= "{date_from}"^^xsd:dateTime)`
   - date_to: `FILTER(?timestamp <= "{date_to}"^^xsd:dateTime)`
2. Join filters with `\n  ` as `filter_block`
3. Execute SPARQL using GROUP_CONCAT to collapse multiple affectedIRI rows:
```sparql
PREFIX sempkm: <urn:sempkm:>
PREFIX xsd: <http://www.w3.org/2001/XMLSchema#>

SELECT ?event ?timestamp ?opType (GROUP_CONCAT(STR(?affectedIRI); separator=",") AS ?affected) ?performedBy ?description
WHERE {
  GRAPH ?event {
    ?event a sempkm:Event ;
           sempkm:timestamp ?timestamp ;
           sempkm:operationType ?opType ;
           sempkm:affectedIRI ?affectedIRI .
    OPTIONAL { ?event sempkm:performedBy ?performedBy }
    OPTIONAL { ?event sempkm:description ?description }
  }
  FILTER(STRSTARTS(STR(?event), "urn:sempkm:event:"))
  {filter_block}
}
GROUP BY ?event ?timestamp ?opType ?performedBy ?description
ORDER BY DESC(?timestamp)
LIMIT 51
```
4. Parse JSON SPARQL results: `result["results"]["bindings"]`. Extract each binding's `.value` field for each var. Split `?affected` on comma to get `affected_iris` list (filter empty strings).
5. Build list of `EventSummary`. Take first 50. If 51 rows returned, `next_cursor = rows[49].timestamp`; else `next_cursor = None`.
6. Return `(events[:50], next_cursor)`.

**Fallback for GROUP_CONCAT**: If RDF4J returns an error (catch `Exception` from the query call), retry with a plain SELECT (no GROUP_CONCAT) and do Python-side grouping using an `OrderedDict` keyed on event IRI, appending `affectedIRI` values.

**Imports needed**: `from app.triplestore.client import TriplestoreClient`, `from dataclasses import dataclass, field`, standard `from __future__ import annotations` for type hints.

Add route `GET /browser/events`:

```python
@router.get("/events")
async def event_log(
    request: Request,
    user: User = Depends(get_current_user),
    client: TriplestoreClient = Depends(get_triplestore_client),
    label_service: LabelService = Depends(get_label_service),
    cursor: str | None = QueryParam(default=None),
    op: str | None = QueryParam(default=None),
    user_filter: str | None = QueryParam(default=None, alias="user"),
    obj: str | None = QueryParam(default=None),
    date_from: str | None = QueryParam(default=None),
    date_to: str | None = QueryParam(default=None),
):
    from app.events.query import EventQueryService
    templates = request.app.state.templates
    query_svc = EventQueryService(client)
    events, next_cursor = await query_svc.list_events(
        cursor_timestamp=cursor, op_type=op, user_iri=user_filter,
        object_iri=obj, date_from=date_from, date_to=date_to,
    )
    all_iris = [iri for e in events for iri in e.affected_iris if iri]
    labels = await label_service.resolve_batch(all_iris) if all_iris else {}

    # Resolve user display names: extract UUID from urn:sempkm:user:{uuid}
    # Use request.app.state.db (SQLAlchemy session) to look up users by id
    # Build user_names dict: {user_iri: display_name or email}
    user_iris = list({e.performed_by for e in events if e.performed_by})
    user_names = {}
    if user_iris:
        try:
            from app.auth.models import User as UserModel
            from sqlalchemy import select as sa_select
            import re
            db = request.app.state.db  # synchronous session factory or async
            # Extract UUID strings
            for uiri in user_iris:
                m = re.match(r"urn:sempkm:user:(.+)$", uiri)
                if m:
                    uuid_str = m.group(1)
                    # Use the existing db session pattern from auth routes
                    # db is a sessionmaker; open a session
                    with db() as session:
                        result = session.execute(
                            sa_select(UserModel).where(UserModel.id == uuid_str)
                        ).scalar_one_or_none()
                        if result:
                            user_names[uiri] = result.display_name or result.email
        except Exception:
            pass  # degrade gracefully if user lookup fails

    active_filters = []
    if op:
        active_filters.append({"param": "op", "value": op, "label": f"op: {op}"})
    if obj:
        active_filters.append({"param": "obj", "value": obj, "label": f"object: {labels.get(obj, obj[:30] + '...' if len(obj) > 30 else obj)}"})
    if user_filter:
        active_filters.append({"param": "user", "value": user_filter, "label": f"user: {user_names.get(user_filter, user_filter)}"})
    if date_from:
        active_filters.append({"param": "date_from", "value": date_from, "label": f"from: {date_from}"})
    if date_to:
        active_filters.append({"param": "date_to", "value": date_to, "label": f"to: {date_to}"})

    return templates.TemplateResponse(request, "browser/event_log.html", {
        "request": request,
        "events": events,
        "labels": labels,
        "user_names": user_names,
        "next_cursor": next_cursor,
        "active_filters": active_filters,
        "current_params": dict(request.query_params),
    })
```

Use whatever pattern exists — do not invent a new one. If the session is async, use `await`. The user lookup is best-effort; wrap in try/except and return empty user_names dict if it fails.

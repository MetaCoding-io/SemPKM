<task id="27-02-1">
  <title>Create backend/app/vfs/write.py — body parsing and event store bridge</title>
  <action>
Create `backend/app/vfs/write.py` with two functions:

**`parse_dav_put_body(raw_bytes: bytes) -> str`**

Accepts the raw PUT body bytes and returns only the Markdown body string (without frontmatter).

Logic:
1. Decode bytes as UTF-8
2. If the content starts with `---\n` or `---\r\n`, scan for the closing `---` delimiter on its own line
3. Everything after the closing `---` (stripping one leading newline) is the body
4. If no frontmatter delimiters found: the entire content is the body (pass through as-is)

Example:
```
Input:
---
iri: http://example.org/note/1
type: Note
---
# My Note

Body content here.

Output:
# My Note

Body content here.
```

**`write_body_via_event_store(object_iri: str, body: str, event_store, user_iri: str, user_role: str) -> None`**

This function is called from the wsgidav WSGI thread. Because EventStore uses `async def commit()`, we need to bridge the sync/async boundary.

Implementation:
```python
import asyncio

async def _commit_body_set(object_iri, body, event_store, user_iri_str, user_role):
    from rdflib import URIRef
    from app.commands.handlers.body_set import handle_body_set
    from app.commands.schemas import BodySetParams
    from app.config import settings

    params = BodySetParams(iri=object_iri, body=body)
    operation = await handle_body_set(params, settings.base_namespace)
    user_iri = URIRef(user_iri_str) if user_iri_str else None
    await event_store.commit([operation], performed_by=user_iri, performed_by_role=user_role)

def write_body_via_event_store(object_iri, body, event_store, user_iri=None, user_role="member"):
    """Bridge sync wsgidav context to async EventStore.commit().

    Creates a new event loop in the current thread (safe in WSGI thread pool
    where no event loop is running).
    """
    asyncio.run(_commit_body_set(object_iri, body, event_store, user_iri, user_role))
```

Note: `asyncio.run()` creates a new event loop per call — correct here because wsgidav threads have no running loop. Do NOT use `asyncio.get_event_loop().run_until_complete()` which raises RuntimeError if a loop is already running in the main thread.

def get_etag(self):
    """Return ETag as SHA-256 hex of the current body content."""
    import hashlib
    content = self._get_content_bytes()  # existing method that returns file bytes
    return hashlib.sha256(content).hexdigest()

<task id="27-02-3">
  <title>Implement write_data() on ResourceFile to handle PUT with ETag check</title>
  <action>
Add `write_data()` to `ResourceFile` in `backend/app/vfs/provider.py`. This is the wsgidav hook called when a client sends `PUT /dav/{model}/{type}/{label}.md`.

wsgidav calls `write_data(content_file, content_type, content_length, dry_run)` where `content_file` is a file-like object.

Implementation:
```python
def write_data(self, content_file, content_type, content_length, dry_run):
    """Handle DAV PUT: parse body, check ETag, commit body.set via event store.

    ETag concurrency:
    - If If-Match header present and doesn't match current ETag: raise DAVError(412)
    - If If-Match absent or matching: proceed with write

    Frontmatter in PUT body is silently ignored — only body content is written.
    """
    from wsgidav.dav_error import DAVError, HTTP_PRECONDITION_FAILED
    from app.vfs.write import parse_dav_put_body, write_body_via_event_store

    if dry_run:
        return

    # Check ETag concurrency
    environ = self._environ  # wsgidav passes environ during resource construction
    if_match = environ.get("HTTP_IF_MATCH")
    if if_match:
        # Strip quotes from ETag value: "abc123" -> abc123
        current_etag = self.get_etag()
        client_etag = if_match.strip('"')
        if client_etag != "*" and client_etag != current_etag:
            raise DAVError(HTTP_PRECONDITION_FAILED)

    # Read full PUT body
    raw_bytes = content_file.read()

    # Strip frontmatter, extract body only
    body = parse_dav_put_body(raw_bytes)

    if not dry_run:
        # Get user context from environ (set by SemPKMTokenAuthenticator)
        user_iri = environ.get("sempkm.user_id")
        user_role = environ.get("sempkm.user_role", "member")
        user_iri_str = f"urn:sempkm:user:{user_iri}" if user_iri else None

        # Get event store from the wsgidav config (stored during app startup)
        event_store = self._event_store  # see next task for wiring

        write_body_via_event_store(
            self._object_iri,
            body,
            event_store,
            user_iri=user_iri_str,
            user_role=user_role,
        )
```

`self._object_iri` is the RDF IRI of the object this file represents (set in Phase 26 when constructing ResourceFile).
`self._environ` is the WSGI environ dict from the current request (wsgidav passes it — check Phase 26's ResourceFile constructor to see if it's already stored, or add it).
`self._event_store` is the async EventStore instance (to be wired in the next task).

Also update `get_content_type()` to return `"text/markdown"` if not already done.

Finally, remove any Phase 26 PUT/DELETE/MOVE/COPY method stubs that returned 405 — instead, let write_data() handle PUT, and ensure DELETE/MOVE/COPY/MKCOL still return 405. In wsgidav, the presence of `write_data()` signals that PUT is supported; absence of `delete()` means DELETE returns 405 automatically.

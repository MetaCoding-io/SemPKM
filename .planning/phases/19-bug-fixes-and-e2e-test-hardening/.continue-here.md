<context>
User is doing a live browser walkthrough, fixing bugs one by one. Dev stack at http://localhost:3000.
Docker container must be restarted after Python changes (no hot-reload): docker restart sempkm-api-1.

Key architectural note: RDF4J transaction API (POST to txn_url with Content-Type:
application/sparql-update) accepts ONE update statement per call. Multiple statements
joined by ';' are silently mishandled — the behavior was that the second statement
would run but the first would not, or vice versa.

The fix pattern for future store changes: always call transaction_update in a loop,
once per operation, not combined into a single multi-statement string.

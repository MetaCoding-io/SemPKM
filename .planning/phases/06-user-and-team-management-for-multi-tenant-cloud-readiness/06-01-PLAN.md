1. **backend/app/db/__init__.py** — Empty package init.

2. **backend/app/db/base.py** — Create the SQLAlchemy declarative base:
   ```python
   from sqlalchemy.orm import DeclarativeBase
   class Base(DeclarativeBase):
       pass
   ```

3. **backend/app/db/engine.py** — Create dual-database engine factory per Research Pattern 1:
   - Import `create_async_engine` from `sqlalchemy.ext.asyncio`
   - Import `settings` from `app.config`
   - Function `create_engine()` returns `AsyncEngine`
   - If URL starts with `"sqlite"`, add `connect_args={"check_same_thread": False}`
   - Pass `echo=settings.debug`

4. **backend/app/db/session.py** — Create AsyncSession factory per Research code example:
   - `engine = create_engine()` at module level
   - `async_session_factory = async_sessionmaker(engine, expire_on_commit=False)`
   - `async def get_db_session()` as FastAPI dependency yielding session with commit/rollback

5. **backend/app/auth/__init__.py** — Empty package init.

6. **backend/app/auth/models.py** — Create 4 ORM models per Research Pattern 5:
   - **User**: id (UUID PK), email (String 255, unique, indexed), display_name (String 255, nullable), role (String 20, not Enum -- use String to avoid SQLite/PG Enum differences per Pitfall 1), created_at (DateTime timezone=True, server_default=func.now()). Relationship to sessions.
   - **UserSession**: id (UUID PK), token (String 64, unique, indexed), user_id (FK to users.id), created_at, expires_at (DateTime timezone=True). Relationship to user.
   - **Invitation**: id (UUID PK), email (String 255, indexed), role (String 20), token (String 64, unique, indexed), invited_by (FK to users.id), created_at, expires_at, accepted_at (nullable).
   - **InstanceConfig**: key (String 255, PK), value (String 4096), updated_at.
   - IMPORTANT: Use `String(20)` for role fields instead of `Enum(native_enum=False)` to avoid SQLite/PostgreSQL dialect differences (Research Pitfall 1). Validate roles at the application layer instead.

7. **backend/alembic.ini** — Standard Alembic config:
   - `script_location = migrations`
   - `sqlalchemy.url` can be placeholder (overridden by env.py from app config)

8. **backend/migrations/env.py** — Async Alembic env per Research code example:
   - Import and use `async_engine_from_config` with `pool.NullPool`
   - Set `render_as_batch=True` in both offline and online modes (CRITICAL for SQLite per Pitfall 3)
   - Import Base metadata and all auth models for autogenerate detection
   - Override sqlalchemy.url from `app.config.settings.database_url`

9. **backend/migrations/script.py.mako** — Standard Alembic template file.

10. **backend/migrations/versions/001_initial_auth_tables.py** — Hand-written initial migration:
    - Create `users` table with all columns and unique constraint on email
    - Create `sessions` table with FK to users and unique constraint on token
    - Create `invitations` table with FK to users
    - Create `instance_config` table
    - Add indexes on sessions.token, users.email, invitations.token, invitations.email
    - Downgrade drops all 4 tables

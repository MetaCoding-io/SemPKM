1. **backend/app/auth/tokens.py** — Token generation and verification using `itsdangerous` per Research code example:
   - `_get_serializer()` function: Lazy init of `URLSafeTimedSerializer` using `settings.secret_key`. If secret_key is empty, read from `settings.secret_key_path` file; if file doesn't exist, generate via `secrets.token_urlsafe(64)`, write to file, use that. This handles the auto-generation per Research Open Question 1.
   - `create_setup_token() -> str`: Generate a `secrets.token_urlsafe(32)` (NOT signed -- just random, stored on disk)
   - `load_or_create_setup_token(path: str) -> str`: Read from file if exists, create and write if not. Delete after setup completes.
   - `create_magic_link_token(email: str) -> str`: Sign email with salt "magic-link" using itsdangerous
   - `verify_magic_link_token(token: str, max_age_seconds: int = 600) -> str | None`: Verify and return email or None
   - `create_invitation_token(email: str, role: str) -> str`: Sign {email, role} with salt "invitation"
   - `verify_invitation_token(token: str, max_age_seconds: int = 604800) -> dict | None`: Verify and return {email, role} or None

2. **backend/app/auth/schemas.py** — Pydantic request/response models:
   - `SetupRequest(BaseModel)`: token (str)
   - `SetupResponse(BaseModel)`: message (str), user_id (str)
   - `MagicLinkRequest(BaseModel)`: email (EmailStr)
   - `MagicLinkResponse(BaseModel)`: message (str)
   - `VerifyTokenRequest(BaseModel)`: token (str)
   - `AuthResponse(BaseModel)`: user_id (str), email (str), role (str), display_name (str | None)
   - `LogoutResponse(BaseModel)`: message (str)
   - `InviteRequest(BaseModel)`: email (EmailStr), role (str) with validation that role is "member" or "guest"
   - `InviteResponse(BaseModel)`: message (str), invitation_id (str)

3. **backend/app/auth/service.py** — Auth business logic:
   - `class AuthService` taking `async_session_factory` as constructor arg
   - `async def get_user_count() -> int`: Count users in database
   - `async def is_setup_complete() -> bool`: Check if any user with role "owner" exists
   - `async def create_owner(email: str) -> User`: Create user with role "owner". Raise if owner already exists.
   - `async def create_user(email: str, role: str) -> User`: Create user with specified role
   - `async def get_user_by_email(email: str) -> User | None`: Lookup by email
   - `async def create_session(user: User) -> UserSession`: Generate `secrets.token_urlsafe(32)` token, compute expires_at from `settings.session_duration_days`, insert into DB, return session
   - `async def verify_session(token: str) -> User | None`: Find session by token where expires_at > now. If session is past 50% of its lifetime, extend it by full duration (sliding window per Research Open Question 5). Return associated user or None.
   - `async def revoke_session(token: str) -> bool`: Delete session row, return True if found
   - `async def revoke_all_sessions(user_id: UUID) -> int`: Delete all sessions for user, return count
   - `async def create_invitation(email: str, role: str, invited_by: UUID) -> Invitation`: Create invitation record with signed token
   - `async def accept_invitation(token: str) -> tuple[User, UserSession] | None`: Verify invitation token, create or get user, create session, mark invitation accepted

1. **backend/app/auth/dependencies.py** — FastAPI dependency chain per Research Pattern 2:
   - `async def get_session_token(sempkm_session: str | None = Cookie(None)) -> str`: Extract session token from cookie. Raise 401 if missing.
   - `async def get_current_user(token: str = Depends(get_session_token), db: AsyncSession = Depends(get_db_session)) -> User`: Look up session in DB (token match + not expired). If past 50% lifetime, extend via sliding window. Return User or raise 401.
   - `def require_role(*roles: str)`: Factory returning a dependency that checks `user.role in roles`, raises 403 if not. Returns the User.
   - `async def optional_current_user(sempkm_session: str | None = Cookie(None), db: AsyncSession = Depends(get_db_session)) -> User | None`: Same as get_current_user but returns None instead of 401. For endpoints that work differently for authenticated vs unauthenticated users (e.g., setup page).

2. **backend/app/auth/router.py** — Auth endpoints:
   - `POST /api/auth/setup`: Accept SetupRequest (token). Verify against app.state.setup_token. If valid, prompt is implicit (no email needed for local -- use a default like "owner@localhost" or accept email from request body; per user decision local owner uses setup token, no email needed initially). Actually, for simplicity: SetupRequest takes token + optional email. If no email, use "owner@local". Create owner user, create session, set httpOnly cookie, delete setup token file, set app.state.setup_mode = False. Return SetupResponse.
   - `POST /api/auth/magic-link`: Accept MagicLinkRequest (email). Generate magic link token. If SMTP configured, send email (placeholder -- actual email sending deferred). Return MagicLinkResponse with generic "check your email" message (never reveal whether email exists).
   - `POST /api/auth/verify`: Accept VerifyTokenRequest (token). Verify magic link token. Get or create user by email. Create session. Set httpOnly cookie. Return AuthResponse.
   - `POST /api/auth/logout`: Requires get_current_user. Revoke session. Clear cookie. Return LogoutResponse.
   - `GET /api/auth/me`: Requires get_current_user. Return AuthResponse with user info.
   - `GET /api/auth/status`: Public endpoint. Return whether setup is complete, whether setup_mode is active.
   - Cookie settings: `httponly=True`, `samesite="lax"`, `secure=False` for development (add config flag later for production), `max_age=settings.session_duration_days * 86400`, key name `"sempkm_session"`.

Alternative: use `Base.metadata.create_all()` with engine for simplicity in development (Alembic for production migrations).      d. Store `async_session_factory` on app.state

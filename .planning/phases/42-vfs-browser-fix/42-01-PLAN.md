  <action>
Add `once` modifier to `hx-trigger="revealed"` in both VFS templates to prevent infinite retry loops when requests fail:

1. **vfs_browser.html line 16:** Change `hx-trigger="revealed"` to `hx-trigger="revealed once"` on the model children div.

2. **_vfs_types.html line 10:** Change `hx-trigger="revealed"` to `hx-trigger="revealed once"` on the type children div.

The `once` modifier ensures the htmx request fires exactly one time per element, regardless of success or failure. If the request fails, "Loading..." stays visible (acceptable) rather than flooding the network with repeated 500 requests.

**1. Create `backend/app/vfs/strategies.py`** -- Strategy enum and SPARQL query builders.

Define an enum `DirectoryStrategy` with values: `FLAT = "flat"`, `BY_TYPE = "by-type"`, `BY_DATE = "by-date"`, `BY_TAG = "by-tag"`, `BY_PROPERTY = "by-property"`.

Helper function `build_scope_filter(mount: MountDefinition) -> str`:
- If mount.sparql_scope == "all": return empty string (no filter)
- If mount.saved_query_id is set: build a sub-select using the saved query's query_text (look up via SPARQL from the SQLite model -- but since we are in sync VFS context, just use the scope as an IRI filter pattern or pass the scope query text as-is). Actually, for simplicity in the sync VFS context, store the actual SPARQL filter as `sparql_scope` when it is inline, or look up the saved query text at mount load time. For the initial implementation: if sparql_scope == "all", no filter. Otherwise, treat sparql_scope as a SPARQL WHERE clause fragment that binds `?iri` (e.g., `?iri a <SomeType>` or a more complex pattern). Wrap it as a sub-select: `{ SELECT ?iri WHERE { {sparql_scope} } }`.
- Use `FROM <urn:sempkm:current>` in all strategy queries.

Helper functions per strategy that return SPARQL query strings:
- `query_flat_objects(scope_filter: str) -> str`: SELECT ?iri ?label with label resolution (COALESCE pattern from existing collections.py)
- `query_type_folders(scope_filter: str) -> str`: SELECT DISTINCT ?typeIri ?typeLabel from shapes graphs + scope filter
- `query_date_folders(date_property: str, scope_filter: str) -> str`: SELECT DISTINCT year/month from the date property
- `query_tag_folders(tag_property: str, scope_filter: str) -> str`: SELECT DISTINCT ?tagValue
- `query_property_folders(group_property: str, scope_filter: str) -> str`: SELECT DISTINCT ?groupValue ?groupLabel with label resolution for IRI values
- `query_objects_in_folder(strategy, folder_value, mount, scope_filter) -> str`: Returns the query to list objects within a specific folder (e.g., objects of a given type, objects with a given tag value, objects in a date range)

**2. Create `backend/app/vfs/mount_collections.py`** -- DAVCollection subclasses for mount directories.

Import `DAVCollection` from wsgidav, `DAVError`/`HTTP_FORBIDDEN`, `SyncTriplestoreClient`, `MountDefinition`, `_slugify` from collections.py, `cached_get_member_names` from cache.py, strategy query helpers.

**`MountRootCollection(DAVCollection)`**: Top-level mount directory (e.g., `/my-notes/`).
- `__init__(self, path, environ, client, mount, event_store=None)`: Store mount definition and client.
- `get_member_names()`: Dispatch to strategy. Use `cached_get_member_names()` with key `mount:{mount.id}:folders`. For flat strategy: return file names directly. For others: return folder names.
- `get_member(name)`: For flat: return MountedResourceFile. For others: return strategy-specific sub-collection or MountedResourceFile if name ends with .md.
- Read-only enforcement methods (create_empty_resource, create_collection, handle_delete, handle_move, handle_copy all raise HTTP_FORBIDDEN).

**`StrategyFolderCollection(DAVCollection)`**: A subfolder within a mount (e.g., `/my-notes/machine-learning/`).
- `__init__(self, path, environ, client, mount, folder_value, event_store=None)`: The folder_value is the tag/type/date/property value that defines this folder.
- `get_member_names()`: Query objects matching this folder's criteria. Build file map (slug -> object info). Cache with key `mount:{mount.id}:{folder_value}:files`. Return filenames list.
- `get_member(name)`: Return MountedResourceFile for the filename.
- Internal `_build_file_map()` that maps filename -> {iri, label, type_iri} (same dedup pattern as TypeCollection).
- `get_object_by_filename(filename)` for resource lookup.
- Read-only enforcement methods.

Add a special `_UNCATEGORIZED = "_uncategorized"` constant. When listing folder names, also check for objects missing the grouping property and add `_uncategorized` if any exist. In `StrategyFolderCollection`, when folder_value is `_uncategorized`, query objects where the grouping property is NOT bound (FILTER NOT EXISTS).

For **by-date strategy**: folder names are formatted as `YYYY/MM-MonthName` (e.g., `2024/01-January`). This means by-date has TWO levels of folders: year folders at mount root, month folders within year. Implement this as: `MountRootCollection` returns year strings, `StrategyFolderCollection` at year level returns month strings, and a second `StrategyFolderCollection` at month level returns files.

For **multi-valued grouping properties** (per CONTEXT.md): objects with multiple tag values appear in each tag folder. The file map builder for a folder should query `?iri <tag_prop> <folder_value>` -- an object with tags [A, B] will match both folder A and folder B queries naturally.

**1. Create `backend/app/vfs/mount_resource.py`** -- MountedResourceFile class.

This is a `DAVNonCollection` subclass that renders objects as Markdown+frontmatter using SHACL shape metadata, and writes frontmatter property changes back via `object.patch`.

**SHACL shape cache** (module-level):
- `_shape_cache: TTLCache = TTLCache(maxsize=32, ttl=300)` -- Cache PropertyShape data keyed by type_iri. 5-minute TTL since shapes change only on model install.
- `_shape_cache_lock = threading.Lock()`
- Helper `_get_shapes_for_type(type_iri: str, client: SyncTriplestoreClient) -> list[PropertyShape]`:
  - Check _shape_cache first.
  - On miss: query the shapes graph to get property shapes for the type. Use SPARQL CONSTRUCT or SELECT against all `urn:sempkm:model:*:shapes` graphs to find the NodeShape targeting this type, then extract its property shapes (sh:path, sh:name, sh:datatype, sh:class/sh:node for target_class).
  - Parse into `PropertyShape` dataclass instances (import from `app.services.shapes`).
  - Cache and return.

**MountedResourceFile(DAVNonCollection)** class:
- `__init__(self, path, environ, client, object_iri, object_label, type_iri, event_store=None)`: Unlike the existing ResourceFile which looks up object info from parent TypeCollection, this receives IRI/label/type directly from the mount collection that instantiated it.
- `_render_shacl_frontmatter() -> tuple[dict, str]`: Returns (frontmatter_dict, body_text).
  - Query all triples for the object: `SELECT ?p ?o FROM <urn:sempkm:current> WHERE { <{iri}> ?p ?o }`
  - Get SHACL shapes for the type via `_get_shapes_for_type()`.
  - System keys first: `object_iri`, `type_iri`.
  - For each PropertyShape in order:
    - Find matching triples by predicate path.
    - Use `sh:name` as the frontmatter key (human-readable).
    - If `target_class` is set (object reference): format as `{label: ..., iri: ...}` dict per CONTEXT.md. For multi-valued: list of dicts. Look up labels via SPARQL or use QName fallback.
    - If single value: use string. If multiple values: use list.
    - If predicate local name is "body": extract as body_text, skip in frontmatter.
  - For properties NOT in SHACL shapes: skip (SHACL shapes define the writable boundary).
  - Return (fm_dict, body_text).
- `_render() -> bytes`: Call `_render_shacl_frontmatter()`, build `frontmatter.Post(content=body, **fm)`, return `frontmatter.dumps(post).encode("utf-8")`. Cache result.
- `get_content_length()`, `get_content_type()`, `get_content()`: Same pattern as ResourceFile.
- `get_etag() -> str`: Return `hashlib.sha256(self._object_iri.encode()).hexdigest()` -- per CONTEXT.md, ETag is derived from object IRI (not content), so all paths to the same object share the same ETag.
- `support_etag()`, `support_content_length()`: Return True.
- `begin_write(*, content_type=None) -> io.BytesIO`: Same buffer pattern as ResourceFile.
- `end_write(*, with_errors: bool)`:
  - Parse new content via `frontmatter.loads()`.
  - Get current frontmatter via `_render_shacl_frontmatter()`.
  - Get SHACL shapes via `_get_shapes_for_type()`.
  - Diff: for each PropertyShape, compare old vs new frontmatter values.
  - If body changed: call `write_body_via_event_store()` (existing function).
  - If properties changed: call `write_properties_via_event_store()` (new function from write.py).
  - Invalidate cached content.
- `handle_delete`, `handle_move`, `handle_copy`: Raise HTTP_FORBIDDEN.

**Reserved frontmatter key handling**: Keys `object_iri` and `type_iri` are system keys. If a SHACL property's sh:name collides with these, prefix with underscore (e.g., `_object_iri`). Check on render and write-back.

**2. Extend `backend/app/vfs/write.py`** -- Add property write-back function.

Add `write_properties_via_event_store()`:
```python
async def _commit_property_patch(object_iri, properties, event_store, user_iri_str, user_role):
    from rdflib import URIRef
    from app.commands.handlers.object_patch import handle_object_patch
    from app.commands.schemas import ObjectPatchParams
    from app.config import settings

    params = ObjectPatchParams(iri=object_iri, properties=properties)
    operation = await handle_object_patch(params, settings.base_namespace)
    user_iri = URIRef(user_iri_str) if user_iri_str else None
    await event_store.commit([operation], performed_by=user_iri, performed_by_role=user_role)

def write_properties_via_event_store(object_iri, properties, event_store, user_iri=None, user_role="member"):
    """Bridge sync wsgidav context to async object.patch commit."""
    asyncio.run(_commit_property_patch(object_iri, properties, event_store, user_iri, user_role))
```

The `properties` dict maps predicate IRI -> new value. Values can be:
- String literal -> pass as-is
- `{label: str, iri: str}` dict -> extract IRI and pass as URIRef string
- List of strings/dicts -> convert each element

Add a helper `_frontmatter_to_rdf_properties(new_fm: dict, old_fm: dict, shapes: list[PropertyShape]) -> dict[str, Any]`:
- For each shape, map the sh:name key back to predicate IRI.
- Compare old vs new values. Only include changed predicates.
- Convert `{label, iri}` reference dicts back to IRI strings.
- Skip `object_iri` and `type_iri` system keys.
- Return dict suitable for ObjectPatchParams.properties.

  <name>Task 2: Add mount form JavaScript for dynamic fields, preview, and CRUD</name>
  <files>frontend/static/js/workspace.js</files>
  <action>
Add mount management JavaScript functions to `frontend/static/js/workspace.js`. Place in a clearly commented section near other settings-related functions. These functions handle: strategy field toggling, property dropdown population, scope dropdown population, live preview, mount CRUD, and edit/delete operations.

**Initialization function** `initMountForm()`:
- Called when the settings/VFS tab is loaded (detect via existing settings page load pattern or call on DOMContentLoaded if settings panel is present).
- Fetch `/api/vfs/mounts/properties` and populate the "Group By Property" and "Date Field" dropdowns with options `<option value="{iri}">{name} ({types})</option>`.
- Fetch `/api/sparql/queries` and populate the "Scope" dropdown: keep "All objects" as default, add saved queries as `<option value="{id}">{name}</option>`, add a "Custom SPARQL..." option at the end with value "custom".
- Fetch `/api/vfs/mounts` and render the active mounts list via `renderMountList(mounts)`.

**Dynamic strategy fields** `mountStrategyChanged(strategy)`:
- Hide all `.mount-field-by-tag`, `.mount-field-by-property`, `.mount-field-by-date` elements.
- Show the relevant ones based on strategy:
  - "by-tag": show `.mount-field-by-tag`
  - "by-property": show `.mount-field-by-property`
  - "by-date": show `.mount-field-by-date`
  - "flat", "by-type": show nothing extra

**Form submission** `mountSubmitForm(event)`:
- Prevent default.
- Collect form values: name, path, strategy, group_by_property (if visible), date_property (if visible), scope, visibility.
- Check if editing (mount-edit-id has a value): if so, PUT to `/api/vfs/mounts/{id}`, else POST to `/api/vfs/mounts`.
- On success: clear form, reload mount list via `loadMountList()`, show brief success indicator.
- On 400 error: show error message (path conflict, validation error) near the form.
- Reset edit mode after submission.

**Preview** `mountPreview()`:
- Collect current form values.
- POST to `/api/vfs/mounts/preview` with the form data as JSON.
- Render the returned directory tree in `#mount-preview-tree`:
  - Show `#mount-preview`.
  - For each directory: render as `<div class="mount-preview-folder">{name} <span class="mount-preview-count">({file_count} files)</span></div>`.
  - For nested directories (by-date year/month): indent with padding.
- On error: show error message in preview area.

**Mount list rendering** `renderMountList(mounts)`:
- Clear `#mount-list-items`.
- If no mounts: show `<p class="mount-list-empty">No custom mounts yet.</p>`.
- For each mount: render a `.mount-list-item` div:
  ```html
  <div class="mount-list-item" id="mount-item-{id}">
    <div class="mount-list-item-info">
      <span class="mount-list-item-name">{name}</span>
      <span class="mount-list-item-meta">/dav/{path}/ &middot; {strategy} &middot; {visibility}</span>
    </div>
    <div class="mount-list-item-actions">
      <button class="btn-secondary-sm" onclick="mountEdit('{id}')">Edit</button>
      <button class="btn-danger-sm" onclick="mountDelete('{id}', '{name}')">Delete</button>
    </div>
  </div>
  ```

**Load mount list** `loadMountList()`:
- Fetch `/api/vfs/mounts` and call `renderMountList()`.

**Edit mount** `mountEdit(id)`:
- Fetch `/api/vfs/mounts` (or use cached list), find the mount by id.
- Populate form fields with mount values.
- Set `#mount-edit-id` value to the mount id.
- Change submit button text to "Update Mount".
- Show cancel button.

**Cancel edit** `mountCancelEdit()`:
- Clear form fields and hidden id.
- Reset submit button text to "Create Mount".
- Hide cancel button.

**Delete mount** `mountDelete(id, name)`:
- Confirm: `Are you sure you want to delete the mount "${name}"? The directory will no longer appear in WebDAV.`
- On confirm: DELETE `/api/vfs/mounts/{id}`.
- On success: remove `#mount-item-{id}` from DOM or reload list.

**Custom SPARQL scope**: When scope dropdown value is "custom", show a textarea below the dropdown for inline SPARQL entry. The textarea value becomes the sparql_scope field. Hide the textarea for other scope values.

**Auto-populate path from name**: When the name input loses focus and the path input is empty, auto-generate a slug from the name (lowercase, replace spaces/special chars with hyphens, strip leading/trailing hyphens).

**Error handling**: All fetch calls should handle non-OK responses gracefully. Show error messages inline near the form (not alerts). Use a `<div id="mount-form-error" class="mount-form-error"></div>` element.

**Init trigger**: Add mount form initialization to the settings page load path. Look for the existing pattern where VFS settings are loaded (likely via htmx or a tab activation handler). If the mount section element `#vfs-mount-section` exists when the page loads, call `initMountForm()`. Also call it after any htmx swap that loads the VFS settings partial.

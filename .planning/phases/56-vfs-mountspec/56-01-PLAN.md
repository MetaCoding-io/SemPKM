@prefix sempkm: <urn:sempkm:> .

mount:{id} a sempkm:MountSpec ;
    sempkm:mountName "My Notes" ;
    sempkm:mountPath "my-notes" ;
    sempkm:directoryStrategy "by-tag" ;  # by-type|by-date|by-tag|by-property|flat
    sempkm:groupByProperty <predicate-iri> ;  # for by-tag, by-property
    sempkm:dateProperty <predicate-iri> ;     # for by-date

    sempkm:createdBy <urn:sempkm:user:uuid> ;

    sempkm:createdAt "iso-datetime"^^xsd:dateTime .

  <action>
Create `backend/app/vfs/mount_service.py` with a `SyncMountService` class that performs CRUD on mount definitions stored as RDF in `urn:sempkm:mounts` named graph.

**RDF vocabulary constants** (module-level):
- `NS_SEMPKM = "urn:sempkm:"` (prefix for vocabulary terms)
- `NS_MOUNT = "urn:sempkm:mount:"` (prefix for mount instances)
- `GRAPH_MOUNTS = "urn:sempkm:mounts"` (named graph)
- Predicate constants: `MOUNT_NAME`, `MOUNT_PATH`, `DIRECTORY_STRATEGY`, `GROUP_BY_PROPERTY`, `DATE_PROPERTY`, `SPARQL_SCOPE`, `SAVED_QUERY_ID`, `CREATED_BY`, `VISIBILITY`, `CREATED_AT` -- all as `urn:sempkm:{camelCase}` strings

**Data class** `MountDefinition`:
- Fields: `id` (str, UUID), `name` (str), `path` (str, URL-safe prefix), `strategy` (str, one of flat/by-type/by-date/by-tag/by-property), `group_by_property` (str|None), `date_property` (str|None), `sparql_scope` (str, default "all"), `saved_query_id` (str|None), `created_by` (str, user URN), `visibility` (str, "shared"|"personal"), `created_at` (str, ISO datetime)

**SyncMountService class** (takes `SyncTriplestoreClient`):
- `list_mounts(user_iri: str) -> list[MountDefinition]`: SELECT all mounts from `urn:sempkm:mounts` where visibility="shared" OR createdBy=user_iri. Parse bindings into MountDefinition list.
- `get_mount_by_id(mount_id: str) -> MountDefinition | None`: SELECT specific mount by IRI.
- `get_mount_by_prefix(prefix: str) -> MountDefinition | None`: SELECT mount where mountPath=prefix. Used by provider dispatch.
- `create_mount(mount: MountDefinition) -> MountDefinition`: INSERT DATA into urn:sempkm:mounts graph. Before insert, validate: (1) path matches `[a-z0-9][a-z0-9-]*` regex, (2) path does not conflict with installed model IDs (query urn:sempkm:models for modelId values), (3) path not already taken by another mount. Raise ValueError on validation failure. Generate UUID for id, set created_at to now.
- `update_mount(mount_id: str, updates: dict) -> MountDefinition`: DELETE old triples for the mount, INSERT new triples. Only update fields present in `updates` dict.
- `delete_mount(mount_id: str) -> bool`: DELETE all triples for the mount IRI from urn:sempkm:mounts.

**Validation helper** `_validate_mount_path(path: str, client: SyncTriplestoreClient, exclude_id: str | None = None)`:
- Regex check: `^[a-z0-9][a-z0-9-]*$`
- Reserved names check: reject "_uncategorized", names starting with "."
- Model conflict check: query urn:sempkm:models for all modelId values, reject if path matches any
- Uniqueness check: query urn:sempkm:mounts for existing mount with same mountPath (excluding exclude_id for updates)

**Cache invalidation**: In `backend/app/vfs/cache.py`, add a `clear_mount_cache()` function that removes all cache keys starting with "mount:" from the listing_cache. Use the existing `_cache_lock`. Call this from create/update/delete methods.

Also add `clear_all_mount_keys()` to cache.py that iterates listing_cache keys and removes any starting with "mount:" or "root:" (root because mount directories appear in root listing).

<task type="auto">
  <name>Task 2: Create mount CRUD and preview REST API endpoints</name>
  <files>backend/app/vfs/mount_router.py, backend/app/main.py</files>
  <action>
Create `backend/app/vfs/mount_router.py` with a FastAPI APIRouter providing mount management REST endpoints. These endpoints use the async `TriplestoreClient` (not sync) since they run in FastAPI routes.

**Important**: The mount_router endpoints need an async-compatible way to call mount service operations. Create a thin `AsyncMountService` wrapper that uses the async `TriplestoreClient` -- same SPARQL queries as `SyncMountService` but with `await client.query()` / `await client.update()`. Alternatively, make the SPARQL strings shared between sync and async by having helper functions that return query strings, with both service classes using them. Choose the simplest approach (likely duplicate the small SPARQL queries in async variants since the queries are short).

**Endpoints:**
1. `GET /api/vfs/mounts` -- List mounts visible to current user. Returns JSON array of mount objects. Query params: none. Auth: `get_current_user`. Filter: shared mounts + personal mounts where createdBy matches user.
2. `POST /api/vfs/mounts` -- Create a new mount. JSON body: `{name, path, strategy, group_by_property?, date_property?, sparql_scope?, saved_query_id?, visibility?}`. Default visibility="personal". Only owners can set visibility="shared" (check `user.role`). Returns 201 with created mount. On validation error: return 400 with error message.
3. `PUT /api/vfs/mounts/{mount_id}` -- Update a mount. JSON body: partial update fields. Only mount creator or owner can update. Returns 200 with updated mount.
4. `DELETE /api/vfs/mounts/{mount_id}` -- Delete a mount. Only mount creator or owner can delete. Returns 204.
5. `POST /api/vfs/mounts/preview` -- Preview directory structure for a given mount config (dry-run). JSON body: same as create. Returns JSON `{directories: [{name, children: [{name, file_count}]}]}`. Cap at first 100 objects per folder. This requires querying the triplestore using the strategy's SPARQL pattern (by-type: group by rdf:type, by-tag: group by groupByProperty values, by-date: group by year/month, by-property: group by property values, flat: just count).
6. `GET /api/vfs/mounts/properties` -- List available properties from SHACL shapes for strategy field dropdowns. Returns JSON `{properties: [{iri, name, types: [type_label]}]}` extracted from all shapes graphs. This endpoint supports the settings UI dropdowns for "Tag property", "Group by", "Date field".

**Register the router**: In `backend/app/main.py`, import and include the mount_router. Find the existing line where vfs router is included (should be `app.include_router(vfs_router)` or similar) and add `app.include_router(mount_router)` alongside it.

**Role check for shared mounts**: Use the existing user.role field. If user.role != "owner" and visibility == "shared", return 403.

**Cache invalidation**: After create/update/delete succeed, call `clear_mount_cache()` from cache.py to invalidate the sync VFS cache (so WebDAV picks up changes without waiting for TTL expiry).

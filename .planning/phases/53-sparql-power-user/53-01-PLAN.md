  <name>Task 2: Add CRUD endpoints for history, saved queries, vocabulary, and enriched results</name>
  <files>backend/app/sparql/router.py</files>
  <action>
Extend `backend/app/sparql/router.py` with these new endpoints. Add required imports: `get_db_session`, `AsyncSession`, the new models and schemas, `LabelService`, `IconService`, `PrefixRegistry`, and `settings` from `app.config`.

**History endpoints:**

1. `GET /api/sparql/history` — List user's query history (most recent first, limit 100).
   - Depends: get_current_user, get_db_session
   - Call `_enforce_sparql_role(user, "", False)` to reject guests
   - Query: `select(SparqlQueryHistory).where(user_id == user.id).order_by(executed_at.desc()).limit(100)`
   - Return: list of QueryHistoryOut

2. `DELETE /api/sparql/history` — Clear user's history.
   - Depends: get_current_user, get_db_session
   - Delete all rows where user_id == user.id

**Modify existing `sparql_post` endpoint** to auto-save history after successful execution:
   - After `_execute_sparql` returns a successful result (status 200), save the query to history
   - Deduplicate: if the most recent history entry for this user has the same stripped query_text, update its `executed_at` timestamp instead of inserting
   - Prune: after insert, if count > 100 for this user, delete oldest entries beyond 100
   - This requires adding `db: AsyncSession = Depends(get_db_session)` to `sparql_post`

**Saved query endpoints:**

3. `GET /api/sparql/saved` — List user's saved queries (newest first).
   - Depends: get_current_user, get_db_session
   - Return: list of SavedQueryOut

4. `POST /api/sparql/saved` — Create a saved query.
   - Body: SavedQueryCreate
   - Depends: get_current_user, get_db_session
   - Return: SavedQueryOut (201)

5. `PUT /api/sparql/saved/{query_id}` — Update a saved query.
   - Body: SavedQueryUpdate
   - Depends: get_current_user, get_db_session
   - Verify ownership (user_id matches), return 404 if not found or not owned
   - Return: SavedQueryOut

6. `DELETE /api/sparql/saved/{query_id}` — Delete a saved query.
   - Depends: get_current_user, get_db_session
   - Verify ownership, return 404 if not found
   - Return: 204

**Vocabulary endpoint:**

7. `GET /api/sparql/vocabulary` — Return classes, predicates, and prefixes from installed Mental Model ontology graphs.
   - Depends: get_current_user, get_triplestore_client, get_prefix_registry
   - Call `_enforce_sparql_role(user, "", False)` to reject guests
   - SPARQL query against ontology graphs (graphs matching `urn:sempkm:model:*:ontology`): SELECT DISTINCT ?entity ?type ?label WHERE { GRAPH ?g { ?entity a ?type . FILTER(?type IN (owl:Class, owl:ObjectProperty, owl:DatatypeProperty, owl:AnnotationProperty)) . OPTIONAL { ?entity rdfs:label ?label } } FILTER(STRSTARTS(STR(?g), "urn:sempkm:model:") && STRENDS(STR(?g), ":ontology")) }
   - For each entity: compact IRI via `prefix_registry.compact()`, determine badge ("C"/"P"/"D"), include label
   - Return prefixes from `prefix_registry.get_all_prefixes()` and the items list
   - Include a `model_version` counter (use a simple approach: hash of sorted model graph URIs, or just return current timestamp as a cache-buster — the frontend will cache until it changes)

**Result enrichment helper:**

Add `_enrich_sparql_results(raw_results: dict, label_service, icon_service, prefix_registry, base_namespace: str) -> dict`:
   - Scan all bindings for URI values
   - Detect object IRIs: URIs starting with `base_namespace` are objects. URIs starting with known vocabulary prefixes (rdf, rdfs, owl, dcterms, skos, schema, foaf, xsd, sh, dash, urn:sempkm:) are NOT objects.
   - Collect unique object IRIs, call `label_service.resolve_batch(list(object_iris))`
   - Run a single SPARQL VALUES query to get `rdf:type` for all object IRIs
   - For each object IRI, call `icon_service.get_type_icon(type_iri, "tab")` for the icon
   - Compact each object IRI via `prefix_registry.compact(iri)`
   - Attach `_enrichment` dict to the response: `{iri: {label, type_iri, icon, qname}}`

Modify `sparql_post` to call `_enrich_sparql_results` on successful results before returning. Add `label_service`, `prefix_registry` as dependencies. Create `get_icon_service` locally following the pattern from `browser/router.py` (or import it). Import `settings` from `app.config` for `base_namespace`.

Note: The `sparql_get` endpoint can remain unenriched (it is primarily for SPARQL Protocol compliance, not the UI).

API Endpoints (created in Plan 01):
```
POST /api/sparql         — Execute SPARQL, returns enriched results with _enrichment dict
  Request: {"query": str, "all_graphs": bool}
  Response: SPARQL Results JSON + {"_enrichment": {iri: {label, type_iri, icon: {icon, color}, qname}}}

GET  /api/sparql/history — List recent queries (100 max, newest first)
  Response: [{id, query_text, executed_at}, ...]

DELETE /api/sparql/history — Clear user's history

GET  /api/sparql/saved   — List saved queries (newest first)
  Response: [{id, name, description, query_text, created_at, updated_at}, ...]

POST /api/sparql/saved   — Save a query
  Request: {"name": str, "description": str|null, "query_text": str}
  Response: {id, name, description, query_text, created_at, updated_at}

PUT  /api/sparql/saved/{query_id} — Update saved query
DELETE /api/sparql/saved/{query_id} — Delete saved query

GET  /api/sparql/vocabulary — Ontology autocomplete data
  Response: {"prefixes": {prefix: namespace}, "items": [{qname, full_iri, badge, label}], "model_version": int}
```

- Parse SPARQL Results JSON: `head.vars` for column headers, `results.bindings` for rows.

- Click handler: `openTab(iri, label)` — the function is available on `window` from workspace.js
- After rendering pills, call `lucide.createIcons()` to initialize the Lucide SVGs in the new elements.

**1. Add ontology autocomplete to `sparql-console.js`:**

In `initSparqlConsole()`, after the CM6 editor is created:

- Fetch `GET /api/sparql/vocabulary` once, store in module-level `vocabCache` (items array) and `prefixCache` (prefix map) and `cachedModelVersion` (version number).
- Create a custom CM6 completion source function `sparqlCompletions(context)`:
  - Match the current word with `context.matchBefore(/[\w:?]*/)`
  - If no match or empty and not explicit, return null
  - Build options array from multiple sources:
    a. **SPARQL keywords**: SELECT, WHERE, FILTER, OPTIONAL, UNION, BIND, GROUP BY, ORDER BY, LIMIT, OFFSET, HAVING, VALUES, ASK, CONSTRUCT, DESCRIBE, PREFIX, BASE, DISTINCT, REDUCED, AS, FROM, NAMED, NOT EXISTS, EXISTS, MINUS, SERVICE, IN, NOT IN, a (for rdf:type shorthand), GRAPH, INSERT, DELETE (even though writes are blocked, autocomplete is helpful for learning). Each with `detail: "K"` badge and `type: "keyword"`.
    b. **Prefixed names** (when text contains a colon): split on colon, filter vocabCache items matching the prefix, offer completions with their qname. Badge is the item's badge field ("C", "P", "D").
    c. **PREFIX declarations** (when text starts with P/p): for each prefix in prefixCache, offer `PREFIX {prefix}: <{namespace}>` with `apply` that inserts the full declaration + newline. Badge "D".
    d. **Variable names** (when text starts with ?): scan the current document for all `?\w+` tokens, deduplicate, offer as completions with `type: "variable"` and `detail: "V"`.
  - Return `{ from: word.from, options, validFor: /^[\w:?]*$/ }`

- Register the completion source via the `autocompletion({ override: [sparqlCompletions] })` extension in the EditorState. This should be set up during editor creation (update the placeholder from Task 1).
- Set `activateOnTyping: true` in the autocompletion config for always-on (aggressive) behavior per user decision.

- **Type badges in dropdown**: CM6 autocomplete renders `detail` as a badge. The detail field on each option ("K", "C", "P", "D", "V") will display as a small badge. Optionally add CSS to style `.cm-completionDetail` with a monospace font, slightly colored background, and border-radius for a badge appearance.

- **Cache invalidation**: On each initSparqlConsole call (which happens on tab switch if not init), check if model_version changed by re-fetching the vocabulary endpoint. If changed, update vocabCache. For simplicity, re-fetch vocabulary every time the SPARQL tab is activated (it's a fast query and happens infrequently).

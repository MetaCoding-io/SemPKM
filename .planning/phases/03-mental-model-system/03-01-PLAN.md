Add `"pyyaml>=6.0"` to the dependencies list in `backend/pyproject.toml`.

Create `backend/app/models/__init__.py` as an empty package init.

Create `backend/app/models/manifest.py`:
- Import yaml, pathlib.Path, and pydantic (BaseModel, Field, field_validator, model_validator).
- Define `ManifestEntrypoints(BaseModel)` with fields: `ontology: str`, `shapes: str`, `views: str`, `seed: str | None`. Defaults use `{modelId}` placeholder pattern (e.g., `"ontology/{modelId}.jsonld"`).
- Define `ManifestSchema(BaseModel)` with fields:
  - `modelId: str` — Field with pattern `r'^[a-z][a-z0-9-]*$'`, min_length=2, max_length=64
  - `version: str` — Field with pattern `r'^\d+\.\d+\.\d+$'`
  - `name: str` — Field with min_length=1, max_length=200
  - `description: str` — Field with default="" and max_length=2000
  - `namespace: str` — validated to equal `f"urn:sempkm:model:{modelId}:"`
  - `prefixes: dict[str, str]` — default_factory=dict, model-specific prefix mappings
  - `entrypoints: ManifestEntrypoints` — default_factory=ManifestEntrypoints
- Add a `model_validator(mode="after")` that resolves `{modelId}` placeholders in entrypoints using the actual modelId value.
- Add a `field_validator("namespace")` that checks the value matches `f"urn:sempkm:model:{modelId}:"`.
- Define `parse_manifest(model_dir: Path) -> ManifestSchema` that reads `manifest.yaml` via `yaml.safe_load()` and validates with `ManifestSchema.model_validate()`. Raise ValueError with context if file is missing or parsing fails.

Create `backend/app/models/loader.py`:
- Import pathlib.Path, rdflib.Graph, and the ManifestSchema type.
- Define `load_jsonld_file(file_path: Path) -> Graph` that calls `Graph().parse(str(file_path), format="json-ld")` and returns the graph. Raise FileNotFoundError if file_path doesn't exist.
- Define `ModelArchive` dataclass holding: `manifest: ManifestSchema`, `ontology: Graph`, `shapes: Graph`, `views: Graph`, `seed: Graph | None`.
- Define `load_archive(model_dir: Path, manifest: ManifestSchema) -> ModelArchive` that resolves entrypoint paths relative to model_dir, loads each JSON-LD file into a Graph via `load_jsonld_file`, and returns a `ModelArchive`. If seed entrypoint is None or file doesn't exist, set seed to None.
- Use inline @context only validation: check that no loaded JSON-LD has remote @context URLs (per Research Pitfall 1). Read the raw JSON, check for any @context value that starts with "http://" or "https://", and raise a ValueError if found. This prevents Docker environment failures from remote context fetches.

Create `backend/app/models/validator.py`:
- Import rdflib (Graph, URIRef, BNode), rdflib.namespace (RDF, SH), and typing.
- Define `@dataclass class ValidationIssue` with fields: `file: str`, `subject: str`, `rule: str`, `message: str`, `severity: str = "error"` (either "error" or "warning").
- Define `@dataclass class ArchiveValidationReport` with fields: `errors: list[ValidationIssue]`, `warnings: list[ValidationIssue]`. Add property `is_valid` returning `len(self.errors) == 0`.
- Define `ALLOWED_EXTERNAL_NAMESPACES` as a set of strings: `{"http://www.w3.org/", "http://purl.org/dc/", "https://schema.org/", "http://xmlns.com/foaf/"}`.
- Define `validate_iri_namespacing(graph: Graph, model_namespace: str, file_name: str) -> list[ValidationIssue]`: iterate all unique subjects in the graph, skip BNodes, check that each subject IRI either starts with `model_namespace` or one of `ALLOWED_EXTERNAL_NAMESPACES`. Return a list of ValidationIssue (severity="error") for violations.
- Define `validate_reference_integrity(ontology: Graph, shapes: Graph, views: Graph, seed: Graph | None, model_namespace: str) -> list[ValidationIssue]`: (1) Collect all OWL classes from ontology where subject has `rdf:type owl:Class` and subject starts with model_namespace. (2) Check shapes: for each `sh:NodeShape` with `sh:targetClass`, if the target class starts with model_namespace, verify it's in the ontology classes set. (3) Check seed data: for each triple `(?s, rdf:type, ?o)` where `?o` starts with model_namespace, verify it's in the ontology classes set. (4) Check views: for each view with `sempkm:targetClass`, if it starts with model_namespace, verify it's in the ontology classes set. Return list of ValidationIssue for all failures.
- Define `validate_archive(archive: ModelArchive) -> ArchiveValidationReport` that runs both validators on all graphs in the archive, collects all issues, separates into errors and warnings, and returns an ArchiveValidationReport. For warnings: add a warning (not error) if ontology classes have no rdfs:label.

Create `backend/app/models/registry.py`:
- Import dataclasses, datetime, rdflib (URIRef, Literal, Graph), rdflib.namespace (RDF, XSD), typing, and TriplestoreClient.
- Define `MODELS_GRAPH = "urn:sempkm:models"` constant.
- Define `SEMPKM_NS = "urn:sempkm:"` constant for the sempkm vocabulary namespace.
- Define `@dataclass class ModelGraphs` with a `model_id: str` field and properties for each named graph IRI: `ontology` -> `f"urn:sempkm:model:{model_id}:ontology"`, `shapes` -> `f"urn:sempkm:model:{model_id}:shapes"`, `views` -> `f"urn:sempkm:model:{model_id}:views"`, `seed` -> `f"urn:sempkm:model:{model_id}:seed"`, and `all_graphs` -> list of all four.
- Define `@dataclass class InstalledModel` with fields: `model_id: str`, `version: str`, `name: str`, `description: str`, `namespace: str`, `installed_at: str`.
- Define async function `register_model(client: TriplestoreClient, manifest: ManifestSchema, installed_at: str) -> None`:
  Build SPARQL INSERT DATA that writes model metadata triples into `GRAPH <urn:sempkm:models>`. Triples: `<urn:sempkm:model:{modelId}> a <urn:sempkm:MentalModel>`, plus `sempkm:modelId`, `sempkm:version`, `dcterms:title`, `dcterms:description`, `sempkm:namespace`, `sempkm:installedAt`. Use the `_rdf_term_to_sparql` pattern from `app/services/validation.py` for proper literal escaping (copy or import the helper).
- Define async function `unregister_model(client: TriplestoreClient, model_id: str) -> None`:
  Execute `DELETE WHERE { GRAPH <urn:sempkm:models> { <urn:sempkm:model:{model_id}> ?p ?o . } }`.
- Define async function `list_models(client: TriplestoreClient) -> list[InstalledModel]`:
  Execute SELECT query against `GRAPH <urn:sempkm:models>` for all installed models with their metadata. Return list of InstalledModel dataclasses.
- Define async function `is_model_installed(client: TriplestoreClient, model_id: str) -> bool`:
  Execute ASK query checking if `<urn:sempkm:model:{model_id}>` exists in the models graph.
- Define async function `write_graph_to_named_graph(client: TriplestoreClient, graph: Graph, named_graph_iri: str) -> None`:
  Serialize the rdflib Graph triple-by-triple using `_rdf_term_to_sparql` (not N-Triples format, per Research Pitfall 2). Build SPARQL INSERT DATA with GRAPH clause. Skip if graph is empty.
- Define async function `clear_model_graphs(client: TriplestoreClient, model_id: str) -> None`:
  Execute `CLEAR SILENT GRAPH` for each of ModelGraphs(model_id).all_graphs.
- Define async function `check_user_data_exists(client: TriplestoreClient, model_namespace: str, ontology: Graph) -> list[str]`:
  For each OWL class in the ontology that starts with model_namespace, execute ASK query against `GRAPH <urn:sempkm:current>` checking if any instances exist. Return list of class IRIs that have user data.
  </action>

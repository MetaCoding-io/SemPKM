**Helptext validation bug fix** (workspace.js ~line 1547):
In the global `focusout` event listener that handles required-field validation, add a check for `e.relatedTarget`. If focus is moving to another element within the same `.form-field` container (e.g., the helptext toggle button), skip the validation check. The fix:
```javascript
var field = input.closest('.form-field');
if (e.relatedTarget && field && field.contains(e.relatedTarget)) return;
```
This prevents false "required" errors when users click the helptext expand button on a filled field.

**Keyboard shortcuts reliability** (workspace.js ~line 735):
Investigate root cause by examining `_keydownHandler` and `initKeyboardShortcuts()`. The likely issue is one of:
1. htmx body swaps removing the listener — if `initKeyboardShortcuts()` is called on htmx:afterSwap or similar, and the old handler reference is lost
2. Dockview/CodeMirror capturing events before they bubble to document
3. The command palette (ninja-keys) consuming events

Fix approach:
- Ensure the keydown listener uses `{ capture: true }` so it fires before child elements can `stopPropagation()`. This is the most robust fix for focus-stealing by dockview panels or embedded editors.
- Verify `initKeyboardShortcuts()` properly removes old handler before adding new one (guard against duplicate listeners).
- If the handler is removed by htmx swaps, re-register on `htmx:afterSettle` for body-level swaps.

**Event console form visibility** (event_console.html):
The inline `<script>` calling `switchCommandForm(...)` at the bottom of the template likely runs before `app.js` is loaded. Fix by wrapping the call in a `DOMContentLoaded` listener or, if the page has already loaded, call immediately:
```javascript
if (document.readyState === 'loading') {
  document.addEventListener('DOMContentLoaded', function() {
    switchCommandForm(document.getElementById('command-type').value);
  });
} else {
  switchCommandForm(document.getElementById('command-type').value);
}
```
Alternatively, check if `app.js` is loaded with `defer` and the inline script runs before it — if so, add `defer` to the inline script or move the init call to app.js itself.

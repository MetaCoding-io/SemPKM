   - Open `SEED.notes.architecture.iri` (Note type) via `openTab()`
   - `waitForIdle()`
   - Read `--tab-accent-color` from `.dv-active-group` via `page.evaluate(() => getComputedStyle(document.querySelector('.dv-active-group')).getPropertyValue('--tab-accent-color').trim())`
   - Open `SEED.people.alice.iri` (Person type) via `openTab()`
   - `waitForIdle()`
   - Read `--tab-accent-color` again from `.dv-active-group`
   - Assert the two colors are DIFFERENT (not equal)
   - Also assert both are non-empty strings

2. **"BUG-05: card view borders render"** — Open a card view for any type. Use `openViewTab()` from dockview helpers or navigate to a view. Locate `.card-grid` and `[data-testid="card-item"]` elements. For at least one card, verify it has a visible border by checking `border` or `border-color` CSS property is not "none" and not transparent. Use `toHaveCSS()` or `page.evaluate()` to read computed border style.

3. **"BUG-06: Ctrl+K opens command palette"** — Navigate to workspace. Press `Control+k` via `ownerPage.keyboard.press('Control+k')`. Wait for `ninja-keys` element to become visible (it's a web component). Verify `ninja-keys` is visible on the page. Then press Escape to close it. Verify it's hidden again.

4. **"BUG-07: inactive tabs do not show accent bleed"** — Open two objects so there are multiple tabs. The inactive tab should NOT have the accent color applied. Check that only `.dv-active-group .dv-tab.dv-active-tab` has the accent border, and that `.dv-tab:not(.dv-active-tab)` does NOT have a colored bottom border. Use `page.evaluate()` to read computed `border-bottom-color` on an inactive tab and verify it's transparent or matches the default background.

5. **"BUG-08: panel chevron icons visible"** — Open a seed object to populate the right pane sections. Look for `.right-section-chevron` elements (or similar collapsible section chevrons). Verify at least one is visible and has non-zero dimensions (`boundingBox()` with width > 0 and height > 0).

6. **"BUG-09: concept search and linking"** — Open the command palette with Ctrl+K. Type a concept name like "Knowledge Management" (from `SEED.concepts.km.label`). Verify that search results appear containing the concept name. This tests that FTS search finds concepts. Close the palette. Alternatively, test concept linking by opening a Note in edit mode, finding a relationship/link field, and searching for a concept — but the command palette search is simpler and more reliable.

All tests use default theme only (no dual-theme testing per user decision). Use `const BASE_URL = process.env.TEST_BASE_URL || 'http://localhost:3901';` at module level.

IMPORTANT: For CSS property assertions, use computed RGB values (not hex or CSS variable names). For the accent color test (BUG-04), compare that two values differ rather than checking exact colors.

    def to_dict(self) -> dict[str, Any]: ...
    @classmethod
    def from_dict(cls, data: dict[str, Any]) -> MappingConfig: ...

Follow the exact serialization pattern from VaultScanResult.to_dict/from_dict. TypeMapping serializes as `{"target_type_iri": str, "target_type_label": str}`. None values serialize as JSON null. PropertyMapping serializes similarly with the `source` field.

Update NoteTypeGroup serialization in VaultScanResult.to_dict() to include `"frontmatter_keys"` and in from_dict() to deserialize it. During note classification, maintain a dict `group_fm_keys: dict[tuple[str, str], dict[str, dict]]` keyed by `(type_name, signal_source)`.   <name>Task 2: Add wizard step and auto-save router endpoints</name>
  <files>backend/app/obsidian/router.py</files>
  <action>
Add 5 new endpoints to the obsidian router. Inject ShapesService via `get_shapes_service` from dependencies.py.

Helper functions (private):
- `_load_mapping(user, import_id) -> MappingConfig`: Load mapping_config.json from import dir, return empty MappingConfig if not exists.
- `_save_mapping(user, import_id, config)`: Write mapping_config.json to import dir.
- `_load_scan_result(user, import_id) -> VaultScanResult`: Load scan_result.json (already done in existing get_results, refactor to reuse).

**Step endpoints** (GET, return HTML partials):

1. `GET /{import_id}/step/type-mapping` — Load scan_result and mapping_config. Call `await shapes_service.get_types()` for dropdown options. Return `obsidian/partials/type_mapping.html` with context: scan_result, mapping_config, available_types, import_id, current_step=3.

2. `GET /{import_id}/step/property-mapping` — Load scan_result and mapping_config. For each mapped type (non-null in type_mappings), call `await shapes_service.get_form_for_type(type_iri)` to get PropertyShape list. Handle many-to-one: merge frontmatter_keys from all groups mapped to the same type (union keys, sum counts, combine samples). Return `obsidian/partials/property_mapping.html` with context: per-type property tables data, mapping_config, import_id, current_step=4.

3. `GET /{import_id}/step/preview` — Load scan_result and mapping_config. For each mapped type, take up to 3 sample notes from the group's sample_notes, re-read their frontmatter via `python-frontmatter`, apply property mappings to produce key-value pairs, check if body content exists. Return `obsidian/partials/preview.html` with context: previews list, import_id, current_step=5.

**Auto-save endpoints** (POST, return minimal response):

4. `POST /{import_id}/mapping/type` — Accept form data: `group_key` (str) and `target_type` (str, empty = skip). Load mapping, update type_mappings entry (TypeMapping if target_type provided, None if empty). Save. Return 200 with empty body or small status indicator HTML. Use `hx-swap="none"` on client.

5. `POST /{import_id}/mapping/property` — Accept form data: `type_iri`, `fm_key`, `target_property` (IRI or empty), `property_label`, `source` ("shacl" or "custom"). Load mapping, update property_mappings[type_iri][fm_key]. Save. Return 200.

All endpoints should use the htmx partial response pattern: `block_name = "content" if request.headers.get("HX-Request") else None`. Templates include the step_bar partial with correct current_step.

For the ShapesService dependency, add it as a parameter: `shapes_service: ShapesService = Depends(get_shapes_service)`. Check if `get_shapes_service` exists in dependencies.py; if not, create it following the existing dependency injection pattern (likely needs TriplestoreClient).

<task type="auto">
  <name>Task 1: Side-by-side preview with toggle and live sync</name>
  <files>frontend/static/js/vfs-browser.js, frontend/static/css/vfs-browser.css</files>
  <action>
1. In `vfs-browser.js`, modify the `openFile()` function to create a side-by-side layout instead of the current separate preview/source view tabs:
   - Replace the existing mutually-exclusive Preview/Source tab approach with a single container that has a left pane (CodeMirror) and a right pane (rendered markdown), with a drag handle between them.
   - The right pane (preview) is hidden by default. A toggle button in the file tab bar shows/hides it.
   - Layout HTML structure:
     ```
     <div class="vfs-split-container" id="vfs-split-{pathId}">
       <div class="vfs-split-left">
         <div class="vfs-cm-container" id="vfs-cm-{pathId}"></div>
       </div>
       <div class="vfs-split-handle" style="display:none"></div>
       <div class="vfs-split-right" style="display:none">
         <div class="vfs-preview-container" id="vfs-preview-{pathId}">
           <div class="markdown-body"></div>
         </div>
       </div>
     </div>
     ```
   - Only show the preview toggle button for markdown files (`.md` extension). For non-markdown files, the button is hidden.

2. Add a preview toggle button in the file tab bar area. Each file tab should have a small icon button (e.g., Lucide `book-open` or `eye` icon) that toggles the preview pane:
   - When toggled ON: show the `.vfs-split-handle` and `.vfs-split-right`, set left pane width to 50%.
   - When toggled OFF: hide handle and right pane, set left pane width to 100%.
   - Store toggle state per file in a variable (default: off).

3. Add a debounced `EditorView.updateListener` extension to the CodeMirror editor instances. When `update.docChanged` is true and preview is visible:
   - After 300ms debounce, get editor content via `view.state.doc.toString()`.
   - Render markdown using `marked.parse()` + `DOMPurify.sanitize()` (the same pattern used in `markdown-render.js` and already used in the existing VFS preview).
   - Set the `.markdown-body` innerHTML in the preview pane.

4. Extend the existing VFS browser's `initResize()` pattern (or create a new one) for the split handle between editor and preview:
   - Mousedown on `.vfs-split-handle` starts drag.
   - Mousemove adjusts widths of `.vfs-split-left` and `.vfs-split-right` as percentages.
   - Mouseup stops drag.
   - Minimum width: 20% for each pane.

5. In `vfs-browser.css`, add split layout styling:
   ```css
   .vfs-split-container {
       display: flex;
       flex: 1;
       overflow: hidden;
   }
   .vfs-split-left {
       flex: 1;
       overflow: hidden;
       min-width: 20%;
   }
   .vfs-split-handle {
       width: 4px;
       background: var(--color-border);
       cursor: col-resize;
       flex-shrink: 0;
   }
   .vfs-split-handle:hover {
       background: var(--color-accent);
   }
   .vfs-split-right {
       flex: 1;
       overflow: auto;
       min-width: 20%;
       padding: 12px;
   }
   ```

Per CONTEXT.md: Preview toggled via button, default off. Side-by-side horizontal split. Uses existing marked.js + DOMPurify.
Per Research Pitfall 5: Add debounced updateListener so preview stays in sync with editor.

<task type="auto">
  <name>Task 2: File operation polish (dirty indicator, icons, loading states, error feedback) and WebDAV help</name>
  <files>frontend/static/js/vfs-browser.js, frontend/static/css/vfs-browser.css, backend/app/templates/browser/vfs_browser.html</files>
  <action>
1. **Dirty indicator (dot on tab):** In `vfs-browser.js`, track dirty state per file. When CodeMirror content changes (via the `EditorView.updateListener` already added for preview sync), mark the file as dirty. Show a small dot indicator on the file tab (add a `.vfs-tab-dirty` class to the tab element, styled as a small colored dot before/after the filename). When the file is saved successfully, remove the dirty indicator.

2. **Saved flash:** After a successful PUT (save), briefly show a "Saved" indicator. Options:
   - Add a `.vfs-saved-flash` element near the tab or toolbar that fades in/out over 1.5 seconds.
   - OR change the dirty dot to a checkmark briefly, then hide.
   - Use CSS animation: `opacity: 1` -> `opacity: 0` over 1.5s with `animation-fill-mode: forwards`.

3. **Edit/read toggle icons:** Replace the existing text-based edit/save buttons with Lucide icons:
   - Read mode (locked): `<i data-lucide="lock"></i>` with tooltip "Click to edit"
   - Edit mode (unlocked): `<i data-lucide="lock-open"></i>` with tooltip "Click to lock"
   - Per CLAUDE.md: Size Lucide icons via CSS (`flex-shrink: 0`), not inline styles.
   - Call `lucide.createIcons()` after inserting new icon elements.

4. **Loading states (spinners):**
   - When loading the VFS tree (`loadTree()`): Show a CSS spinner in the tree container. Use a CSS-animated spinner (a rotating border-based spinner) -- simpler and more reliable than dynamically inserting Lucide loader icons.
   - When loading file content (`openFile()`): Show spinner in the content area while fetching.
   - When saving: Show spinner or disable the save button during PUT.
   - CSS spinner pattern:
     ```css
     .vfs-spinner {
         width: 20px; height: 20px;
         border: 2px solid var(--color-border);
         border-top-color: var(--color-accent);
         border-radius: 50%;
         animation: spin 0.6s linear infinite;
     }
     @keyframes spin { to { transform: rotate(360deg); } }
     ```

5. **Error feedback:** On save failure (PUT returns non-2xx), show an inline error message or toast near the file tab area. Use a simple approach:
   - Create a `showVfsToast(message, type)` function that shows a temporary message (success/error).
   - Styled as a small notification bar at the top of the VFS content area.
   - Auto-dismiss after 3 seconds.
   - Red for errors, green/neutral for success.

6. **Inline VFS help:** In `vfs_browser.html`, add a collapsible help banner at the top of the VFS browser (below the tree header, above the tree content):
   - Small info icon + "Connect your OS to this WebDAV endpoint" text.
   - Clicking expands to show:
     - The WebDAV URL (derive from current location: `{origin}/vfs/dav/`)
     - Brief OS-specific mount instructions:
       - **macOS**: Finder > Go > Connect to Server > paste URL
       - **Windows**: File Explorer > Map Network Drive > paste URL
       - **Linux**: `mount -t davfs {url} /mnt/sempkm` or use file manager
   - Collapsible via a `<details>` element for simplicity. Per CONTEXT.md: "small info section or tooltip" -- a `<details>` with a summary is clean and native.
   - Style with subtle background and small font.
   - Per CONTEXT.md deferred: Do NOT add a Driver.js guided tour -- only inline help text.

7. In `vfs-browser.css`, add styles for all new elements:
   - `.vfs-tab-dirty::after` (small colored dot)
   - `.vfs-saved-flash` (fade animation)
   - `.vfs-spinner` (rotating spinner)
   - `.vfs-toast` (notification bar)
   - `.vfs-help-banner` (collapsible help section)
   - `.vfs-toolbar-btn svg` (icon sizing for lock/lock-open)

Per CLAUDE.md: Size Lucide icons via CSS, flex-shrink: 0.
Per CONTEXT.md: All four polish areas (save indicator, edit/read toggle, loading states, error feedback) plus inline VFS help.

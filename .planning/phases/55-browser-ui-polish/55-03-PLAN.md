   ```python
   @router.get("/edge-provenance")
   async def get_edge_provenance(
       request: Request,
       subject: str = Query(...),
       predicate: str = Query(...),
       target: str = Query(...),
       source: str = Query("user"),
       user: User = Depends(get_current_user),
       client: TriplestoreClient = Depends(get_triplestore_client),
       prefix_registry: PrefixRegistry = Depends(get_prefix_registry),
   ):
   ```
   - Accept query params: `subject`, `predicate`, `target`, `source`.
   - Generate the predicate QName using `prefix_registry.to_qname(predicate)` (or equivalent). Check how prefix_registry is used elsewhere in the codebase.
   - If `source == "inferred"`: Return JSON `{"predicate_qname": qname, "source": "inferred", "description": "Inferred by OWL 2 RL reasoning"}`. No event lookup needed.
   - If `source == "user"`: Query event named graphs to find when this triple was created. Use a SPARQL query against event graphs:
     ```sparql
     SELECT ?event ?timestamp ?performedBy WHERE {
       GRAPH ?event {
         ?event <urn:sempkm:timestamp> ?timestamp .
         OPTIONAL { ?event <urn:sempkm:performedBy> ?performedBy }
         # Look for the direct triple in the event data
         <SUBJECT> <PREDICATE> <TARGET> .
       }
       FILTER(STRSTARTS(STR(?event), "urn:sempkm:event:"))
     }
     ORDER BY DESC(?timestamp)
     LIMIT 1
     ```
   - Also try searching for edge resources in event graphs (for edges created via `edge.create` command):
     ```sparql
     SELECT ?event ?timestamp ?performedBy WHERE {
       GRAPH ?event {
         ?event <urn:sempkm:timestamp> ?timestamp .
         OPTIONAL { ?event <urn:sempkm:performedBy> ?performedBy }
         ?edge a <urn:sempkm:Edge> ;
               <urn:sempkm:source> <SUBJECT> ;
               <urn:sempkm:predicate> <PREDICATE> ;
               <urn:sempkm:target> <TARGET> .
       }
       FILTER(STRSTARTS(STR(?event), "urn:sempkm:event:"))
     }
     ORDER BY DESC(?timestamp)
     LIMIT 1
     ```
   - Try the edge resource query first (more specific), fall back to direct triple query.
   - Return JSON: `{"predicate_qname": qname, "source": "user", "event_iri": event_iri, "timestamp": timestamp, "performed_by": performed_by_label}`.
   - If no event found, return partial data: `{"predicate_qname": qname, "source": "user", "event_iri": null, "timestamp": null, "performed_by": null}`.

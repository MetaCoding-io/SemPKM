2. In `workspace.js`, add multi-select state management:
   ```javascript
   var selectedIris = new Set();
   var lastClickedLeaf = null;

   function handleTreeLeafClick(e, iri, label) {
       if (e.shiftKey && lastClickedLeaf) {
           e.preventDefault();
           selectRange(lastClickedLeaf, iri);
       } else if (e.ctrlKey || e.metaKey) {
           e.preventDefault();
           toggleSelection(iri);
           lastClickedLeaf = iri;
       } else {
           clearSelection();
           openTab(iri, label);
           lastClickedLeaf = iri;
       }
       updateSelectionUI();
   }
   ```

3. Implement helper functions in `workspace.js`:
   - `selectRange(fromIri, toIri)`: Get all visible `.tree-leaf` elements via `document.querySelectorAll('#section-objects .tree-leaf')`. Find indices of fromIri and toIri by matching `data-iri`. Select all leaves between those indices (inclusive). The DOM order IS the visual order, so this works across type groups. Clear previous selection first, then add range.
   - `toggleSelection(iri)`: If iri is in selectedIris, remove it; otherwise add it.
   - `clearSelection()`: Clear the Set, remove `.selected` class from all tree-leaf elements.
   - `updateSelectionUI()`: Apply/remove `.selected` class on all `.tree-leaf[data-iri]` elements based on Set membership. Show/hide selection badge and trash button in header: if selectedIris.size > 0, show `#selection-badge` with count text "[N selected]", show `#bulk-delete-btn`, add `.has-selection` class to `.explorer-header-actions`. Otherwise hide them.

4. Listen for `htmx:afterSwap` on `#section-objects` to re-apply `.selected` class after tree partial swaps (type node expansion). This prevents selection from visually disappearing when a type group loads its children.

   ```python
   @router.post("/objects/delete")
   async def bulk_delete_objects(
       request: Request,

       client: TriplestoreClient = Depends(get_triplestore_client),
       event_store: EventStore = Depends(get_event_store),

   ```
   - Accept JSON body `{"iris": ["iri1", "iri2", ...]}`.
   - For each IRI, query ALL triples in `urn:sempkm:current` where that IRI is the subject: `SELECT ?p ?o WHERE { GRAPH <urn:sempkm:current> { <IRI> ?p ?o } }`.

   - Commit all operations in a single `event_store.commit()` call for atomicity.
   - Return `{"event_iri": str(event_result.event_iri), "deleted_count": len(iris)}`.
   - CRITICAL: Do NOT use raw SPARQL DELETE. Use `materialize_deletes` via EventStore to maintain the immutable audit trail (per Research Pitfall 1).

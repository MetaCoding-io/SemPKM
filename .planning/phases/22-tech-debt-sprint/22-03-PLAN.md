1. Add the `cachetools` import at the top of `backend/app/views/service.py`, after the existing imports:

```python
from cachetools import TTLCache
```

2. Modify the `ViewSpecService.__init__` method to accept optional `ttl` and `maxsize` parameters and initialize a TTLCache:

Change from:
```python
def __init__(self, client: TriplestoreClient, label_service: LabelService) -> None:
    self._client = client
    self._label_service = label_service
```

To:
```python
def __init__(
    self,
    client: TriplestoreClient,
    label_service: LabelService,
    ttl: int = 300,
    maxsize: int = 64,
) -> None:
    self._client = client
    self._label_service = label_service
    self._specs_cache: TTLCache[str, list[ViewSpec]] = TTLCache(
        maxsize=maxsize, ttl=ttl
    )
```

3. In `get_all_view_specs`, add cache check at the very beginning of the method (after the docstring, before line 72 `model_sparql = ...`):

```python
cache_key = "all_specs"
if cache_key in self._specs_cache:
    logger.debug("ViewSpec cache hit")
    return self._specs_cache[cache_key]
```

4. At the end of `get_all_view_specs`, just before the final `return specs` (line 139), add cache storage:

```python
self._specs_cache[cache_key] = specs
logger.info(

    len(specs), len(model_ids), self._specs_cache.timer() and int(self._specs_cache.ttl) or 300
)
```

Actually, simplify the log line -- `TTLCache.ttl` is a direct attribute:

```python
self._specs_cache[cache_key] = specs

    "ViewSpec cache miss -- loaded %d specs from %d model(s)",
    len(specs), len(model_ids),
```

5. Add an `invalidate_cache` method after `get_all_view_specs` (before `get_view_specs_for_type`):

This follows the exact pattern from LabelService which has `invalidate()` and `set_language_prefs()` methods that clear the cache.

Build the lint data model, storage layer, and REST API endpoints for querying structured SHACL validation results.

Output: New `backend/app/lint/` package with models, service, and router. Extended ValidationReport with structured triple generation. Working `/api/lint/results`, `/api/lint/status`, and `/api/lint/diff` endpoints.

   - Add `to_structured_triples(self, run_iri: str, trigger_source: str = "user_edit", source_model_map: dict[str, str] | None = None) -> list[tuple]` method to `ValidationReport`.
   - Generates run metadata triples: `<run_iri> a sempkm:LintRun ; sempkm:timestamp ; sempkm:conforms ; sempkm:triggerSource ; sempkm:violationCount ; sempkm:warningCount ; sempkm:infoCount`.
   - Generates per-result triples: `<urn:sempkm:lint-result:{uuid}> a sempkm:LintResult ; sempkm:inRun <run_iri> ; sh:focusNode ; sh:resultSeverity ; sh:resultPath ; sh:resultMessage ; sh:sourceShape ; sh:sourceConstraintComponent ; sempkm:sourceModel ; sempkm:orphaned false`.
   - Use existing `SEMPKM` namespace. Reuse W3C SHACL predicates where they exist. Add `_severity_uri()` helper mapping "Violation" -> `SH.Violation` etc.

4. Modify `backend/app/services/validation.py` `_store_report()`:
   - After storing raw report graph and summary triples, also store structured result triples.
   - Generate a new run IRI: `f"urn:sempkm:lint-run:{uuid.uuid4()}"`.
   - Call `report.to_structured_triples(run_iri, trigger_source)` to get triples.
   - INSERT DATA into `GRAPH <{run_iri}>` (run graph IRI = run IRI, convention).
   - Update latest run pointer in `urn:sempkm:validations`: DELETE old `<urn:sempkm:lint-latest> sempkm:latestRun ?old` and `sempkm:previousRun ?oldPrev`, INSERT new `latestRun` and `previousRun` (old latest becomes previous).
   - Add `trigger_source: str = "user_edit"` parameter to `_store_report()` and `validate()`.
   - Update `validate()` signature to accept `trigger_source` and pass it through.

5. Create `backend/app/lint/service.py` with `LintService`:
   - Constructor takes `TriplestoreClient` and `LabelService`.
   - `async def get_results(self, page=1, per_page=50, severity=None, object_type=None, run_id=None, detail=False) -> LintResultsResponse`: SPARQL query against latest run (or specified run_id) named graph with OFFSET/LIMIT pagination. Validate severity against SEVERITY_ALLOWLIST. Validate object_type starts with `http://` or `urn:`. Use COUNT query for total. Resolve labels via `LabelService.resolve_batch()`. If detail=False, exclude source_shape/constraint_component/source_model from response.
   - For object_type filter: query `urn:sempkm:current` graph for `?focusNode a <object_type>` and use VALUES clause to filter.
   - `async def get_status(self) -> LintStatusResponse`: Query run metadata from latest run pointer.
   - `async def get_diff(self) -> LintDiffResponse`: Query latest and previous run fingerprints (focus_node + severity + source_shape + constraint_component + path tuple). Compute set difference for new_issues and resolved_issues.
   - `async def get_latest_run_iri(self) -> str | None`: Query `<urn:sempkm:lint-latest> sempkm:latestRun ?run` from validations graph.

Create `backend/app/obsidian/` package with four files:

**models.py** — Dataclasses for scan results:
- `FrontmatterKeySummary(key: str, count: int, sample_values: list[str])` — up to 5 unique samples
- `TagSummary(tag: str, count: int)`
- `NoteTypeGroup(type_name: str, signal_source: str, count: int, sample_notes: list[str])` — up to 10 sample paths
- `ScanWarning(severity: str, category: str, message: str, file_path: str)` — categories: broken_link, empty_note, malformed_frontmatter
- `VaultScanResult` — complete result with vault_name, total_files, markdown_files, attachment_files, other_files, folders, type_groups, frontmatter_keys, tags, wikilink_targets, warnings, scan_duration_seconds, extract_path, import_id
- Add `to_dict()` and `from_dict()` class methods on VaultScanResult for JSON serialization

**scanner.py** — `VaultScanner` class:
- `__init__(self, extract_path: Path, import_id: str, broadcast: ScanBroadcast)`
- `async def scan(self) -> VaultScanResult` — runs `_do_scan()` in `asyncio.to_thread()` to avoid blocking event loop
- `_do_scan(self) -> VaultScanResult` — synchronous scan logic:
  1. Auto-detect vault root: if single top-level dir containing `.obsidian/`, use that; else use extract root
  2. Walk all files, classify as markdown (.md), attachment (images/PDFs/etc), or other
  3. For each markdown file: parse with `python-frontmatter`, extract wiki-links and tags from body
  4. Apply multi-signal type detection heuristic:
     - Signal 1 (highest): frontmatter fields ["type", "category", "class", "kind", "note_type", "notetype"] (case-insensitive key match)
     - Signal 2: immediate parent folder name (skip: attachments, assets, images, templates, archive, daily, inbox)
     - Signal 3: first tag from frontmatter `tags`/`tag` field or body tags
     - Signal 4: "Uncategorized" (no signal)
  5. Collect frontmatter keys across all notes with counts and up to 5 sample values each
  6. Collect tags (both frontmatter `tags`/`tag` fields and body `#tag` patterns) with counts
  7. Collect wiki-link targets with counts
  8. Generate warnings: empty notes (no body after frontmatter), malformed frontmatter (parse errors), broken wiki-links (targets that don't match any vault file)
  9. Push SSE progress every 10 files via broadcast
  10. Return VaultScanResult
- Wiki-link regex: `(?<!!)\[\[([^\]\|#]+)(?:#[^\]\|]*)?\s*(?:\|[^\]]*)?\]\]` — excludes `![[embeds]]`, captures target
- Tag regex: `(?:^|\s)#([a-zA-Z][a-zA-Z0-9_/-]*)` — preceded by whitespace or line start
- Strip fenced code blocks (```` ``` ```` pairs) before extracting links and tags
- Handle frontmatter `tags` field as list or comma-separated string; also check `tag` (singular)
- Recognize both `cssclass` and `cssclasses` as known Obsidian keys (not type indicators)

**broadcast.py** — Clone `backend/app/lint/broadcast.py` pattern:
- `ScanBroadcast` class with `asyncio.Queue` fan-out
- `subscribe() -> asyncio.Queue`, `unsubscribe(queue)`, `publish(event: dict)`
- `stream_sse(queue)` async generator yielding `data: {json}\n\n` lines with 30s keepalive
- Events: `scan_progress` (scanned, total, current_file), `scan_complete` (import_id), `scan_error` (message)
- Thread-safe publish via `loop.call_soon_threadsafe()` since scanner runs in thread

**__init__.py** — Empty or minimal imports.

**router.py** — FastAPI router at prefix `/browser/import`:
- `GET /browser/import` — Serves the import page (full page or htmx partial via `block_name="content"`, following browser/router.py pattern). Returns upload form template initially. Check if user has an existing import in progress — if so, show resume/discard options instead of upload form.
- `POST /browser/import/upload` — Accepts `UploadFile` (ZIP). Saves to `/app/data/imports/{user_id}/{timestamp}/vault.zip`. Extracts ZIP using `zipfile.ZipFile.extractall(filter='data')` in `asyncio.to_thread()`. Deletes ZIP after extraction. Returns htmx partial with scan trigger (shows "Scanning..." state with SSE connection info). Uses `asyncio.to_thread()` for both file write and extraction to avoid blocking event loop. Write file in chunks (8192 bytes) — do NOT use aiofiles (not in deps), use `asyncio.to_thread()` wrapping regular `open()`.
- `POST /browser/import/scan/{import_id}` — Triggers scan of extracted vault. Creates VaultScanner, runs `scanner.scan()`, persists result as JSON at `/app/data/imports/{user_id}/{timestamp}/scan_result.json`. Returns htmx partial with scan results dashboard.
- `GET /browser/import/scan/{import_id}/stream` — SSE endpoint. Subscribes to ScanBroadcast, returns StreamingResponse with `text/event-stream` content type.
- `POST /browser/import/{import_id}/discard` — Removes entire import directory (`shutil.rmtree`). Returns upload form partial.
- `GET /browser/import/{import_id}/results` — Returns persisted scan results as htmx partial (for re-rendering after page refresh).
- Store active ScanBroadcast instances in a module-level dict keyed by import_id.
- All endpoints require authenticated user via `get_current_user` dependency.
- import_id format: `{user_id}_{timestamp}` (e.g., "3_1709856000")

**Register router** in `backend/app/main.py` — add `from backend.app.obsidian.router import router as obsidian_router` and `app.include_router(obsidian_router)`. Follow the same pattern as the browser router registration.

**nginx.conf** — Add two location blocks BEFORE the catch-all `location /`:
```nginx
# Obsidian vault upload — unlimited size, no buffering
location /browser/import/upload {
    client_max_body_size 0;
    proxy_request_buffering off;
    proxy_pass http://api:8000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

# Obsidian scan SSE stream — no buffering for real-time events
location ~ /browser/import/scan/.*/stream$ {
    proxy_pass http://api:8000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header Connection '';
    proxy_http_version 1.1;
    chunked_transfer_encoding off;
    proxy_buffering off;
    proxy_cache off;
    add_header X-Accel-Buffering no;
}
```

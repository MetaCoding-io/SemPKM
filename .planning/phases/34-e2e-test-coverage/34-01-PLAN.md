**VFS WebDAV (modify `e2e/tests/vfs-webdav.spec.ts`):**

The critical issue: wsgidav uses `SemPKMWsgiAuthenticator` which ONLY accepts Basic auth (email + API token), NOT session cookies. The current tests send session cookies and hit 401, triggering test.skip().

Fix approach -- use a `vfsBasicAuth` custom fixture that creates an API token and returns the Basic auth header string:

```typescript
import { test as authTest, expect, OWNER_EMAIL, BASE_URL } from '../fixtures/auth';
import { request } from '@playwright/test';

const test = authTest.extend<{ vfsBasicAuth: string }>({
  vfsBasicAuth: async ({ ownerSessionToken }, use) => {
    const ctx = await request.newContext();
    const resp = await ctx.post(`${BASE_URL}/api/auth/tokens`, {
      headers: { Cookie: `sempkm_session=${ownerSessionToken}` },
      data: { name: 'e2e-vfs-' + Date.now() },
    });
    const body = await resp.json();
    const auth = 'Basic ' + Buffer.from(OWNER_EMAIL + ':' + body.token).toString('base64');
    await use(auth);
    // Cleanup: revoke the token
    await ctx.delete(`${BASE_URL}/api/auth/tokens/${body.id}`, {
      headers: { Cookie: `sempkm_session=${ownerSessionToken}` },
    });
    await ctx.dispose();
  },
});
```

Then apply these changes to the test file:

1. Replace ALL `Cookie: sempkm_session=${ownerSessionToken}` headers in WebDAV requests with `Authorization: ${vfsBasicAuth}`. Update every test function signature to destructure `vfsBasicAuth` instead of `ownerSessionToken`.

2. Remove ALL `test.skip()` calls entirely. Remove all conditional guards for 404/401/403. These features are live. If something fails, the test should FAIL, not skip.

3. Keep the existing test logic (OPTIONS, PROPFIND, directory listing, GET .md file, PUT rejection) but with the updated auth headers.

4. Keep `ownerPage` in test signatures since `ownerPage.context().request.fetch()` is the existing pattern for making HTTP calls.

  <action>
1. **backend/app/sparql/__init__.py**: Empty init file.

2. **backend/app/sparql/client.py**: SPARQL scoping utilities:
   - `CURRENT_GRAPH = "urn:sempkm:current"`
   - `def scope_to_current_graph(query: str) -> str`: Inject `FROM <urn:sempkm:current>` into SPARQL SELECT queries if no FROM or GRAPH clause is already present. Parse the query minimally: find the WHERE clause position, insert FROM before it. For CONSTRUCT queries, inject FROM similarly. If the query already contains `FROM` or `GRAPH <urn:sempkm:current>`, leave it unchanged. This prevents event graph data from leaking into user queries (per research Pitfall 3).
   - `def inject_prefixes(query: str) -> str`: Prepend common PREFIX declarations (from `rdf/namespaces.py` COMMON_PREFIXES) to the query if not already declared. This enables users to write queries using `rdf:`, `rdfs:`, `sempkm:`, etc. without declaring them.
   - Optional `all_graphs: bool` parameter support -- if explicitly requested, skip scoping (for admin/debug use).

3. **backend/app/sparql/router.py**: `GET` and `POST /api/sparql` endpoint:
   - POST accepts form-encoded `query` parameter (application/x-www-form-urlencoded) -- standard SPARQL protocol
   - Also accept JSON body `{"query": "..."}` for convenience from the dev console
   - Also support GET with `query` as a query parameter
   - Apply `inject_prefixes()` then `scope_to_current_graph()` to the query
   - Execute via `triplestore_client.query(scoped_query)`
   - Return the SPARQL results JSON with content type `application/sparql-results+json`
   - Handle triplestore errors (malformed SPARQL, timeout) and return appropriate HTTP error responses (400 for bad queries, 502 for triplestore errors)
   - Register router on the FastAPI app in main.py (add `app.include_router(sparql_router)`)
  </action>
  <verify>
With Docker stack running and some test data created via /api/commands:

1. Query via POST (form-encoded):
```bash
curl -X POST http://localhost:8000/api/sparql -d 'query=SELECT ?s ?p ?o WHERE { ?s ?p ?o } LIMIT 10'
```
Expect JSON results containing objects from current state graph, NOT event metadata.

2. Query with prefixes auto-injected:
```bash
curl -X POST http://localhost:8000/api/sparql -d 'query=SELECT ?s ?label WHERE { ?s rdfs:label ?label }'
```
Expect results without "prefix not defined" errors.

3. Verify event graphs are NOT leaked:
```bash
curl -X POST http://localhost:8000/api/sparql -d 'query=SELECT ?s WHERE { ?s a sempkm:Event }'
```
Expect empty results (events are in separate named graphs, not the current state).
  </verify>

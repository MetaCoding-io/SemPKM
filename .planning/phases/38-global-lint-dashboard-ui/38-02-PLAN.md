  <action>
Add three pieces of functionality to `workspace.js`:

**1. Health badge initialization and update function:**

Create a function `updateLintBadge(data)` that:
- Takes an object with `violation_count`, `warning_count`, `conforms` fields (from /api/lint/status or SSE event data).
- Finds `#lint-badge` element.
- If `violation_count > 0`: set text to violation count, remove all `lint-badge--*` classes, add `lint-badge--violations`, set `style.display = 'inline-block'`.
- Else if `warning_count > 0`: set text to warning count, add `lint-badge--warnings`, display inline-block.
- Else if `conforms === true`: set text to checkmark character, add `lint-badge--pass`, display inline-block.
- Else (conforms is null, no run yet): hide the badge (`style.display = 'none'`).

On page load (in the main init block, after `initCommandPalette()`), fetch `/api/lint/status` via `fetch()` and call `updateLintBadge()` with the response JSON to show the badge immediately.

**2. SSE listener for auto-refresh:**

Reuse the existing `window._lintSSE` EventSource connection (created by the per-object lint panel). If it does not exist yet, create it: `window._lintSSE = new EventSource('/api/lint/stream')`. Add an event listener for `validation_complete`:

```javascript
function initLintDashboardSSE() {
    if (!window._lintSSE) {
        window._lintSSE = new EventSource('/api/lint/stream');
    }
    window._lintSSE.addEventListener('validation_complete', function(e) {
        var data = JSON.parse(e.data);
        // Update the health badge
        updateLintBadge(data);
        // If the lint dashboard pane is currently visible, refresh its content
        var pane = document.getElementById('panel-lint-dashboard');
        if (pane && pane.classList.contains('active')) {
            htmx.ajax('GET', '/browser/lint-dashboard', {
                target: '#lint-dashboard-results',
                swap: 'innerHTML'
            });
        }
    });
}
```

Call `initLintDashboardSSE()` from the main initialization block. Place it BEFORE any per-object lint panel SSE setup so the shared connection is established first. If the per-object lint panel script also creates `window._lintSSE`, that's fine — the `if (!window._lintSSE)` guard prevents duplicates.

**3. Command Palette registration:**

In `initCommandPalette()`, add a command to the `ninja.data` array:
```javascript
{
    id: 'toggle-lint-dashboard',
    title: 'Toggle Lint Dashboard',
    section: 'View',
    handler: function() {
        panelState.activeTab = 'lint-dashboard';
        if (!panelState.open) panelState.open = true;
        _applyPanelState();
        savePanelState();
    }
}
```

Place it near the other "Toggle" commands in the array (like "Toggle Bottom Panel").

**Important:** When refreshing via SSE, include current filter values by reading the filter inputs from the pane. Build the URL with query params from `.lint-dashboard-filter-severity`, `.lint-dashboard-filter-type`, `.lint-dashboard-filter-search`, `.lint-dashboard-filter-sort` values so the refresh preserves the user's active filters:

```javascript
var params = new URLSearchParams();
var sev = pane.querySelector('[name="severity"]');
if (sev && sev.value) params.set('severity', sev.value);
var typ = pane.querySelector('[name="object_type"]');
if (typ && typ.value) params.set('object_type', typ.value);
var srch = pane.querySelector('[name="search"]');
if (srch && srch.value) params.set('search', srch.value);
var srt = pane.querySelector('[name="sort"]');
if (srt && srt.value) params.set('sort', srt.value);
htmx.ajax('GET', '/browser/lint-dashboard?' + params.toString(), {target: '#lint-dashboard-results', swap: 'innerHTML'});
```

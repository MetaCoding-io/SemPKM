2. In `backend/app/lint/service.py`, extend `get_results()` signature with `search: str | None = None` and `sort: str = "severity"` params:
   - **Search (LINT-06):** After the existing `severity_filter` and `object_type_filter` blocks, add a search filter. Sanitize the search string by escaping backslashes and quotes. Add a SPARQL `FILTER(CONTAINS(LCASE(?message), LCASE("{escaped_search}")))` clause to the count and results queries. Also match on `STR(?focusNode)` and `STR(?path)` with `||` so search covers message text, object IRI, and property path per LINT-06 requirement.
   - **Sort (LINT-07):** Replace the hardcoded `ORDER BY ?focusNode ?severity` with a dynamic ORDER BY based on the `sort` param. Allowlist: `"severity"` -> `ORDER BY ?severity ?focusNode`, `"object"` -> `ORDER BY ?focusNode ?severity`, `"path"` -> `ORDER BY ?path ?focusNode`. Default to severity sort. Validate against allowlist to prevent injection.

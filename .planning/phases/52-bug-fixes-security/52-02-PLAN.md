def require_role(*roles: str):
    """Factory returning a dependency that checks the user's role.
    Usage:
        user: User = Depends(require_role("owner", "member"))
    """
    async def _check_role(current_user: User = Depends(get_current_user)) -> User:
        if current_user.role not in roles:
            raise HTTPException(status_code=403, detail=f"Requires role: {', '.join(roles)}")
        return current_user
    return _check_role

**SPARQL client — member query safety check:**
In `backend/app/sparql/client.py`, add a function `check_member_query_safety(query: str) -> None` that:
1. Checks the query (case-insensitive) for `FROM` clauses using the existing regex pattern: `re.search(r'\bFROM\s+', upper)`
2. Checks for `GRAPH` clauses: `re.search(r'\bGRAPH\s+', upper)`
3. If either is found, raise `HTTPException(status_code=403, detail="FROM/GRAPH clauses not allowed for member role")`
4. Import `HTTPException` from `fastapi` at the top of the file

**SPARQL router — role enforcement:**
In `backend/app/sparql/router.py`:
1. Add import: `from fastapi import HTTPException` (already imported via FastAPI, check)
2. Add import: `from app.sparql.client import check_member_query_safety`
3. In `sparql_get()`: Keep `user: User = Depends(get_current_user)` (do NOT switch to require_role — we need differentiated behavior per role, not a blanket gate). Add role checks at the start of the function body:
   - If `user.role == "guest"`: raise `HTTPException(status_code=403, detail="SPARQL access denied")`
   - If `user.role == "member" and all_graphs`: raise `HTTPException(status_code=403, detail="Requires owner role for all_graphs")`

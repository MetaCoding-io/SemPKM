## Gaps

### VFS-01: VFS Browser Sidebar Link Does Not Navigate (User-Reported)

**Severity:** Blocker for VFS-01 requirement
**Phase:** 41
**Code path:** `_sidebar.html:79-83` onclick -> `workspace.js:701` `openVfsTab()` -> `workspace-layout.js:112` htmx.ajax

The code exists and the integration checker verified the full chain is wired correctly on the workspace page. However, the user reports clicking the VFS Browser sidebar link has no effect.

**Root cause candidates:**
1. User was on a non-workspace page where `window._dockview` is null (silent no-op)
2. Race condition: sidebar renders before workspace.js assigns `window.openVfsTab`
3. The `typeof openVfsTab==='function'` guard fails if workspace.js hasn't loaded yet

**Fix:** Debug why the link doesn't work on the workspace page. Add fallback behavior (navigate to workspace first, then open VFS tab) or show toast when workspace is unavailable.

## Gaps

### VFS-01: VFS Browser API Errors and UX Issues (User-Reported)

**Severity:** Blocker for VFS-01 requirement
**Phase:** 41
**Code path:** `_sidebar.html:79-83` onclick -> `workspace.js:701` `openVfsTab()` -> `workspace-layout.js:112` htmx.ajax -> `/browser/vfs` -> tree expansion endpoints

The VFS tab opens correctly from the sidebar link. However, two runtime issues prevent it from being functional:

1. **API 501 errors:** The VFS endpoints (`/browser/vfs`, `/browser/vfs/{model_id}/types`, `/browser/vfs/{model_id}/objects`) return 501 status codes, likely due to missing service dependencies or unimplemented backend logic.
2. **Infinite retry loop:** The frontend (htmx `hx-trigger="revealed"` or similar) retries the failing API calls in a loop, flooding the network with repeated 501 requests.
3. **UX awkwardness:** The VFS tree interface needs usability improvements (layout, interaction patterns, visual feedback).

**Fix:** Debug 501 errors in the VFS router endpoints, add error handling to prevent infinite retries, and improve the VFS tree UX.
